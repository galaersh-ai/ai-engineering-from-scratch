# Нормы и расстояния

> Ваша функция расстояния определяет, что означает «похожесть». Выберите её неправильно — и всё, что идёт дальше, начнёт ломаться.

**Тип:** Построение
**Язык:** Python
**Предварительные требования:** Фаза 1, Уроки 01 (Интуитивное понимание линейной алгебры), 02 (Векторы, матрицы и операции)
**Время:** ~90 минут

## Цели обучения

- Реализовать с нуля расстояния L1 и L2, косинусное сходство, расстояние Махаланобиса, сходство Яккарда и редакционное расстояние
- Выбирать подходящую метрику расстояния для заданной задачи машинного обучения и объяснять, почему альтернативы не работают
- Связывать нормы L1 и L2 с регуляризацией LASSO и Ridge и их геометрическими областями ограничений
- Показывать, как один и тот же набор данных может давать разные ближайшие соседи при разных метриках

## Задача

У Вас есть два вектора. Возможно, это эмбеддинги слов. Возможно, это профили пользователей. Возможно, это массивы пикселей. И Вам нужно понять: насколько они близки?

Ответ целиком зависит от того, какую функцию расстояния Вы выберете. Две точки данных могут быть ближайшими соседями по одной метрике и находиться далеко друг от друга по другой. От этого выбора зависят классификатор KNN, рекомендательная система, векторная база данных, алгоритм кластеризации и даже функция потерь. Ошибитесь здесь — и модель будет оптимизировать не то, что Вам нужно.

Универсально лучшей метрики не существует. L2 хорошо работает для пространственных данных. Косинусное сходство доминирует в NLP. Яккард подходит для множеств. Редакционное расстояние — для строк. Махаланобис учитывает корреляции. Вассерштейново расстояние описывает перенос вероятностной массы. Каждая метрика кодирует своё представление о том, что считать «похожестью».

В этом уроке Вы реализуете основные метрики расстояния с нуля, разберёте, когда каждая из них уместна, и увидите, как один и тот же набор данных даёт совершенно разных ближайших соседей в зависимости от выбранной метрики.

## Концепция

### Нормы: измерение величины вектора

Норма измеряет "размер" вектора. Каждая функция расстояния между двумя векторами может быть записана как норма их разности: d(a, b) = ||a - b||. Поэтому понимание норм — это понимание расстояний.

### Норма L1 (манхэттенское расстояние)

Норма L1 суммирует абсолютные значения всех компонентов.

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

Она называется манхэттенским расстоянием, потому что измеряет, как далеко Вы идёте по городской сетке, где можно двигаться только вдоль осей. Без диагоналей.

```
Point A = (1, 1)
Point B = (4, 5)

L1 distance = |4-1| + |5-1| = 3 + 4 = 7

On a grid, you walk 3 blocks east and 4 blocks north.
```

Когда использовать L1:
- Высокоуровневные разреженные данные (текстовые признаки, одиночное кодирование)
- Когда нужно устойчивость к выбросам (один большой различие не доминирует)
- Задачи выбора признаков (регуляризация L1 промышленность разреженности)

Связь с регуляризацией L1 (Lasso): добавление ||w||_1 к функции потерь накладывает штраф на сумму абсолютных значений весов. Это толкает малые веса до нуля, выполняя автоматический выбор признаков. Штраф L1 создает ромбовидные области ограничения в пространстве весов, и углы ромба лежат на осях, где некоторые веса равны нулю.

Связь с функциями потерь: среднее абсолютное отклонение (MAE) — это среднее L1 расстояние между прогнозами и целевыми значениями. Оно штрафует все ошибки линейно, делая его устойчивым к выбросам по сравнению с MSE.

### Норма L2 (евклидово расстояние)

Норма L2 — это прямолинейное расстояние. Квадратный корень из суммы квадратов компонентов.

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

Это то самое расстояние, которое Вы изучали в геометрии. Теорема Пифагора в n измерениях.

```
Point A = (1, 1)
Point B = (4, 5)

L2 distance = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

The straight line, cutting diagonally through the grid.
```

Когда использовать L2:
- Низкоуровневые или средние измерения непрерывных данных
- Когда масштабы признаков сравнимы
- Физические расстояния (пространственные данные, показания датчиков)
- Сходство изображений на уровне пикселей

Связь с регуляризацией L2 (Ridge): добавление ||w||_2^2 к функции потерь накладывает штраф на большие веса. В отличие от L1, она не толкает весы до нуля. Она уменьшает все весы пропорционально. Штраф L2 создает круговые области ограничения, поэтому нет углов на осях. Весы становятся малыми, но редко точно равными нулю.

Связь с функциями потерь: среднее квадратичное отклонение (MSE) — это среднее квадратов L2 расстояний. Квадратирование штрафует большие ошибки более серьезно, чем малые.

```
MAE (L1 loss):  |y - y_hat|         Linear penalty. Robust to outliers.
MSE (L2 loss):  (y - y_hat)^2       Quadratic penalty. Sensitive to outliers.
```

### Нормы Lp: общая семейство

L1 и L2 являются специальными случаями нормы Lp:

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

Разные значения p создают разные формы "единичных шаров" (множество всех точек на расстоянии 1 от начала координат):

```
p=1:    Diamond shape      (corners on axes)
p=2:    Circle/sphere      (the usual round ball)
p=3:    Superellipse       (rounded square)
p=inf:  Square/hypercube   (flat sides along axes)
```

### Норма L-бесконечность (Чебышёвское расстояние)

При p, стремящемся к бесконечности, норма Lp сходится к максимальному абсолютному компоненту.

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

Расстояние между двумя точками определяется одним измерением, где они различаются больше всего. Все остальные измерения игнорируются.

```
Point A = (1, 1)
Point B = (4, 5)

L-inf distance = max(|4-1|, |5-1|) = max(3, 4) = 4
```

Когда использовать L-бесконечность:
- Когда важен худший случай отклонения в любом измерении
- Доски для игр (король в шахматах перемещается по L-бесконечности: один шаг в любую сторону стоит 1)
- Технические допуски (каждое измерение должно быть в пределах нормы)

### Косинусная похожесть и косинусное расстояние

Косинусная похожесть измеряет угол между двумя векторами, игнорируя их величины.

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

Она изменяется от -1 (противоположные направления) до +1 (одинаковые направления). Перпендикулярные векторы имеют косинусную похожесть 0.

Косинусное расстояние преобразует это в расстояние: косинусное расстояние = 1 - косинусная похожесть. Это изменяется от 0 (идентичное направление) до 2 (противоположное направление).

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

Почему косинусная похожесть доминирует в NLP и эмбеддингах: в тексте длина документа не должна влиять на сходство. Документ о кошках, который в два раза длиннее другого документа о кошках, всё ещё должен быть "похожим". Косинусная похожесть игнорирует величину (длину) и заботится только о направлении. Два документа с одинаковым распределением слов, но разной длиной, указывают в одном направлении и получают косинусную похожесть 1.0.

Когда использовать косинусную похожесть:
- Сходство текстов (векторы TF-IDF, эмбеддинги слов, эмбеддинги предложений)
- Любая область, где величина — шум, а направление — сигнал
- Системы рекомендаций (векторы предпочтений пользователей)
- Поиск по эмбеддингам (векторные базовые наборы данных почти всегда используют косинус или скалярное произведение)

### Скалярное произведение похожести против косинусной похожести

Скалярное произведение двух векторов:

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(angle)
```

Косинусная похожесть — это скалярное произведение, нормированное обоими величинами. Когда оба вектора уже единичные (величина равна 1), скалярное произведение и косинусная похожесть одинаковы.

```
If ||a|| = 1 and ||b|| = 1:
    a . b = cos(angle between a and b)
```

Когда они различаются: скалярное произведение включает информацию о величине. Вектор с большей величиной получает более высокий балл по скалярному произведению. Это важно для некоторых систем извлечения, где вы хотите, чтобы "популярные" элементы ранжировались выше. Величина действует как неявный сигнал качества или важности.

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

Both agree on direction, but dot product also reflects magnitude.
```

На практике:
- Используйте косинусную похожесть, когда хотите чистую похожесть по направлению
- Используйте скалярное произведение, когда величины содержат значимую информацию
- Многие векторные базовые наборы данных (Pinecone, Weaviate, Qdrant) позволяют выбирать между ними
- Если ваши эмбеддинги L2-нормированы, выбор не имеет значения

### Расстояние Махаланобиса

Евклидово расстояние считает все измерения равными. Но если ваши признаки коррелированы или имеют разные масштабы, L2 дает ложные результаты.

Расстояние Махаланобиса учитывает структуру ковариации данных.

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

где S — матрица ковариации данных.

Интуитивно: расстояние Махаланобиса сначала декоррелирует и нормирует данные (белирование), затем вычисляет L2 расстояние в этом преобразованном пространстве. Если S — единичная матрица (нерассеянные, однородные признаки), расстояние Махаланобиса уменьшается до евклидова.

```
Example: height and weight are correlated.
Someone 6'2" and 180 lbs is not unusual.
Someone 5'0" and 180 lbs is unusual.

Euclidean distance might say they are equally far from the mean.
Mahalanobis distance correctly identifies the second as an outlier
because it accounts for the height-weight correlation.
```

Когда использовать расстояние Махаланобиса:
- Обнаружение выбросов (точки с большим расстоянием Махаланобиса от среднего — это выбросы)
- Классификация при наличии различных масштабов и корреляций признаков
- Когда у вас достаточно данных для оценки надежной матрицы ковариации
- Контроль качества в производстве (многомерный мониторинг процесса)

### Сходство Яккарда (для множеств)

Сходство Яккарда измеряет степень пересечения двух множеств.

```
J(A, B) = |A intersect B| / |A union B|
```

Оно изменяется от 0 (пересечения нет) до 1 (множества идентичны). Расстояние Яккарда = 1 - сходство Яккарда.

```
A = {cat, dog, fish}
B = {cat, bird, fish, snake}

Intersection = {cat, fish}         size = 2
Union = {cat, dog, fish, bird, snake}  size = 5

Jaccard similarity = 2/5 = 0.4
Jaccard distance = 0.6
```

Когда использовать сходство Яккарда:
- Сравнение множеств тегов, категорий или признаков
- Похожесть документов на основе присутствия слов (не частоты)
- Обнаружение близких дубликатов (приближенное значение похожести Яккарда с помощью MinHash)
- Сравнение двоичных векторов признаков (данные о наличии/отсутствии)

### Редакционное расстояние (расстояние Левенштейна)

Редакционное расстояние считает минимальное количество односимвольных операций, необходимых для преобразования одной строки в другую. Операции: вставка, удаление или замена.

```
"kitten" -> "sitting"

kitten -> sitten  (substitute k -> s)
sitten -> sittin  (substitute e -> i)
sittin -> sitting (insert g)

Edit distance = 3
```

Вычисляется с помощью динамического программирования. Заполняет матрицу, где элемент (i, j) — это редакционное расстояние между первыми i символами строки A и первыми j символами строки B.

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

Когда использовать редакционное расстояние:
- Проверка орфографии и исправление ошибок
- Выравнивание последовательностей ДНК (с взвешенными операциями)
- Приближенное сравнение строк
- Удаление дубликатов в грязных текстовых данных

### KL Divergence (не расстояние, но используется как такое)

KL Divergence измеряет, насколько одна вероятностная распределение отличается от другой. Обсуждается в уроке 09, но относится к этому обсуждению потому что люди используют его как "расстояние", несмотря на то, что это не так.

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

Критическое свойство: KL Divergence НЕ симметрично.

```
D_KL(P || Q) != D_KL(Q || P)
```

Это означает, что он не удовлетворяет базовому требованию метрики расстояния. Он также не удовлетворяет неравенству треугольника. Это дивергенция, а не расстояние.

Прямая KL (D_KL(P || Q)) — "поиск средних": Q пытается охватить все моды P.
Обратная KL (D_KL(Q || P)) — "поиск мод": Q фокусируется на одной моде P.

Когда вы видите KL Divergence:
- VAEs (термин в ELBO толкает распределение потенциальных переменных к априорному)
- Обучение с уменьшением знаний (студент пытается соответствовать распределению учителя)
- RLHF (штраф KL сохраняет настроенную модель близкой к базовой модели)
- Методы градиентного обновления политики

### Расстояние Вассерштейна (расстояние переноса масс)

Расстояние Вассерштейна измеряет минимальное "работу", необходимую для преобразования одной вероятностной распределения в другое. Представьте себе: если одно распределение — это куча земли, а другое — яма, сколько земли нужно переместить и насколько далеко?

```
W(P, Q) = inf over all transport plans gamma of E[d(x, y)]
```

Для одномерных распределений оно упрощается до интеграла абсолютной разности кумулятивных функций распределения:

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

Почему Вассерштейн важно:
- Это настоящая метрика (симметрична, удовлетворяет неравенству треугольника)
- Она предоставляет градиенты даже тогда, когда распределения не пересекаются (KL Divergence стремится к бесконечности)
- Эта свойство сделало его центральным для Вассерштейновских GANов (WGAN), которые решили проблему нестабильного обучения оригинальных GAN

```
Distributions with no overlap:

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL divergence: infinity (log of zero)
Wasserstein: 4 (move all mass 4 bins)

Wasserstein gives a meaningful gradient. KL does not.
```

Когда использовать Вассерштейн:
- Обучение GAN (WGAN, WGAN-GP)
- Сравнение распределений, которые могут не пересекаться
- Задачи оптимального транспорта
- Поиск изображений (сравнение гистограмм цветов)

### Почему разные задачи требуют разных расстояний

| Задача | Лучшее расстояние | Почему |
|------|--------------|-----|
| Сходство текстов | Косинусное расстояние | Величина — шум, направление — смысл |
| Сравнение пикселей изображений | L2 | Пространственные отношения важны, признаки сравнимого масштаба |
| Разреженные высокомерные признаки | L1 | Устойчиво к выбросам, не усиливает редкие большие различия |
| Пересечение множеств (теги, категории) | Яккард | Данные являются естественными множествами, а не векторами |
| Сравнение строк | Редакционное расстояние | Операции соответствуют интуитивному пониманию редактирования |
| Обнаружение выбросов | Расстояние Махаланобиса | Учитывает корреляции признаков и масштабы |
| Сравнение распределений | KL Divergence | Измеряет потерянную информацию от использования Q вместо P |
| Обучение GAN | Вассерштейн | Предоставляет градиенты даже когда распределения не пересекаются |
| Эмбеддинги (векторные базовые наборы данных) | Косинусное расстояние или скалярное произведение | Эмбеддинги обучены для кодирования смысла в направлении |
| Рекомендации | Скалярное произведение | Величина может кодировать популярность или уверенность |
| Последовательности ДНК | Взвешенное редакционное расстояние | Замены стоят по-разному в зависимости от пары нуклеотидов |
| Контроль качества производства | L-бесконечность | Худший случай отклонения в любом измерении важен |

### Связь с функциями потерь

Функции потерь — это расстояния, примененные к прогнозам и целевым значениям.

```
Loss function       Distance it uses       Behavior
MSE                 L2 squared             Penalizes large errors heavily
MAE                 L1                     Penalizes all errors equally
Huber loss          L1 for large errors,   Best of both: robust to outliers,
                    L2 for small errors    smooth gradient near zero
Cross-entropy       KL divergence          Measures distribution mismatch
Hinge loss          max(0, margin - d)     Only penalizes below margin
Triplet loss        L2 (typically)         Pulls positives close, pushes
                                           negatives away
Contrastive loss    L2                     Similar pairs close, dissimilar
                                           pairs beyond margin
```

### Связь с регуляризацией

Регуляризация добавляет штраф на норму весов к функции потерь.

```
L1 regularization (Lasso):   loss + lambda * ||w||_1
  -> Sparse weights. Some weights become exactly zero.
  -> Automatic feature selection.
  -> Solution has corners (non-differentiable at zero).

L2 regularization (Ridge):   loss + lambda * ||w||_2^2
  -> Small weights. All weights shrink toward zero.
  -> No feature selection (nothing goes to exactly zero).
  -> Smooth solution everywhere.

Elastic Net:                  loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> Combines sparsity of L1 with stability of L2.
  -> Groups of correlated features are kept or dropped together.
```

Почему L1 приводит к разреженности, а L2 — нет: представьте область ограничения в пространстве весов. L1 — ромб, L2 — круг. Контуры функции потерь (эллипсы) наиболее вероятно касаются ромба на углу, где один из весов равен нулю. Они касаются круга на гладкой точке, где оба веса не равны нулю.

### Поиск ближайших соседей

Каждая функция расстояния подразумевает задачу поиска ближайшего соседа: данная точка запроса, найти ближайшие точки в наборе данных.

Точный поиск ближайшего соседа имеет сложность O(n * d) на запрос в наборе данных из n точек с d измерениями. Для больших наборов данных это слишком медленно.

Алгоритмы приближенного поиска ближайшего соседа (ANN) обменивают небольшую точность на огромные ускорения:

```
Algorithm         Approach                      Used by
KD-trees          Axis-aligned space partition   scikit-learn (low-dim)
Ball trees        Nested hyperspheres            scikit-learn (medium-dim)
LSH               Random hash projections        Near-duplicate detection
HNSW              Hierarchical navigable         FAISS, Qdrant, Weaviate
                  small-world graph
IVF               Inverted file index with       FAISS (billion-scale)
                  cluster-based search
Product quant.    Compress vectors, search       FAISS (memory-constrained)
                  in compressed space
```

HNSW (Иерархическая навигационная малая сеть) — доминирующий алгоритм современных базовых наборов данных. Он строит многослойный граф, где каждый узел соединен с его приближенными ближайшими соседями. Поиск начинается на верхнем уровне (реже, большие прыжки) и спускается к нижнему уровню (плотнее, короткие прыжки).

## Использование

Наиболее распространенная практическая задача: поиск похожих элементов в базе данных эмбеддингов.

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"Top 5 most similar to item 0: {top_k}")
print(f"Similarities: {similarities[top_k]}")
```

Когда вы вызываете `model.encode(text)` и затем ищете в базе данных эмбеддингов, это то, что происходит под капотом. Модель эмбеддинга преобразует текст в векторы. База данных эмбеддингов вычисляет косинусное расстояние (или скалярное произведение) между вашим запросным вектором и каждым сохраненным вектором, используя алгоритмы ANN для избежания проверки всех.

## Основные термины

| Термин | Что люди говорят | На что это действительно указывает |
|------|----------------|----------------------|
| Норма | "Размер вектора" | Функция, которая отображает вектор в неотрицательное скалярное значение, удовлетворяет неравенству треугольника, абсолютной однородности и равна нулю только для нулевого вектора |
| Норма L1 | "Манхэттенское расстояние" | Сумма абсолютных значений компонентов. Производит разреженность при оптимизации. Устойчиво к выбросам |
| Норма L2 | "Евклидово расстояние" | Квадратный корень из суммы квадратов компонентов. Прямолинейное расстояние в евклидовом пространстве |
| Нормы Lp | "Обобщенная норма" | p-й корень из суммы p-х степеней абсолютных значений компонентов. L1 и L2 — специальные случаи |
| Норма L-бесконечность | "Максимальная норма" или "Чебышёвское расстояние" | Максимальное абсолютное значение компонента. Предельное значение Lp при p, стремящемся к бесконечности |
| Косинусная похожесть | "Угол между векторами" | Скалярное произведение нормированное обоими величинами. Изменяется от -1 до +1. Игнорирует длину вектора |
| Косинусное расстояние | "1 минус косинусная похожесть" | Преобразует косинусную похожесть в расстояние. Изменяется от 0 до 2 |
| Скалярное произведение | "Ненормированное косинусное" | Сумма компонентных произведений. Равно косинусной похожести умноженной на обе величины |
| Расстояние Махаланобиса | "Корреляционная расстояние" | Евклидово расстояние в пространстве, которое было белено (декоррелировано и нормировано) с помощью матрицы ковариации данных |
| Похожесть Яккарда | "Пересечение множеств" | Размер пересечения деленный на размер объединения. Для множеств, а не векторов |
| Редакционное расстояние | "Расстояние Левенштейна" | Минимальные вставки, удаления и замены для преобразования одной строки в другую |
| KL Divergence | "Расстояние между распределениями" | Не настоящая метрика (не симметрична). Измеряет дополнительные биты от использования Q для кодирования P |
| Расстояние Вассерштейна | "Расстояние переноса масс" | Минимальная работа, необходимая для транспортировки массы из одного распределения в другое. Настоящая метрика |
| Приближенный поиск ближайших соседей | "ANN поиск" | Алгоритмы (HNSW, LSH, IVF), которые находят приближенно ближайшие точки намного быстрее точного поиска |
| HNSW | "Алгоритм базового набора данных" | Иерархическая навигационная малая сеть. Многослойный граф для быстрого приближенного поиска ближайших соседей |
| Регуляризация L1 | "Lasso" | Добавление нормы L1 весов к функции потерь. Толкает веса до нуля (разреженность) |
| Регуляризация L2 | "Ridge" или "weight decay" | Добавление квадратичной нормы L2 весов к функции потерь. Уменьшает веса к нулю без разреженности |
| Elastic Net | "L1 + L2" | Объединяет регуляризацию L1 и L2. Лучше обрабатывает связанные группы признаков, чем каждый отдельно |
