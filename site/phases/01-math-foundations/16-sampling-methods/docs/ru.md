# Методы выборки

> Выборка — это способ, которым ИИ исследует пространство возможностей.

**Тип:** Построение
**Язык:** Python
**Необходимые условия:** Фаза 1, Уроки 06-07 (Вероятность, Теорема Байеса)
**Время:** ~120 минут

## Цели обучения

- Реализовать обратное CDF, отвержение и важностную выборку с нуля, используя только равномерные случайные числа
- Создать выборку температуры, top-k и top-p (ядерная) для генерации токенов языковой модели
- Объяснить трюк переформулировки и почему он позволяет обратному распространению через выборку в VAE
- Выполнить Метрополиса-Хастингса MCMC, чтобы выбрать из ненормализованного целевого распределения

## Проблема

Языковая модель завершает обработку Вашего запроса и выдаёт вектор из 50 000 логитов — по одному на каждый токен в словаре. Теперь ей нужно выбрать один. Как именно это сделать?

Если модель всегда выбирает токен с наибольшей вероятностью, каждый ответ будет одинаковым: предсказуемым и скучным. Если же она выбирает токены равновероятно случайно, результат превратится в бессмысленный шум. Правильный ответ лежит где-то между этими крайностями — и именно выборка определяет, где именно.

Выборка не ограничивается генерацией текста. Методы обучения с подкреплением оценивают градиенты политики по выборкам траекторий. VAE учатся на скрытых представлениях, беря выборки из обученных распределений и распространяя градиенты через случайность. Диффузионные модели генерируют изображения, начиная с шума и постепенно устраняя его. Методы Монте-Карло оценивают интегралы, у которых нет замкнутой формулы. Алгоритмы MCMC исследуют высокоразмерные постериорные распределения, которые невозможно перечислить явно.

Любая генеративная система ИИ по сути является системой выборки. Стратегия выборки определяет качество, разнообразие и управляемость результата. В этом уроке Вы реализуете основные методы выборки с нуля: от равномерных случайных чисел до техник, на которых держатся современные LLM и генеративные модели.

## Концепция

### Почему важна выборка

Выборка появляется в четырёх фундаментальных ролях в ИИ и машинном обучении:

**Генерация.** Языковые модели, модели диффузии и GAN генерируют выходы через выборку. Алгоритм выборки напрямую контролирует креативность, согласованность и разнообразие. Температура, top-k и ядерная выборка — это регуляторы, которые инженеры поворачивают ежедневно.

**Обучение.** Стохастический градиентный спуск выбирает мини-батчи случайным образом. Dropout выбирает нейроны для деактивации случайным образом. Усиление данных выбирает случайные трансформации. Важностная выборка пересчитывает выборки, чтобы уменьшить дисперсию градиента в обучении с подкреплением (PPO, TRPO).

**Оценка.** Многие величины в машинном обучении не имеют замкнутых формул решений. Ожидаемая потеря по распределению данных, функция частиц энергетической модели, доказательство в байесовской инференции. Методы Монте-Карло аппроксимируют все это усреднением выборок.

**Исследование.** Алгоритмы MCMC исследуют постериорные распределения в байесовской инференции. Стратегии эволюционного обучения выбирают параметрические возмущения случайным образом. Выборка Томпсона сбалансировано сочетает исследование и эксплуатацию в задачах многократного выбора.

Основная проблема: можно выбирать напрямую только из простых распределений (равномерное, нормальное). Для всего остального нужно метод, который преобразует простые выборки в выборки из вашего целевого распределения.

### Равномерная случайная выборка

Все методы выборки начинаются здесь. Генератор равномерных случайных чисел производит значения на [0, 1), где каждый подинтервал одинаковой длины имеет равную вероятность.

```
U ~ Uniform(0, 1)

P(a <= U <= b) = b - a    for 0 <= a <= b <= 1

Properties:
  E[U] = 0.5
  Var(U) = 1/12
```

Чтобы равномерно выбрать из дискретного набора n элементов, сгенерируйте U и верните floor(n * U). Чтобы выбрать из непрерывного диапазона [a, b], вычислите a + (b - a) * U.

Ключевое понимание: одно равномерное случайное число содержит именно ту степень случайности, которая необходима для получения одной выборки из любого распределения. Трюк в том, чтобы найти правильное преобразование.

### Метод обратного CDF (обратная трансформация)

Функция распределения вероятностей (CDF) отображает значения на вероятности:

```
F(x) = P(X <= x)

Properties:
  F is non-decreasing
  F(-inf) = 0
  F(+inf) = 1
  F maps the real line to [0, 1]
```

Обратная функция CDF отображает вероятности обратно на значения. Если U ~ Uniform(0, 1), то X = F_inverse(U) следует целевому распределению.

```
Algorithm:
  1. Generate u ~ Uniform(0, 1)
  2. Return F_inverse(u)

Why it works:
  P(X <= x) = P(F_inverse(U) <= x) = P(U <= F(x)) = F(x)
```

**Пример распределения экспоненциального:** 

```
PDF: f(x) = lambda * exp(-lambda * x),   x >= 0
CDF: F(x) = 1 - exp(-lambda * x)

Solve F(x) = u for x:
  u = 1 - exp(-lambda * x)
  exp(-lambda * x) = 1 - u
  x = -ln(1 - u) / lambda

Since (1 - U) and U have the same distribution:
  x = -ln(u) / lambda
```

Это работает идеально, когда вы можете записать F_inverse в замкнутой форме. Для нормального распределения нет замкнутого обратного CDF, поэтому мы используем другие методы (Box-Muller или численное приближение).

**Дискретная версия:** Для дискретных распределений строим CDF как кумулятивную сумму, генерируем U и находим первый индекс, где кумулятивная сумма превышает U. Это то, как работает `sample_categorical` в Уроке 06.

### Метод отвержения

Когда вы не можете обратить CDF, но можете оценить целевую PDF до константы, метод отвержения работает.

```
Target distribution: p(x)  (can evaluate, possibly unnormalized)
Proposal distribution: q(x)  (can sample from)
Bound: M such that p(x) <= M * q(x) for all x

Algorithm:
  1. Sample x ~ q(x)
  2. Sample u ~ Uniform(0, 1)
  3. If u < p(x) / (M * q(x)), accept x
  4. Otherwise, reject and go to step 1

Acceptance rate = 1/M
```

Чем теснее граница M, тем выше ставка принятия. В низких измерениях (1-3) метод отвержения работает хорошо. В высоких измерениях ставка принимает экспоненциальное падение, потому что большая часть объёма предложения отвергается. Это проклятие размерности для метода отвержения.

**Пример: выборка из обрезанного нормального распределения.** Используйте равномерное предложение в обрезанном диапазоне. Оболочка M — это максимальная нормальная PDF в этом диапазоне.

**Пример: выборка из полуокружности.** Предложите равномерно внутри ограничивающего прямоугольника. Примите, если точка попадает внутрь полуокружности. Это то, как Монте-Карло вычисляет пи: ставка принятия равна соотношению площадей pi/4.

### Важностная выборка

Иногда вам не нужны выборки из целевого распределения p(x). Вам нужно оценить математическое ожидание под p(x), и у вас есть выборки из другого распределения q(x).

```
Goal: estimate E_p[f(x)] = integral of f(x) * p(x) dx

Rewrite:
  E_p[f(x)] = integral of f(x) * (p(x)/q(x)) * q(x) dx
            = E_q[f(x) * w(x)]

where w(x) = p(x) / q(x)  are the importance weights.

Estimator:
  E_p[f(x)] ~ (1/N) * sum(f(x_i) * w(x_i))    where x_i ~ q(x)
```

Это критично в обучении с подкреплением. В PPO (Proximal Policy Optimization), вы собираете траектории под старой политикой pi_old, но хотите оптимизировать новую политику pi_new. Важностный вес равен pi_new(a|s) / pi_old(a|s). PPO ограничивает эти веса, чтобы предотвратить слишком большое отклонение новой политики от старой.

Дисперсия оценки важностной выборки зависит от того, насколько похожи q и p. Если q очень отличается от p, несколько выборок получают огромные веса и доминируют над оценкой. Самонормализованная важностная выборка делит на сумму весов для уменьшения этой проблемы:

```
E_p[f(x)] ~ sum(w_i * f(x_i)) / sum(w_i)
```

### Метод Монте-Карло

Метод Монте-Карло аппроксимирует интегралы, усредняя случайные выборки. Закон больших чисел гарантирует сходимость.

```
Goal: estimate I = integral of g(x) dx over domain D

Method:
  1. Sample x_1, ..., x_N uniformly from D
  2. I ~ (Volume of D / N) * sum(g(x_i))

Error: O(1 / sqrt(N))   regardless of dimension
```

Степень ошибки не зависит от размерности. Именно поэтому методы Монте-Карло доминируют в высоких измерениях, где сеточные интеграции невозможны.

**Оценка пи:**

```
Sample (x, y) uniformly from [-1, 1] x [-1, 1]
Count how many fall inside the unit circle: x^2 + y^2 <= 1
pi ~ 4 * (count inside) / (total count)
```

**Оценка математических ожиданий:**

```
E[f(X)] ~ (1/N) * sum(f(x_i))    where x_i ~ p(x)

The sample mean converges to the true expectation.
Variance of the estimator = Var(f(X)) / N
```

### Марковские цепи Монте-Карло (MCMC): Метрополиса-Хастингса

Метод MCMC строит марковскую цепь, стационарное распределение которой — целевое распределение p(x). После достаточного числа шагов выборки из цепи (приближённо) являются выборками из p(x).

```
Target: p(x)  (known up to a normalizing constant)
Proposal: q(x'|x)  (how to propose the next state given the current state)

Metropolis-Hastings algorithm:
  1. Start at some x_0
  2. For t = 1, 2, ..., T:
     a. Propose x' ~ q(x'|x_t)
     b. Compute acceptance ratio:
        alpha = [p(x') * q(x_t|x')] / [p(x_t) * q(x'|x_t)]
     c. Accept with probability min(1, alpha):
        - If u < alpha (u ~ Uniform(0,1)): x_{t+1} = x'
        - Otherwise: x_{t+1} = x_t
  3. Discard first B samples (burn-in)
  4. Return remaining samples
```

Для симметричных предложений (q(x'|x) = q(x|x')), соотношение упрощается до p(x')/p(x). Это оригинальный алгоритм Метрополиса.

**Почему это работает.** Правило принятия обеспечивает детальную балансировку: вероятность быть в x и перейти к x' равна вероятности быть в x' и перейти к x. Детальная балансировка подразумевает, что p(x) является стационарным распределением цепи.

**Практические соображения:**
- Период разогрева: отбросьте ранние выборки до тех пор, пока цепь не достигнет равновесия
- Утончение: сохраняйте каждую k-ю выборку для уменьшения автокорреляции
- Шкала предложения: слишком мал и цепь медленно перемещается (высокая ставка принятия, медленное исследование); слишком велико и большинство предложений отвергаются (низкая ставка принятия, застревает на месте)
- Оптимальная ставка принятия для предложения Гаусса в высоких измерениях приблизительно равна 0.234

### Сэмплирование Гиббса

Сэмплирование Гиббса — это специальный случай MCMC для многомерных распределений. Вместо предложения перемещения во всех измерениях одновременно, оно обновляет одно переменное за раз из его условного распределения.

```
Target: p(x_1, x_2, ..., x_d)

Algorithm:
  For each iteration t:
    Sample x_1^{t+1} ~ p(x_1 | x_2^t, x_3^t, ..., x_d^t)
    Sample x_2^{t+1} ~ p(x_2 | x_1^{t+1}, x_3^t, ..., x_d^t)
    ...
    Sample x_d^{t+1} ~ p(x_d | x_1^{t+1}, x_2^{t+1}, ..., x_{d-1}^{t+1})
```

Сэмплирование Гиббса требует, чтобы вы могли выбирать из каждого условного распределения p(x_i | x_{-i}). Это просто для многих моделей:
- Байесовские сети: условные следуют из структуры графа
- Смесь гауссовых распределений: условные — это гауссовы
- Модели Исинга: каждая спиновая переменная зависит только от соседних

Ставка принятия всегда равна 1 (каждое предложение принимается), потому что выбор из точного условного распределения автоматически удовлетворяет детальной балансировке.

**Ограничение.** Когда переменные сильно коррелированы, сэмплирование Гиббса медленно перемешивается, так как обновление одной переменной за раз не может сделать большие диагональные шаги через распределение.

### Сэмплирование температуры (используется в LLM)

Языковые модели выдают логиты z_1, ..., z_V для каждого токена в словаре. Softmax преобразует эти логиты в вероятности. Температура масштабирует логиты перед softmax:

```
p_i = exp(z_i / T) / sum(exp(z_j / T))

T = 1.0: standard softmax (original distribution)
T -> 0:  argmax (deterministic, always picks highest logit)
T -> inf: uniform (all tokens equally likely)
T < 1.0: sharpens the distribution (more confident, less diverse)
T > 1.0: flattens the distribution (less confident, more diverse)
```

**Почему это работает.** Деление логитов на T < 1 усиливает различия между логитами. Если z_1 = 2 и z_2 = 1, деление на T = 0.5 даст z_1/T = 4 и z_2/T = 2, увеличивая разницу. После softmax токен с наибольшим логитом получает гораздо большую долю.

**На практике:**
- T = 0.0: жадное декодирование, лучше всего для фактических вопросов и ответов
- T = 0.3-0.7: немного креативно, хорошо для генерации кода
- T = 0.7-1.0: сбалансировано, хорошо для общего диалога
- T = 1.0-1.5: творческое письмо, мозговой штурм
- T > 1.5: всё более случайно, редко полезно

Температура не изменяет, какие токены возможны. Она изменяет вероятностную массу, распределённую на каждый токен.

### Сэмплирование top-k

Сэмплирование top-k ограничивает набор кандидатов токенами с наибольшими вероятностями, затем нормализует и выбирает из этого ограниченного набора.

```
Algorithm:
  1. Compute softmax probabilities for all V tokens
  2. Sort tokens by probability (descending)
  3. Keep only the top k tokens
  4. Renormalize: p_i' = p_i / sum(p_j for j in top-k)
  5. Sample from the renormalized distribution

k = 1:  greedy decoding
k = V:  no filtering (standard sampling)
k = 40: typical setting, removes long tail of unlikely tokens
```

Top-k предотвращает модель от выбора крайне маловероятных токенов (ошибки ввода, бессмысленные), которые существуют в длинном хвосте распределения словаря. Проблема: k фиксирован независимо от контекста. Когда модель уверена (один токен имеет 95% вероятности), k = 40 всё ещё позволяет 39 альтернатив. Когда модель неуверенна (вероятность распределена по 1000 токенам), k = 40 отбрасывает реальные варианты.

### Сэмплирование top-p (ядерное)

Сэмплирование top-p динамически изменяет размер набора кандидатов. Вместо того чтобы сохранять фиксированное количество токенов, оно сохраняет наименьший набор токенов, чья накопленная вероятность превышает p.

```
Algorithm:
  1. Compute softmax probabilities for all V tokens
  2. Sort tokens by probability (descending)
  3. Find smallest k such that sum of top-k probabilities >= p
  4. Keep only those k tokens
  5. Renormalize and sample

p = 0.9:  keeps tokens covering 90% of probability mass
p = 1.0:  no filtering
p = 0.1:  very restrictive, nearly greedy
```

Когда модель уверена, ядерное сэмплирование сохраняет немного токенов (может быть 2-3). Когда модель неуверенна, она сохраняет много (может быть 200). Эта адаптивная поведение является причиной того, что ядерное сэмплирование обычно производит лучший текст, чем top-k.

**Обычные комбинации:**
- Температура 0.7 + top-p 0.9: хорошая универсальная настройка
- Температура 0.0 (жадное): лучше всего для детерминированных задач
- Температура 1.0 + top-k 50: исходная настройка из статьи Fan et al. (2018)

Top-k и top-p могут быть комбинированы. Сначала примените top-k, затем top-p к оставшемуся набору.

### Трюк переформулировки (используется в VAE)

Вариационные автоэнкодеры (VAE) учатся кодировать входы в распределение в скрытом пространстве, брать из него выборку и декодировать её обратно. Проблема в том, что Вы не можете напрямую выполнять обратное распространение через саму операцию выборки.

```
Standard sampling (not differentiable):
  z ~ N(mu, sigma^2)

  The randomness blocks gradient flow.
  d/d_mu [sample from N(mu, sigma^2)] = ???
```

Трюк переформулировки разделяет случайность от параметров:

```
Reparameterized sampling:
  epsilon ~ N(0, 1)          (fixed random noise, no parameters)
  z = mu + sigma * epsilon   (deterministic function of parameters)

  Now z is a deterministic, differentiable function of mu and sigma.
  d(z)/d(mu) = 1
  d(z)/d(sigma) = epsilon

  Gradients flow through mu and sigma.
```

Это работает, потому что N(mu, sigma^2) имеет то же распределение, что и mu + sigma * N(0, 1). Ключевая идея: перенесите случайность к источнику без параметров (epsilon), а затем выразите выборку как дифференцируемую трансформацию параметров.

**В цикле обучения VAE:**
1. Энкодер выводит mu и log(sigma^2) для каждого входа
2. Выберите epsilon ~ N(0, 1)
3. Вычислите z = mu + sigma * epsilon
4. Декодируйте z для восстановления входа
5. Обратно распространяйтесь через шаги 4, 3, 2, 1 (возможное благодаря дифференцируемости шага 3)

Без трюка переформулировки VAE не могут быть обучены с помощью стандартного обратного распространения. Этот единственный инсайт сделал VAE практичными.

### Гумбель-Софтмакс (дифференцируемое категориальное сэмплирование)

Трюк переформулировки работает для непрерывных распределений (гауссовское). Для дискретных категориальных распределений нам нужен другой подход. Гумбель-Софтмакс предоставляет дифференцируемое приближение к категорическому сэмплированию.

**Трюк Гумбеля (не дифференцируемый):**

```
To sample from a categorical distribution with log-probabilities log(p_1), ..., log(p_k):
  1. Sample g_i ~ Gumbel(0, 1) for each category
     (g = -log(-log(u)), where u ~ Uniform(0, 1))
  2. Return argmax(log(p_i) + g_i)

This produces exact categorical samples.
```

**Гумбель-Софтмакс (дифференцируемое приближение):**

```
Replace the hard argmax with a soft softmax:
  y_i = exp((log(p_i) + g_i) / tau) / sum(exp((log(p_j) + g_j) / tau))

tau (temperature) controls the approximation:
  tau -> 0:  approaches a one-hot vector (hard categorical)
  tau -> inf: approaches uniform (1/k, 1/k, ..., 1/k)
  tau = 1.0: soft approximation
```

Гумбель-Софтмакс производит непрерывное расслабление дискретного выбора. Выход — это вероятностный вектор (мягкий одноразовый) вместо жёсткого одноразового. Градиенты проходят через softmax. В процессе обучения можно использовать "прямой" оценщик: используйте жёсткий argmax для прямого прохода, но мягкие градиенты Гумбель-Софтмакса для обратного прохода.

**Применения:**
- Дискретные скрытые переменные в VAE
- Поиск архитектуры нейронных сетей (выбор дискретных операций)
- Жёсткие механизмы внимания
- Обучение с подкреплением с дискретными действиями

### Стратифицированное сэмплирование

Стандартное сэмплирование Монте-Карло может оставить пробелы в пространстве выборок случайно. Стратифицированное сэмплирование обеспечивает равномерное покрытие, разделяя пространство на слои и выбирая из каждого.

```
Standard Monte Carlo:
  Sample N points uniformly from [0, 1]
  Some regions may have clusters, others gaps

Stratified sampling:
  Divide [0, 1] into N equal strata: [0, 1/N), [1/N, 2/N), ..., [(N-1)/N, 1)
  Sample one point uniformly within each stratum
  x_i = (i + u_i) / N   where u_i ~ Uniform(0, 1),  i = 0, ..., N-1
```

Стратифицированное сэмплирование всегда имеет дисперсию меньше или равную стандартному Монте-Карло:

```
Var(stratified) <= Var(standard Monte Carlo)

The improvement is largest when f(x) varies smoothly.
For piecewise-constant functions, stratified sampling is exact.
```

**Применения:**
- Численное интегрирование (квази-Монте-Карло)
- Разделение обучающих данных (обеспечение баланса классов в каждой фолде)
- Важностное сэмплирование со стратификацией (комбинация обоих методов)
- NeRF (Нейронные поля излучения) использует стратифицированное сэмплирование по лучам камеры

### Связь с моделями диффузии

Модели диффузии генерируют изображения через процесс выборки. Прямой процесс добавляет гауссовский шум к изображению за T шагов, пока оно не станет чистым шумом. Обратный процесс учится удалять шум, восстанавливая исходное изображение пошагово.

```
Forward process (known):
  x_t = sqrt(alpha_t) * x_{t-1} + sqrt(1 - alpha_t) * epsilon
  where epsilon ~ N(0, I)

  After T steps: x_T ~ N(0, I)  (pure noise)

Reverse process (learned):
  x_{t-1} = (1/sqrt(alpha_t)) * (x_t - (1 - alpha_t)/sqrt(1 - alpha_bar_t) * epsilon_theta(x_t, t)) + sigma_t * z
  where z ~ N(0, I)

  Each denoising step is a sampling step.
```

Связь с методами из этого урока:
- Каждый шаг удаления шума использует трюк переформулировки (выборка шума, применение детерминированной трансформации)
- Расписание шума {alpha_t} контролирует форму температурного аннеалинга
- Обучение использует метод Монте-Карло для аппроксимации ELBO (нижняя граница доказательства)
- Аномалийное сэмплирование в моделях диффузии — это марковская цепь (каждый шаг зависит только от текущего состояния)

Весь процесс генерации изображений итеративный: начните с шума, а затем на каждом шаге выбирайте немного менее шумную версию, условленную на обученной модели удаления шума.

```python
    inside = 0
    for _ in range(n):
        x = random.uniform(-1, 1)
        y = random.uniform(-1, 1)
        if x*x + y*y <= 1:

    return 4 * inside / n
```

```python
def metropolis_hastings(target_log_pdf, proposal_sample, proposal_log_pdf, x0, n_samples, burn_in):
    samples = []
    x = x0
    for i in range(n_samples + burn_in):
        x_new = proposal_sample(x)
        log_alpha = (target_log_pdf(x_new) + proposal_log_pdf(x, x_new)
                     - target_log_pdf(x) - proposal_log_pdf(x_new, x))
        if math.log(random.random()) < log_alpha:
            x = x_new
        if i >= burn_in:
            samples.append(x)
    return samples
```

```python
    samples = []
    for i in range(n_samples + burn_in):

        y = conditional_y_given_x(x)
        if i >= burn_in:
            samples.append((x, y))
```

### Шаг 8: Сэмплирование top-k и top-p

```python
def top_k_sample(logits, k):
    indexed = sorted(enumerate(logits), key=lambda x: -x[1])
    top = indexed[:k]
    top_logits = [l for _, l in top]
    probs = softmax(top_logits)
    idx = sample_from_probs(probs)
    return top[idx][0]

def top_p_sample(logits, p):
    probs = softmax(logits)
    indexed = sorted(enumerate(probs), key=lambda x: -x[1])
    cumsum = 0
    selected = []
    for token_idx, prob in indexed:
        cumsum += prob
        selected.append((token_idx, prob))
        if cumsum >= p:
            break
    sel_probs = [pr for _, pr in selected]
    total = sum(sel_probs)
    sel_probs = [pr / total for pr in sel_probs]
    idx = sample_from_probs(sel_probs)
    return selected[idx][0]
```

## Основные термины

| Термин | Что люди говорят | Что это действительно означает |
|------|----------------|----------------------|
| Выборка | "Выбор случайных значений" | Генерация значений согласно вероятностному распределению. Механизм за всеми генеративными ИИ |
| Равномерное распределение | "Все равновозможны" | Каждое значение в [a, b] имеет равную плотность вероятности 1/(b-a). Начальная точка для всех методов выборки |
| Обратная CDF | "Преобразование вероятностей" | F_inverse(U) преобразует равномерный образец в выборку из любого распределения с известной CDF. Точное и эффективное |
| Сэмплирование отвержением | "Предложение и принятие/отвержение" | Генерация из простого предложения, принятие с вероятностью пропорциональной целевому/предложенному соотношению. Точное, но тратит образцы |
| Важностное сэмплирование | "Пересчет выборок" | Оценка математического ожидания под p(x) используя выборки из q(x), учитывая каждый образец весом p(x)/q(x). Ядро для PPO в обучении с подкреплением |
| Метод Монте-Карло | "Среднее случайных выборок" | Приближение интегралов как средние выборки. Ошибка O(1/sqrt(N)) независимо от размерности |
| MCMC | "Случайная прогулка, сходящаяся" | Построение марковской цепи, стационарное распределение которой — целевое. Метрополиса-Хастингса — фундаментальный алгоритм |
| Метрополиса-Хастингса | "Принятие вверх, иногда вниз" | Предложение перемещений, принятие на основе соотношения плотностей. Детальная балансировка обеспечивает сходимость к целевому распределению |
| Сэмплирование Гиббса | "Одна переменная за раз" | Обновление каждой переменной из её условного распределения, держа остальные фиксированными. 100% ставка принятия |
| Температура | "Кнопка уверенности" | Делит логиты на T перед softmax. T<1 острит (более уверенно), T>1 размывает (более разнообразно) |
| Сэмплирование top-k | "Оставьте k лучших" | Обнуление всех, кроме k токенов с наибольшими вероятностями, нормализация и выборка. Фиксированный размер набора кандидатов |
| Ядерное сэмплирование (top-p) | "Оставьте те, которые вероятны" | Оставьте наименьший набор токенов, чья накопленная вероятность превышает p. Адаптивный размер набора кандидатов |
| Трюк переформулировки | "Перемещение случайности вне" | Запишите z = mu + sigma * epsilon где epsilon ~ N(0,1). Сделайте выборку дифференцируемой. Эссенциален для обучения VAE |
| Гумбель-Софтмакс | "Мягкое категориальное сэмплирование" | Дифференцируемое приближение к категорическому сэмплированию, используя шум Гумбеля + softmax с температурой |
| Стратифицированное сэмплирование | "Принудительное покрытие" | Разделите пространство выборок на слои и выберите из каждого. Всегда имеет дисперсию меньше, чем непосредственный Монте-Карло |
| Период разогрева | "Период разогрева" | Начальные MCMC выборки отбрасываются до тех пор, пока цепь не достигнет своего стационарного распределения |
| Детальная балансировка | "Условие обратимости" | p(x) * T(x->y) = p(y) * T(y->x). Условие достаточное для того чтобы p было стационарным распределением марковской цепи |
| Сэмплирование диффузии | "Итеративное удаление шума" | Генерация данных, начиная с шума и применяя обученные шаги удаления шума. Каждый шаг — это условный оператор выборки |
