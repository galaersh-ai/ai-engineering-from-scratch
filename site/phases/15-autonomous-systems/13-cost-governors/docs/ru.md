# Бюджеты действий, лимиты итераций и регуляторы стоимости

> У агента (agent) среднего интернет-магазина ежемесячная стоимость LLM выросла с $1 200 до $4 800 после того, как команда включила навык «отслеживание заказов». Это не ошибка в тарифах. Это агент, обнаруживший новый цикл (loop) и продолжающий тратить средства внутри него. Инструментарий управления агентами от Microsoft (Agent Governance Toolkit, 2 апреля 2026 г.) кодифицирует защиту от этого класса угроз: `max_tokens` на запрос, бюджеты токенов (tokens) и долларов на задачу, дневные/месячные лимиты, лимиты итераций (iteration), каскадная маршрутизация моделей, кэширование промптов, окно контекста, контрольные точки человека в цикле (HITL) для дорогостоящих действий и аварийный выключатель (kill switch) при превышении бюджета. Agent SDK от Anthropic (Claude Code) предоставляет те же примитивы под другими именами. Лимиты финансовой скорости (financial velocity) — например, отключение доступа при расходе более $50 за 10 минут — обнаруживают циклы быстрее, чем месячные лимиты.

**Тип:** Учебный материал
**Языки:** Python (стандартная библиотека, многоуровневый симулятор регулятора стоимости)
**Предварительные знания:** Фаза 15 · 10 (Режимы доступа), Фаза 15 · 12 (Длительное выполнение)
**Время:** ~60 минут

## Проблема

Автономные (autonomous) агенты тратят реальные деньги на каждом шаге. Плохой ответ чат-бота — это просто неудачный ответ; плохой цикл агента — это счёт. Отраслевой термин для этого типа сбоя — «отказ от кошелька» (Denial of Wallet): агент продолжает рассуждать, вызывать инструменты и начислять стоимость, и ничто не останавливает его, потому что ничто для этого не было предусмотрено.

Решение — не одно число. Это набор ограничений (stack) на разных временных масштабах и уровнях детализации: на запрос, на задачу, в час, в день, в месяц. Грамотно спроектированный набор ловит ушедший в бесконечный цикл за минуты, медленную утечку — за часы, а проблемный релиз — за день. Тот же набор позволяет удержать бюджет, когда агент является долгосрочным и автономным.

Это инженерный урок: математика тривиальна, а дисциплина — то, где команды проигрывают. Перечисленные ниже ограничения названы либо в документации Microsoft Agent Governance Toolkit, либо в документации Anthropic Claude Code Agent SDK.

## Концепция

### Набор регуляторов стоимости (cost-governor stack)

1. **`max_tokens` на запрос.** Простое ограничение. Не даёт одному вызову сгенерировать неограниченный ответ.
2. **Бюджет токенов на задачу.** За весь цикл выполнения не превышайте N токенов. Жёсткая остановка при достижении лимита.
3. **Бюджет в долларах на задачу.** То же, что и бюджет токенов, но в валюте. `max_budget_usd` в Claude Code.
4. **Лимит на вызов инструмента.** Не более N вызовов `WebFetch`, N вызовов `shell_exec` и т. д.
5. **Лимит итераций (iteration cap, `max_turns`).** Общее число итераций цикла агента; предотвращает бесконечные циклы рассуждений.
6. **Лимит на минуту / час / день / месяц.** Скользящие окна. Ловит утечки на разных временных масштабах.
7. **Лимит финансовой скорости (financial velocity limit).** Например: «если расход превысил $50 за 10 минут — отключить доступ». Обнаруживает циклический расход до срабатывания месячного лимита.
8. **Каскадная маршрутизация моделей (tiered model routing).** По умолчанию используетсяPLACEHOLDER_MODEL; к более крупной переключаются только тогда, когда классификатор определит, что задача этого требует.
9. **Кэширование промптов (prompt caching).** Системный промпт и стабильный контекст хранятся в кэше провайдера; стоимость повторной отправки токенов стремится к нулю.
10. **Окно контекста (context windowing).** Сжатие / резюмирование для поддержания активного контекста ниже порога; прямое снижение стоимости токенов.
11. **Контрольные точки человека в цикле (HITL checkpoints) для дорогостоящих действий.** Перед действием, которое известно как дорогостоящее (длительный вызов инструмента, большой скачок, переход на более дорогую модель), требуется подтверждение человека.
12. **Аварийный выключатель (kill switch) при превышении бюджета.** Сессия прерывается при срабатывании любого лимита. Лимит фиксируется; для повторного включения требуется отдельный процесс.

### Почему набор, а не один лимит

Один месячный лимит поймает ушедшего в гонку агента только после того, как кошелёк опустеет. Один лимит на запрос не поймает ничего на уровне сессии. Разные типы сбоев требуют разных временных масштабов:

- **Ушедший в бесконечный цикл (runaway loop)** (агент застрял в 5-секундном повторе): ловится лимитом скорости.
- **Медленная утечка (slow leak)** (агент выполняет ~2x ожидаемой работы на задачу): ловится дневным лимитом.
- **Проблемный релиз (bad release)** (новая версия использует 5x токенов): ловится недельным / месячным лимитом.
- **Легитимный всплеск (legitimate surge)** (реальный спрос, а не баг): ловится часовым / дневным лимитом с прозрачной журнализацией.

### Поверхность бюджета Claude Code

Agent SDK от Anthropic (Claude Code) предоставляет (публичная документация):

- `max_turns` — лимит итераций.
- `max_budget_usd` — лимит в долларах; сессия прерывается при превышении.
- `allowed_tools` / `disallowed_tools` — белый и чёрный списки инструментов.
- Точки перехвата (hook points) перед вызовом инструмента для собственной учётной логики стоимости.

Комбинируйте с лестницей режимов доступа (Урок 10). Сессия `autoMode` без `max_budget_usd` — это неуправляемая автономия. Anthropic прямо указывает, что автоматический режим требует контроля бюджета; классификатор ортогонален по отношению к стоимости.

### EU AI Act, OWASP Agentic Top 10

Инструментарий управления агентами от Microsoft охватывает OWASP Agentic Top 10 и требования EU AI Act (статья 14 — контроль со стороны человека). Для развёртывания (deployment) в ЕС журнализация и контроль лимитов не являются опциональными.

### Наблюдаемый случай $1 200 → $4 800

Реальный случай из документации Microsoft: агент интернет-магазина, чья месячная стоимость утроилась после добавления нового инструмента. Инструмент позволял агенту опрашивать статус заказа на каждом шаге. Не было обнаружения циклов. Не было лимита на вызов инструмента. Не было оповещения о недельном росте. Решением стал лимит на вызов инструмента плюс оповещение о дневном росте. Это шаблон: каждый новый инструмент — это новый потенциальный цикл; у каждого нового инструмента должен быть свой лимит и своё оповещение.

## Применение

`code/main.py` имитирует запуск агента с многоуровневым набором регуляторов стоимости и без него. Моделируемый агент через несколько шагов уходит в цикл опроса (polling loop); многоуровневый набор ловит его в рамках окна скорости, тогда как один месячный лимит сработал бы только через дни.

## Корректирующий аудит

`outputs/skill-agent-budget-audit.md` проверяет набор регуляторов стоимости предполагаемого развёртывания агента и отмечает отсутствующие уровни.

## Упражнения

1. Запустите `code/main.py`. Убедитесь, что лимит скорости срабатывает раньше лимита итераций при траектории (trajectory) цикла опроса. Затем отключите лимит скорости и измерьте, сколько агент «тратит», прежде чем лимит итераций его остановит.

2. Спроектируйте набор лимитов на инструменты для браузерного агента (Урок 11). Какой инструмент нуждается в самом жёстком лимите? Какой инструмент можно запускать без ограничений без риска?

3. Прочитайте документацию Microsoft Agent Governance Toolkit. Перечислите каждый тип лимита, упомянутый в инструментарии. Сопоставьте каждый с одним из типов сбоев (уход в бесконечный цикл, медленная утечка, проблемный релиз, всплеск).

4. Рассчитайте стоимость ночного автономного запуска для реалистичной задачи (например, «разбор 50 задач в репозитории»). Установите `max_budget_usd` на 2x от вашей точечной оценки. Обоснуйте множитель 2x.

5. `max_budget_usd` в Claude Code срабатывает по совокупной стоимости сессии. Спроектируйте комплементарный лимит скорости, который вы будете применять внешние. Что запускает отключение, и как выглядит повторное включение?

## Ключевые термины

| Термин | Как говорят | Что это на самом деле |
|---|---|---|
| Отказ от кошелька (Denial of Wallet) | «Ползущий счёт» | Цикл агента, генерирующий расходы без лимита для остановки |
| max_tokens | «Лимит на запрос» | Потолок размера одного ответа |
| max_turns | «Лимит итераций» | Потолок числа итераций цикла агента в сессии |
| max_budget_usd | «Долларовый аварийный выключатель» | Лимит стоимости сессии; прерывается при превышении |
| Лимит скорости (Velocity limit) | «Лимит скорости» | Ограничение расхода на коротком окне (например, $50 / 10 мин) |
| Каскадная маршрутизация (Tiered routing) | «Маленькая модель сначала» | Дешёвая модель по умолчанию; переключение только по решению классификатора |
| Кэширование промптов (Prompt caching) | «Кэшированный системный промпт» | Кэш на стороне провайдера снижает стоимость повторной отправки токенов до минимума |
| Контрольная точка HITL (HITL checkpoint) | «Шлюз одобрения человека» | Требуется подтверждение человека перед дорогостоящим действием |
