     1|# Git и совместная работа
     2|
     3|> Контроль версий обязателен. Каждый эксперимент, каждая модель, каждый урок, который Вы здесь создадите, отслеживается.
     4|
     5|**Тип:** Изучение
     6|**Языки:** --
     7|**Требования:** Фаза 0, Урок 01
     8|**Время:** ~30 минут
     9|
    10|## Цели обучения
    11|
    12|- Настроить учётные данные git и освоить ежедневный рабочий процесс: add, commit, push
    13|- Создавать и сливать ветки для изолированных экспериментов, не ломая main
    14|- Написать `.gitignore`, исключающий чекпоинты моделей и большие бинарные файлы
    15|- Просматривать историю коммитов с `git log`, чтобы понимать эволюцию проекта
    16|
    17|## Проблема
    18|
    19|Вам предстоит написать сотни файлов с кодом в 20 фазах. Без контроля версий Вы будете терять работу, ломать то, что нельзя откатить, и не сможете сотрудничать с другими.
    20|
    21|Git — это инструмент. GitHub — место, где живёт код. Этот урок охватывает только то, что нужно для курса, и ничего лишнего.
    22|
    23|## Концепция
    24|
    25|```mermaid
    26|sequenceDiagram
    27|    participant WD as Working Directory
    28|    participant SA as Staging Area
    29|    participant LR as Local Repo
    30|    participant R as Remote (GitHub)
    31|    WD->>SA: git add
    32|    SA->>LR: git commit
    33|    LR->>R: git push
    34|    R->>LR: git fetch
    35|    LR->>WD: git pull
    36|```
    37|
    38|Три вещи, которые нужно запомнить:
    39|1. Делай коммиты часто (`git commit`)
    40|2. Отправляй изменения на сервер (`git push`)
    41|3. Ответвляйся для экспериментов (`git checkout -b experiment`)
    42|
    43|## Собираем
    44|
    45|### Шаг 1: Настройка git
    46|
    47|```bash
    48|git config --global user.name "Your Name"
    49|git config --global user.email "you@example.com"
    50|```
    51|
    52|### Шаг 2: Ежедневный рабочий процесс
    53|
    54|```bash
    55|git status
    56|git add file.py
    57|git commit -m "Add perceptron implementation"
    58|git push origin main
    59|```
    60|
    61|### Шаг 3: Ветвление для экспериментов
    62|
    63|```bash
    64|git checkout -b experiment/new-optimizer
    65|
    66|# ... make changes, commit ...
    67|
    68|git checkout main
    69|git merge experiment/new-optimizer
    70|```
    71|
    72|### Шаг 4: Работа с репозиторием курса
    73|
    74|```bash
    75|git clone https://github.com/rohitg00/ai-engineering-from-scratch.git
    76|cd ai-engineering-from-scratch
    77|
    78|git checkout -b my-progress
    79|# work through lessons, commit your code
    80|git push origin my-progress
    81|```
    82|
    83|## Используем
    84|
    85|Для этого курса Вам нужны ровно эти команды:
    86|
    87|| Команда | Когда |
    88||---------|-------|
    89|| `git clone` | Получить репозиторий курса |
    90|| `git add` + `git commit` | Сохранить свою работу |
    91|| `git push` | Отправить на GitHub |
    92|| `git checkout -b` | Попробовать что-то, не ломая main |
    93|| `git log --oneline` | Посмотреть, что сделано |
    94|
    95|Всё. Rebase, cherry-pick и submodules для этого курса не нужны.
    96|
    97|## Упражнения
    98|
    99|1. Склонируйте этот репозиторий, создайте ветку `my-progress`, создайте файл, сделайте коммит, сделайте пуш
   100|2. Создай `.gitignore`, исключающий файлы чекпоинтов моделей (`.pt`, `.pth`, `.safetensors`)
   101|3. Посмотрите историю коммитов этого репозитория через `git log --oneline` и прочитайте, как добавлялись уроки
   102|
   103|## Ключевые термины
   104|
   105|| Термин | Что говорят | Что на самом деле |
   106||--------|------------|-------------------|
   107|| Commit | «Сохранение» | Снимок всего проекта на определённый момент времени |
   108|| Branch | «Копия» | Указатель на коммит, который движется вперёд по мере работы |
   109|| Merge | «Слияние кода» | Взятие изменений из одной ветки и применение их к другой |
   110|| Remote | «Облако» | Копия репозитория, размещённая где-то ещё (GitHub, GitLab) |
