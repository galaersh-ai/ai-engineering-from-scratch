# Agno и Mastra: продакшн-рантаймы

> Agno (Python) и Mastra (TypeScript) — рекомендуемая пара продакшн-рантаймов на 2026 год. Agno рассчитан на микросекундную инстанциацию агентов и без состояния FastAPI-бэкенды. Mastra предоставляет агентов, инструменты (tools), воркфлоу (workflows), единую маршрутизацию моделей и составное хранилище на базе Vercel AI SDK.

**Тип:** изучение  
**Языки:** Python, TypeScript  
**Предварительные знания:** Фаза 14 · 01 (агентный цикл), Фаза 14 · 13 (LangGraph)  
**Время:** ~45 минут

## Цели обучения

- Определить целевые показатели производительности Agno и понять, когда они имеют значение.
- Назвать три примитива Mastra — агенты (Agents), инструменты (Tools), воркфлоу (Workflows) — и поддерживаемые серверные адаптеры.
- Объяснить, почему FastAPI-бэкенды на уровне сессии, не сохраняющие состояние, рекомендуется как основной продакшн-путь для Agno.
- Выбрать Agno или Mastra для конкретного технологического стека (Python-first или TypeScript-first).

## Проблема

LangGraph, AutoGen, CrewAI — это фреймворкотяжёлые решения. Команды, которым нужен «просто агентный цикл, быстро, в моём рантайме», обращаются к Agno (Python) или Mastra (TypeScript). Оба варианта отказываются от части примитивов, принадлежащих фреймворку, ради высокой скорости и более плотной интеграции с окружающим стеком.

## Концепция

### Agno

- Python-рантайм, ранее известный как Phi-data.
- «Никаких графов, цепочек и запутанных паттернов — только чистый Python.»
- Целевые показатели производительности из документации: ~2 мкс на инстанциацию агента (agent instantiation), ~3,75 KiB памяти на агента, ~23 поставщика моделей.
- Продакшн-путь: FastAPI-бэкенды на уровне сессии, не сохраняющие состояние. Каждый запрос запускает новый экземпляр агента; состояние сессии хранится в БД.
- Нативная мультимодальность (текст, изображения, аудио, видео, файлы) и агентный RAG.

Целевые показатели скорости имеют значение, когда у вас тысячи краткоживущих агентов в секунду (конвергенция чат-запросов, пайплайны оценки). Они менее значимы, когда один агент работает в течение 10 минут.

### Mastra

- TypeScript, построен на Vercel AI SDK.
- Три примитива: **агенты (Agents)**, **инструменты (Tools)** (с типизацией через Zod), **воркфлоу (Workflows)**.
- Единый маршрутизатор моделей (Unified Model Router) — 3 300+ моделей у 94 поставщиков (март 2026).
- Составное хранилище: память, воркфлоу, наблюдаемость (observability) для различных бэкендов; ClickHouse рекомендуется для наблюдаемости в масштабе.
- Лицензия Apache 2.0, каталоги `ee/` доступны по корпоративной лицензии с открытым исходным кодом.
- Серверные адаптеры для Express, Hono, Fastify, Koa; первоклассная интеграция с Next.js и Astro.
- Включает Mastra Studio (localhost:4111) для отладки.
- 22k+ звёзд на GitHub, 300k+ еженедельных npm-загрузок при версии 1.0 (январь 2026).

### Позиционирование

Ни один из инструментов не пытается быть LangGraph. Они конкурируют по:

- **Соответствию языку.** Agno — для команд, работающих в первую очередь на Python; Mastra — для TypeScript-first.
- **Эргономике рантайма.** Agno = минимальные накладные расходы; Mastra = интеграция с экосистемой Vercel.
- **Наблюдаемости.** Оба интегрируются с Langfuse/Phoenix/Opik (Урок 24), но Mastra Studio — собственный инструмент первой линии.

### Когда выбирать каждый

- **Agno** — Python-бэкенды, множество краткоживущих агентов, высокие требования к производительности, FastAPI-стек.
- **Mastra** — TypeScript-бэкенды, развёртывание на Next.js / Vercel, единая маршрутизация моделей у нескольких поставщиков, инструменты с типизацией через Zod.
- **LangGraph** (Урок 13) — когда важнее сохраняющее состояние (stateful) и явное графовое рассуждение, чем чистая скорость.
- **SDK OpenAI / Claude Agent** — когда нужна продуктовая форма от поставщика (Уроки 16–17).

### Где эта модель даёт сбой

- **Производительность ради производительности.** Выбор Agno, потому что «2 мкс» звучит хорошо, когда рабочая нагрузка — один медленный вызов агента на запрос. Накладные расходы не являются узким местом.
- **Запирание в экосистеме.** Интеграция Mastra с экосистемой Vercel — плюс на Vercel, минус в остальных случаях.
- **Неоднозначность корпоративной лицензии.** Каталоги `ee/` в Mastra доступны с открытым исходным кодом, но не под лицензией Apache 2.0. Ознакомьтесь с условиями лицензий, если планируете форк.

## Реализация

Этот урок носит преимущественно сравнительный характер — ни один кодовый артефакт не способен в равной мере представить оба фреймворка. См. `code/main.py` для примера сравнения: минимальный сценарий «запустить агента, транслировать вывод, сохранить сессию», реализованный дважды (в стиле Agno и в стиле Mastra).

Запуск:

```
python3 code/main.py
```

Два структурно различных, но функционально эквивалентных трейса.

## Применение

- **Agno** — Python-бэкенды, которым нужна скорость и совместимость с FastAPI.
- **Mastra** — TypeScript-бэкенды с множеством поставщиков и примитивами воркфлоу.
- Оба предоставляют собственные средства наблюдаемости (observability). Оба интегрируются с Langfuse.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|--------|------------|----------------------|
| Agno | «Быстрые Python-агенты» | Агентный рантайм без состояния на уровне сессии |
| Mastra | «TypeScript-агенты на Vercel AI SDK» | Агенты + инструменты + воркфлоу + маршрутизатор моделей |
| Единый маршрутизатор моделей (Unified Model Router) | «Доступ к нескольким поставщикам» | Единый клиент для 3 300+ моделей у 94 поставщиков |
| Составное хранилище (Composite storage) | «Несколько бэкендов» | Память/воркфлоу/наблюдаемость — каждый в отдельном хранилище |
| Mastra Studio | «Локальный отладчик» | Интерфейс на localhost:4111 для интроспекции агентов |
| Доступ к исходному коду (Source-available) | «Не OSS» | Лицензия разрешает чтение исходного кода, но ограничивает коммерческое использование |
