# Claude Agent SDK: подагенты и хранилище сессий

> Claude Agent SDK — это библиотечная форма каркаса Claude Code (harness). Встроенные инструменты, подагенты для изоляции контекста, хуки (hooks), распространение трассировки W3C, совместимость хранилища сессий. Claude Managed Agents — это хостируемая альтернатива для длительной асинхронной работы.

**Тип:** Изучение + практическая работа
**Языки:** Python (стандартная библиотека)
**Предварительные требования:** Фаза 14 · 01 (цикл агента), Фаза 14 · 10 (библиотеки навыков)
**Время:** ~75 минут

## Цели обучения

- Объяснить разницу между Anthropic Client SDK (сырой API) и Claude Agent SDK (каркасная форма).
- Описать подагенты — параллелизм и изоляция контекста — и случаи их применения.
- Перечислить интерфейс хранилища сессий Python SDK (`append`, `load`, `list_sessions`, `delete`, `list_subkeys`) и роль `--session-mirror`.
- Реализовать каркас на стандартной библиотеке с встроенными инструментами, созданием подагентов с изолированным контекстом, хуками жизненного цикла и хранилищем сессий.

## Проблема

Сырой LLM API обеспечивает один обмен данными. Продукционный агент (agent) требует выполнения инструментов, серверов MCP, хуков жизненного цикла, создания подагентов, хранения сессий, распространения трассировки. Claude Agent SDK поставляет эту форму в виде библиотеки — тот же каркас, который использует Claude Code, открытый для кастомных агентов.

## Концепция

### Client SDK vs Agent SDK

- **Client SDK (`anthropic`).** Сырой Messages API. Вы управляете циклом (loop), инструментами и состоянием.
- **Agent SDK (`claude-agent-sdk`).** Встроенное выполнение инструментов, подключения MCP, хуки, создание подагентов, хранилище сессий. Цикл Claude Code в виде библиотеки.

### Встроенные инструменты

SDK поставляет более 10 инструментов (tools) из коробки: чтение/запись файлов, shell, grep, glob, веб-запросы и другие. Пользовательские инструменты регистрируются через стандартный интерфейс описания инструментов (tool-schema).

### Подагенты

Две задачи, документированные Anthropic:

1. **Параллелизм.** Параллельное выполнение независимых задач. «Найти тестовый файл для каждого из этих 20 модулей» — это 20 параллельных задач подагентов.
2. **Изоляция контекста.** Подагенты используют собственное окно контекста; только результаты возвращаются оркестратору. Бюджет контекста оркестратора сохраняется.

Недавние дополнения Python SDK: `list_subagents()`, `get_subagent_messages()` для чтения протоколов подагентов.

### Хранилище сессий

Совместимость протокола с TypeScript:

- `append(session_id, message)` — добавить ход.
- `load(session_id)` — восстановить беседу.
- `list_sessions()` — перечислить сессии.
- `delete(session_id)` — с каскадным удалением подсессий подагентов.
- `list_subkeys(session_id)` — список ключей подагентов.

`--session-mirror` (флаг CLI) выводит транскрипт во внешний файл по мере потоковой передачи — для отладки.

### Хуки

Хуки жизненного цикла (lifecycle hooks), которые можно зарегистрировать:

- `PreToolUse`, `PostToolUse` — фильтрация или аудит вызовов инструментов.
- `SessionStart`, `SessionEnd` — настройка и завершение.
- `UserPromptSubmit` — обработка пользовательского ввода до того, как его увидит модель.
- `PreCompact` — выполнение перед компактификацией контекста.
- `Stop` — очистка при выходе агента.
- `Notification` — оповещения по побочному каналу.

Хуки — это способ добавить сквозную логику в рабочий процесс (workflow) и аналогичные системы.

### Трассировка W3C

Активные спаны OTel (OpenTelemetry) вызывающего кода распространяются в дочерний процесс CLI через заголовки трассировки W3C. Вся многопроцессная трассировка отображается как одна трассировка в вашем бэкенде.

### Claude Managed Agents

Хостируемая альтернатива (бета-заголовок `managed-agents-2026-04-01`). Длительная асинхронная работа, встроенное кеширование промптов, встроенная компактификация. Выбор управляемой инфраструктуры вместо самостоятельного контроля.

### Типичные ошибки

- **Чрезмерное создание подагентов.** Создание 100 подагентов для 100 мелких задач. Накладные расходы превышают выгоду. Пакетная обработка предпочтительнее.
- **Разрастание хуков.** Каждая команда добавляет хуки; время запуска растёт. Пересматривайте хуки ежеквартально.
- **Нарастание сессий.** Сессии накапливаются, размер растёт. Используйте `list_sessions` и политику истечения срока.

## Реализация

`code/main.py` реализует форму SDK на стандартной библиотеке:

- `Tool`, `ToolRegistry` со встроенными `read_file`, `write_file`, `list_dir`.
- `Subagent` — приватный контекст, изолированное выполнение, возврат результатов.
- `SessionStore` — append, load, list, delete, list_subkeys.
- `Hooks` — `pre_tool_use`, `post_tool_use`, `session_start`, `session_end`.
- Демо: основной агент параллельно создаёт 3 подагента (каждый изолирован), агрегирует результаты, сохраняет сессию.

Запуск:

```
python3 code/main.py
```

Трассировка показывает изоляцию контекста подагентов (размер контекста оркестратора остаётся ограниченным), выполнение хуков и сохранение сессий.

## Применение

- **Claude Agent SDK** — для продуктов, построенных на базе Claude, которым нужна каркасная форма Claude Code.
- **Claude Managed Agents** — для хостируемой длительной асинхронной работы.
- **OpenAI Agents SDK** (Урок 16) — аналогичное решение для OpenAI.
- **LangGraph + пользовательские инструменты** — если предпочтительна графовая конечный автомат (state machine).

## Упражнения

1. Добавить создатель подагентов, который разбивает 20 задач на группы по 5 параллельных подагентов. Сравнить размер контекста оркестратора с вариантом «одна задача — один подагент».
2. Реализовать хук `PreToolUse`, ограничивающий частоту вызовов `write_file` (5 вызовов в минуту на сессию). Отследить поведение.
3. Подключить `list_subkeys` для визуализации дерева подагентов. Как выглядит глубокая вложенность?
4. Перенести демо на реальный пакет `claude-agent-sdk` Python. Что изменится в регистрации инструментов?
5. Прочитать документацию Claude Managed Agents. В каких случаях стоит перейти с самостоятельного развертывания на управляемый сервис?

## Ключевые термины

| Термин | Как говорят | Что это на самом деле |
|--------|-------------|----------------------|
| Agent SDK | «Claude Code как библиотека» | Каркасная форма: инструменты, MCP, хуки, подагенты, хранилище сессий |
| Подагент (Subagent) | «Дочерний агент» | Отдельный контекст, собственный бюджет; результаты передаются наверх |
| Хранилище сессий (Session store) | «БД бесед» | Сохранение, загрузка, перечисление, удаление ходов с каскадом подагентов |
| Хук (Hook) | «Колбэк жизненного цикла» | До/после инструмента, сессия, отправка промпта, компактификация, остановка |
| Трассировка W3C (W3C trace context) | «Межпроцессная трассировка» | Родительский спан распространяется в дочерний процесс CLI |
| Managed Agents | «Хостируемый каркас» | Длительная асинхронная работа на инфраструктуре Anthropic |
| `--session-mirror` | «Зеркало транскрипта» | Записывает ходы сессии во внешний файл по мере потоковой передачи |
| MCP-сервер (MCP server) | «Интерфейс инструментов» | Внешний источник инструментов/ресурсов, подключённый к агенту |
