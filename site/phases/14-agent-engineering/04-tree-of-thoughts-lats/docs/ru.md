# Дерево мыслей (Tree of Thoughts) и LATS: обдуманный поиск

> Одна траектория (trajectory) цепочки рассуждений (chain-of-thought) не допускает возврата назад. ToT (Yao et al., 2023) превращает рассуждение в дерево с самооценкой (self-evaluation) на каждом узле. LATS (Zhou et al., 2024) объединяет ToT с ReAct и Reflexion на основе поиска методом Монте-Карло (Monte Carlo Tree Search). Задача «Игра в 24» (Game of 24) переходит от 4% (CoT) до 74% (ToT); LATS достигает 92,7% pass@1 на HumanEval.

**Тип:** сборка (Build)
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 14 · 01 (цикл агента), Фаза 14 · 03 (Reflexion)
**Время:** ~75 минут

## Цели обучения

- Формулировать рассуждение как поиск (search): узлы — «мысли» (thoughts), рёбра — «расширения» (expansions), ценность — «насколько перспективно».
- Реализовать поиск BFS по дереву в стиле ToT с помощью stdlib и оценку каждого узла (self-evaluation scoring).
- Расширить до учебного цикла LATS MCTS с этапами: выбор (select) / расширение (expand) / симуляция (simulate) / обратное распространение (backpropagate).
- Определять, когда поиск оправдан повышенным расходом токенов (token) (игра «24», генерация кода), а когда одной траектории достаточно (простой вопрос-ответ).

## Проблема

Цепочка рассуждений (chain-of-thought) — это линейное прохождение. Если первый шаг неверен, каждый последующий шаг опирается на ошибочное допущение. В задаче «Игра в 24» (Game of 24) (использование четырёх цифр с помощью операций +, −, ×, ÷ для получения 24) точность GPT-4 с CoT составляет всего 4%. Модель рано выбирает неправильное подвыражение и не может исправить ошибку.

Для рассуждений необходима способность предлагать несколько вариантов, оценивать их, выбирать перспективные и возвращаться назад при обнаружении тупиков. Это и есть поиск. Дерево мыслей (Tree of Thoughts) и LATS — два канонических подхода.

## Концепция

### Дерево мыслей (Tree of Thoughts) (Yao et al., NeurIPS 2023)

Каждый узел — это связный промежуточный шаг («мысль»). Каждый узел может расширяться до K дочерних мыслей. LLM выполняет самооценку каждого узла с помощью промпта для оценки (scoring prompt). Поиск исследует дерево — BFS, DFS или лучевой поиск (beam search).

```
                     (root: "find 24 from 4 6 4 1")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- Score: HIGH
              /   \              |                  |
          ...    ...          ...                finish
```

Самооценка — ключевой элемент. В статье рассматриваются три варианта: классификация `sure / likely / impossible`, числовой балл `1..10` и голосование среди кандидатов. Все три варианта значительно превосходят CoT в задаче «Игра в 24» (4% → 74% с GPT-4).

### LATS (Zhou et al., ICML 2024)

LATS объединяет ToT, ReAct и Reflexion на основе MCTS. LLM выполняет три роли:

- **Политика (policy)**: предлагает кандидатные следующие действия в стиле ReAct.
- **Функция ценности (value function)**: оценивает частичную траекторию (trajectory) в стиле самооценки ToT.
- **Саморефлексия (self-reflector)**: при неудаче записывает рефлексию (reflection) на естественном языке в стиле Reflexion и использует её для повторной инициализации будущих прокаток (rollouts).

Обратная связь от среды (наблюдения, observations) встраивается в функцию ценности, так что поиск опирается на реальные результаты инструментов (tool output), а не только на «мнение» модели. Результаты на момент публикации: HumanEval pass@1 92,7% с GPT-4 (state of the art), WebShop среднее значение 75,9 с GPT-3.5 (приближение к дообучению с градиентом).

### MCTS в кратком изложении

Четыре этапа на каждой итерации (iteration):

1. **Выбор (select)** — движение от корня к листу с использованием UCT (верхняя граница уверенности для деревьев, upper confidence bound for trees).
2. **Расширение (expand)** — генерация K дочерних узлов с помощью политики.
3. **Симуляция (simulate)** — прокатка (rollout) от дочернего узла с помощью политики; оценка листа функцией ценности (или наградой среды).
4. **Обратное распространение (backpropagate)** — обновление счётчиков посещений и оценок ценности вдоль пути.

Формула UCT: `Q(s, a) + c * sqrt(ln N(s) / N(s, a))`. Первый член — использование (exploitation); второй — исследование (exploration). Параметр `c` настраивается для каждой задачи.

### Реальность затрат

Поиск многократно увеличивает расход токенов. ToT в задаче «Игра в 24» использует 100–1000 раз больше токенов, чем CoT. LATS — аналогично. Поиск не бесплатен; его стоит применять:

- Задачи, где одна траектория явно недостаточна (игра «24», сложный код).
- Задачи, где важнее корректность, чем время выполнения.
- Задачи с недорогой и надёжной функцией ценности (unit-тесты для кода, явная цель для математики).

Если задача имеет единственный правильный ответ и оценщик зашумлён, поиск часто ухудшает результат — он находит неверный ответ с «высоким баллом».

### Позиционирование в 2026 году

Большинство продакшн-агентов (agent) не используют LATS. Они работают на ReAct с верификацией на основе инструментов (CRITIC, Урок 05). Поиск применяется в узкоспециализированных нишах:

- Кодогенерирующие агенты, использующие тесты как функцию ценности (в стиле HumanEval).
- Агенты глубокого исследования, исследующие несколько путей запросов.
- Рабочие процессы (workflow) с интенсивным планированием внутри подграфов LangGraph.

AlphaEvolve (Урок 11) — экстремальный пример 2025 года: эволюционный поиск по коду, проверяемая машинами пригодность, рекордные достижения (первое улучшение матричного умножения 4×4 за 56 лет).

## Реализация

`code/main.py` реализует:

- Миниатюрный поиск BFS в стиле ToT для стилизованной задачи «выбор арифметических операций».

```
python3 code/main.py
```

## Ключевые термины

| Термин | Как говорят | Что это на самом деле |
|--------|-------------|----------------------|
| Tree of Thoughts (дерево мыслей) | «Ветвящийся CoT» | Yao et al. — дерево узлов-мыслей с самооценкой |
| LATS | «MCTS для LLM» | Zhou et al. — объединяет ToT + ReAct + Reflexion на основе MCTS |
| UCT | «Верхняя граница уверенности» | Формула выбора, балансирующая использование (Q) и исследование (ln N / n) |
| Value function (функция ценности) | «Насколько хороша эта ситуация» | Оценка LLM через промпт или награда среды; питает обратное распространение |
| Policy (политика) | «Предлагатель действий» | Генератор в стиле ReAct; порождает кандидатные мысли/действия |
| Rollout (прокатка) | «Моделированная траектория» | Проход от узла до листа с помощью политики; оценка ценностью |
| Backpropagate (обратное распространение) | «Обновление предков» | Передача награды листа вверх по пути, обновление счётчиков посещений и Q |
| Search cost (стоимость поиска) | «Взрыв токенов» | 100–1000× расхода CoT в задаче «Игра в 24»; учитывайте бюджет до внедрения |
