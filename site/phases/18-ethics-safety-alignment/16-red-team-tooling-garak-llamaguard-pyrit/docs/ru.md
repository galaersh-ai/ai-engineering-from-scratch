# Инструменты тестирования на проникновение (red teaming) — Garak, Llama Guard, PyRIT

> Три промышленных инструмента формируют стек тестирования на проникновение (red teaming) 2026 года. Llama Guard (Meta) — классификатор Llama-3.1-8B, дообученный на 14 категориях рисков MLCommons; Llama Guard 4 (2025) — 12B нативно мультимодальный классификатор, полученный прuning из Llama 4 Scout. Garak (NVIDIA) — открытый сканер уязвимостей LLM со статическими, динамическими и адаптивными пробами для галлюцинаций (hallucination), утечек данных (data leakage), инъекций промптов (prompt injection), токсичности и джейлбрейков (jailbreaking). PyRIT (Microsoft) — многотурные кампании тестирования на проникновение с использованием Crescendo, TAP и пользовательских цепочек конвертеров для глубокой эксплуатации. Llama Guard 3 описан в «Llama 3 Herd of Models» (Meta, arXiv:2407.21783); Llama Guard 3-1B-INT4 — в arXiv:2411.17713; архитектура проб Garak — в github.com/NVIDIA/garak. Эти инструменты составляют промышленный интерфейс 2026 года между исследованиями в области тестирования на проникновение (уроки 12–15) и развёртыванием (урок 17 и далее).

**Тип:** Практическая работа
**Языки:** Python (стандартная библиотека, симулятор архитектуры инструментов и имитация классификатора в стиле Llama Guard)
**Предварительные знания:** Фаза 18, уроки 12–15 (джейлбрейки и IPI)
**Время:** ~75 минут

## Цели обучения

- Описать позицию Llama Guard 3/4 в стеке безопасности: классификатор входных данных, классификатор выходных данных или оба.
- Назвать 14 категорий рисков MLCommons и указать одну неочевидную (злоупотребление интерпретатором кода — Code Interpreter Abuse).
- Описать архитектуру проб Garak: пробы (probes), детекторы (detectors), каркасы (harnesses).
- Описать структуру многотурных кампаний PyRIT и их взаимодействие с пробами Garak.

## Проблема

Уроки 12–15 описывают поверхность атак. Промышленное развёртывание требует воспроизводимой и масштабируемой оценки. В 2026 году доминируют три инструмента: Llama Guard (классификатор-защита), Garak (сканер), PyRIT (оркестратор кампаний). Каждый из них ориентирован на другой уровень жизненного цикла тестирования на проникновение.

## Концепция

### Llama Guard (Meta)

Llama Guard 3 — это модель Llama-3.1-8B, дообученная для классификации входных и выходных данных по 14 категориям MLCommons AILuminate:
- Насильственные преступления, ненасильственные преступления, сексуальное содержание, CSAM, клевета
- Специализированные консультации, конфиденциальность, интеллектуальная собственность, массовое оружие, ненависть
- Суицид/самоповреждение, сексуальное содержание, выборы, злоупотребление интерпретатором кода

Поддерживает 8 языков. Применение: размещение перед LLM (модерация входных данных), после LLM (модерация выходных данных) или в обоих местах. Два варианта использования порождают различные распределения обучающих данных — Llama Guard 3 поставляется как единая модель, обрабатывающая оба варианта.

Llama Guard 3-1B-INT4 (arXiv:2411.17713, 440 МБ, ~30 токенов/с на мобильном CPU) — квантизированная мобильная версия.

Llama Guard 4 (апрель 2025) — 12B, нативно мультимодальный, полученный прuning из Llama 4 Scout. Он заменяет как 8B текстовую, так и 11B визуальную предшествующие модели одним классификатором, принимающим текст и изображения.

### Garak (NVIDIA)

Открытый сканер уязвимостей. Архитектура:
- **Пробы (Probes).** Генераторы атак для галлюцинаций, утечек данных, инъекций промптов, токсичности, джейлбрейков. Статические (фиксированные промпты), динамические (генерируемые промпты), адаптивные (реагируют на вывод цели).
- **Детекторы (Detectors).** Оценка выводов по ожидаемым режимам сбоев — токсичный, утечка данных, джейлбрейк.
- **Каркасы (Harnesses).** Управление парами проб-детекторов, запуск кампаний, генерация отчётов.

TrustyAI интегрирует Garak с щитами Llama-Stack (классификатор входных данных Prompt-Guard-86M, классификатор выходных данных Llama-Guard-3-8B) для комплексной оценки щит-цель. Поколенная оценка (TBSA — tier-based scoring) заменяет бинарное прохождение/непрохождение — модель может пройти на уровне серьёзности 3 и не пройти на уровне серьёзности 5 для одной и той же пробы.

### PyRIT (Microsoft)

Python Risk Identity Toolkit (Набор инструментов для идентификации рисков на Python). Многотурные кампании тестирования на проникновение. Основан на:
- **Конвертерах (Converters).** Преобразование исходного промпта — перефразирование, кодирование, перевод, ролевая игра.
- **Оркестраторах (Orchestrators).** Запуск кампании: Crescendo (эскалация), TAP (ветвление), RedTeaming (пользовательский цикл).
- **Оценке (Scoring).** LLM-судья или классификатор-судья.

PyRIT — более тяжёлый инструмент по сравнению с Garak. Garak запускает тысячи однотурных проб; PyRIT проводит глубокие многотурные кампании, предназначенные для вскрытия конкретных режимов сбоев.

### Стек

Разместите Llama Guard с обеих сторон модели. Запускайте Garak ежедневно для регрессионного тестирования. Запускайте PyRIT для кампаний перед релизом. Это конфигурация по умолчанию 2026 года для большинства промышленных развёртываний.

### Ошибки оценки

- **Идентичность судьи.** Все три инструмента могут использовать LLM-судью; калибровка судьи определяет заявленные показатели успеха атаки — ASR (урок 12). Указывайте судью вместе с инструментом.
- **Устаревание проб.** Пробы Garak устаревают по мере исправления моделей. Адаптивные пробы (в форме PAIR) устаревают медленнее, чем статические.
- **Ложноположительные срабатывания Llama Guard на доброкачественном контенте.** Ранние версии Llama Guard чрезмерно флаговали политический и ЛГБТК+ контент; калибровки Llama Guard 3/4 улучшены, но не настраиваются под конкретное развёртывание.

### Место в фазе 18

Уроки 12–15 описывают семейства атак. Урок 16 — промышленные инструменты. Урок 17 (WMDP) — оценка двойственных технологий. Урок 18 — рамки безопасности на переднем рубеже (frontier safety frameworks), объединяющие эти инструменты в политическую структуру.

## Упражнения

1. Запустите `code/main.py`. Сравните частоту обнаружения классификатора в стиле Llama Guard при однотурных и многотурных атаках.

2. Реализуйте новую пробу Garak: запрос, закодированный в base64. Измерьте его обнаружение классификатором в стиле Llama Guard.

3. Дополните цепочку конвертеров в стиле PyRIT конвертером «перевод на французский, затем перефразирование». Повторно измерите успех атаки.

4. Прочитайте список категорий рисков Llama Guard 3. Определите две категории, в которых обучающие данные реалистично приведут к высокой частоте ложноположительных срабатываний на легитимном контенте разработчиков.

5. Сравните принципы проектирования Garak и PyRIT. Обоснуйте развёртывание, в котором каждый из них является оптимальным инструментом.
