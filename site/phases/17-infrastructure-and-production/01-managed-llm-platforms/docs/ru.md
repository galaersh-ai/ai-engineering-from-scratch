# Управляемые платформы LLM — Bedrock, Vertex AI, Azure OpenAI

> Три гиперскейлера, три различных стратегии. AWS Bedrock — это маркетплейс моделей: Claude, Llama, Titan, Stability, Cohere доступны через единый API. Azure OpenAI — эксклюзивное партнёрство с OpenAI и PTU (Provisioned Throughput Units, единицы выделенной пропускной способности) для выделенной ёмкости. Vertex AI ориентирован на Gemini с лучшей поддержкой длинного контекста и мультимодальности. В 2026 году Artificial Analysis измеряет медианную задержку Azure OpenAI на уровне ~50 мс, а Bedrock — ~75 мс на эквивалентах Llama 3.1 405B: PTU объясняют разницу, поскольку выделенная ёмкость превосходит общедоступный shared on-demand. Критерий выбора — не «какая платформа быстрее», а «какой каталог моделей и FinOps-интерфейс соответствуют вашему продукту». Этот урок учит вас выбирать осознанно, с задокументированными компромиссами, а не по интуиции.

**Тип:** Learn
**Языки:** Python (stdlib, упрощённый компаратор стоимости и задержки)
**Предварительные требования:** Фаза 11 (Инженерия LLM), Фаза 13 (Инструменты и протоколы)
**Время:** ~60 минут

## Цели обучения

- Назвать три стратегии платформ (маркетплейс vs эксклюзив vs Gemini-first) и сопоставить каждую с конкретным продуктовым сценарием использования.
- Объяснить, что дают PTU (Provisioned Throughput Units) в Azure OpenAI и почему shared on-demand в Bedrock обычно показывает задержку на ~25 мс выше в сегменте моделей 405B.
- Схематизировать поверхность атрибуции затрат (FinOps) для каждой платформы (Application Inference Profiles в Bedrock vs проект-на-команду в Vertex vs области подписки + резервирование PTU в Azure).
- Сформулировать политику «минимум два провайдера» и объяснить, почему привязка к одному поставщику является дорогостоящей ошибкой в 2026 году.

## Проблема

Вы выбрали Claude 3.7 Sonnet для своего продукта. Теперь вам необходимо его обслуживать (serving). Вы можете вызвать API Anthropic напрямую, вызвать его через AWS Bedrock или использовать шлюз. Прямое API — самый простой вариант; Bedrock добавляет BAAs, VPC-эндпоинты, IAM и атрибуцию через CloudWatch. Шлюз добавляет failover, единое биллинг и ограничения частоты запросов (rate limits) для нескольких провайдеров.

Более глубокий вопрос — каталог моделей. Если вам нужны Claude, Llama и Gemini в одном продукте, вы не можете приобрести их все в одном месте, если только это место не будет одновременно Bedrock, Vertex и Azure OpenAI. Гиперскейлеры не взаимозаменяемы — каждый из них сделал различную ставку на то, кто владеет слоем моделей.

Этот урок описывает три ставки, разрыв в задержке, разрыв в FinOps и риск привязки к платформе.

## Концепция

### Три стратегии

**AWS Bedrock** — маркетплейс. Claude (Anthropic), Llama (Meta), Titan (собственная линейка AWS), Stability (изображения), Cohere (эмбеддинги), Mistral, а также подкаталоги изображений и эмбеддингов. Один API, один интерфейс IAM, один экспорт в CloudWatch. Ставка Bedrock заключается в том, что клиенты ценят вариативность выбора больше, чем единую модель.

**Azure OpenAI** — эксклюзивное партнёрство. GPT-4 / 4o / 5 / o-серия, DALL·E, Whisper и дообучение моделей OpenAI в дата-центрах Azure. Нет моделей, не относящихся к OpenAI, в каталоге «Azure OpenAI Service» — они попадают в Azure AI Foundry (отдельный продукт). Ставка Azure заключается в том, что OpenAI остаётся технологическим лидером, и клиенты хотят управление предприятием именно для этого взаимодействия.

**Vertex AI** — Gemini в первую очередь, всё остальное — во вторую. Gemini 1.5 / 2.0 / 2.5 Flash и Pro, а также Model Garden (сторонние модели). Ставка Vertex — мультимодальность и длинный контекст: контекст Gemini на 1 млн токенов является ключевым конкурентным преимуществом.

### Разрыв в задержке при масштабе

Artificial Analysis проводит непрерывные бенчмарки. На эквивалентных развёртываниях Llama 3.1 405B (общедоступный shared on-demand) медианная задержка первого токена (TTFT) в Azure OpenAI составляет около 50 мс; в Bedrock — около 75 мс. Этот разрыв не является провалом AWS — он обусловлен различиями в модели ёмкости. Azure продаёт PTU (Provisioned Throughput Units), которые резервируют GPU-ёмкость для вашего тенанта. Эквивалент Bedrock (Provisioned Throughput) существует, но начинается от $21/час за единицу, и большинство клиентов остаются на shared on-demand.

Общедоступная ёмкость конкурирует с трафиком всех остальных клиентов. Выделенная — нет. Если ваш SLA по продукту требует TTFT < 100 мс на P99, вы либо приобретаете PTU в Azure, либо выделенную ёмкость Provisioned Throughput в Bedrock, либо принимаете стандартную вариативность.

### Экономика Provisioned Throughput

Azure PTU: зарезервированный блок инференс (inference)-вычислений. Экономия до ~70% по сравнению с on-demand для предсказуемых нагрузок. Фиксированная стоимость в час независимо от трафика — вы платите за резервирование даже в простое. Точка безубыточности обычно достигается при 40–60% устойчивой загрузки.

Bedrock Provisioned Throughput: $21–$50 в час в зависимости от модели и региона. Аналогичный расчёт — точка безубыточности примерно при половине пиковой загрузки. Требуется ежемесячное обязательство.

Ёмкость с выделенным ресурсом в Vertex продаётся по SKU Gemini; стоимость варьируется в зависимости от модели и региона и менее публично рекламируется.

### Поверхность FinOps — настоящий дифференциатор

**Application Inference Profiles в Bedrock** — самая чистая атрибуция в маркетплейсе. Пометьте профиль по полям `team`, `product`, `feature`; направьте через него все вызовы моделей; CloudWatch разбивает стоимость по профилям без постобработки. Введены в 2025 году, по-прежнему самая детализированная нативная реализация среди гиперскейлеров.

**Атрибуция в Vertex** основана на принципе «проект-на-команду» с повсеместным использованием labels. Каждая команда моделируется как отдельный GCP-проект, на все ресурсы накладываются labels, а для сводок используется BigQuery Billing Export + DataStudio. Требуется больше усилий, но BigQuery предоставляет произвольный SQL для анализа данных о затратах.

**Azure** опирается на области подписки/групп ресурсов и теги, причём резервирование PTU является первоклассным объектом стоимости. Теги наследуются от групп ресурсов, а не от запросов, поэтому атрибуция по запросам требует пользовательских метрик Application Insights или шлюза, добавляющего заголовки.

Общая закономерность: Bedrock — самая чистая нативная реализация, Vertex — самая гибкая через BigQuery, Azure — самая непрозрачная без дополнительной инструментации.

### Привязка к платформе — главный риск 2026 года

Привязка к одному гиперскейлеру была допустима, когда одна модель доминировала. В 2026 году технологический рубеж смещается ежеквартально: Claude 3.7 один квартал, Gemini 2.5 — следующий, GPT-5 — третий. Привязка к одной платформе отсекает вас от двух третей передового контента.

Паттерн, который используют успешные команды: минимально два провайдера для любого критически важного LLM-вызова в продукте. Bedrock + Azure OpenAI — самая распространённая пара: Claude от одного, GPT от другого, failover между ними через общий шлюз. Увеличение стоимости минимально, поскольку шлюз оптимально маршрутизирует трафик, а повышение доступности во время сбоев (таких как инцидент Azure OpenAI в январе 2025 года, отключение AWS us-east-1) является решающим фактором.

### Резидентность данных, BAAs и регулируемые отрасли

Bedrock: BAAs в большинстве регионов; VPC-эндпоинты; контрольные механизмы. Типичный выбор для финтех-компаний.

Azure OpenAI: HIPAA, SOC 2, ISO 27001; резидентность данных в ЕС; стандарт для предприятий в регулируемых отраслях.

Vertex: HIPAA, GDPR, резидентность данных в зависимости от региона; стек соответствия Google Cloud.

Все три платформы соответствуют базовым требованиям. Различия касаются политик хранения данных, обработки логов и того, контролирует ли мониторинг злоупотреблений ваш трафик (по умолчанию включено на большинстве платформ; для корпоративных клиентов предусмотрена возможность отключения).

### Числа, которые стоит запомнить

- Медианный TTFT Azure OpenAI на эквивалентах Llama 3.1 405B: ~50 мс (с PTU).
- Медианный TTFT Bedrock на shared on-demand: ~75 мс.
- Bedrock Provisioned Throughput: $21–$50/час за единицу.
- Точка безубыточности PTU в Azure: ~40–60% устойчивой загрузки.
- Экономия PTU по сравнению с on-demand при высокой загрузке: до 70%.

## Ключевые термины

| Термин | Как его называют | Что это на самом деле |
|--------|-----------------|----------------------|
| Bedrock | «LLM-сервис AWS» | Маркетплейс моделей с Claude, Llama, Titan, Mistral, Cohere |
| Azure OpenAI | «ChatGPT от Azure» | Эксклюзивные модели OpenAI в дата-центрах Azure с управлением для предприятий |
| Vertex AI | «LLM от Google» | Платформа с приоритетом Gemini и Model Garden для сторонних моделей |
| PTU | «выделенная ёмкость» | Provisioned Throughput Unit — резервированные GPU для инференса, стоимостью в час |
| Application Inference Profile | «метки Bedrock» | Профиль стоимости/использования по продуктам с тегами, интегрированный с CloudWatch |
| Model Garden | «каталог Vertex» | Раздел сторонних моделей в Vertex AI, отдельный от Gemini |
| Two-provider minimum | «резервирование LLM» | Политика запуска каждого критически важного LLM-пути через ≥2 гиперскейлеров |
| BAA | «бумажки HIPAA» | Соглашение ассоциированного бизнеса (Business Associate Agreement); требуется для PHI; предоставляется всеми тремя провайдерами |
| Мониторинг злоупотреблений | «просмотрщик логов» | Сканирование промптов и выходных данных на стороне провайдера для обеспечения безопасности; отключение доступно для корпоративных клиентов |
