# Генеративные агенты и эмерджентная симуляция

> Park et al. 2023 (UIST '23, arXiv:2304.03442) населил **Smallville**, песочницу из 25 агентов, трёхкомпонентной архитектурой: **поток памяти** (лог на естественном языке), **рефлексия** (синтезы более высокого порядка, которые агент генерирует о собственном потоке) и **план** (поведение на уровне дня, затем подпланы). Веховый результат — эмерджент вечеринки в День святого Валентина: один агент с установкой «хочет устроить вечеринку в День святого Валентина», без дополнительного сценария, породил приглашения, распространившиеся по популяции, согласование дат, и вечеринка состоялась — из 24 агентов, которые не знали о ней. Абляции показывают, что все три компонента необходимы для правдоподобия. Задокументированные отказы — ошибки пространственных норм (вход в закрытые магазины, совместное использование одиночных туалетов). Это референсная архитектура для агентных симуляций и мультиагентной социальной оценки в 2026 году.

**Тип:** Изучение + Практическая работа
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 16 · 04 (Примитивная модель), Фаза 16 · 13 (Общая память)
**Время:** ~75 минут

## Проблема

Большинство мультиагентных систем — это строго сценарные команды: планировщик планирует, кодер кодит, ревьюер проверяет. Это работает для чётко определённых задач. Это не отражает эмерджентное, несценарное поведение, которое возникает, когда агенты имеют память, приоритеты и открытый мир. Исследования, симуляции общества и всё больше игровой ИИ нуждаются в этом втором типе.

Архитектура Smallville — бенчмарк для этого. До Park 2023 лучшие агентные симуляции были поверхностными последователями сценария; после него паттерн стал дефолтом для генеративных агентов в открытых мирах. Если вы строите агентную симуляцию в 2026 году, вы либо используете три компонента Smallville, либо явно обосновываете, почему не используете.

## Концепция

### Три компонента

**Поток памяти.** Лог только добавления для наблюдений, действий, рефлексий и планов. Каждая запись имеет временную метку, тип, описание (на естественном языке) и производные метаданные: **актуальность**, **важность** (самооценка 1-10 агентом) и **релевантность** (косинусное сходство с текущим запросом).

```
[2026-02-14 09:12:03] наблюдение: Isabella Rodriguez спросила меня, нравится ли мне джаз
[2026-02-14 09:14:22] рефлексия:   Я люблю долгие разговоры о музыке
[2026-02-14 10:05:00] план:         Посетить вечеринку Isabella в День святого Валентина сегодня вечером
```

Извлечение памяти комбинирует три оценки: `score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`. Top-k записей попадают в текущий промпт.

**Рефлексия.** Периодически (каждые N воспоминаний или при важных событиях) агент генерирует синтезы более высокого порядка из недавних воспоминаний. Записи рефлексий возвращаются в поток и извлекаются как любое другое воспоминание. Так агенты строят «понимания» — эквивалент долгосрочных убеждений в архитектуре.

**План.** Декомпозиция сверху вниз. Сначала план на день в общих чертах («пойти на работу, поужинать с Клаусом»). Затем планы на час. Затем планы на действие. Планы пересматриваются: когда наблюдение противоречит плану, агент перепланирует затронутый сегмент.

### Почему все три важны (абляции)

Park et al. провели абляции, удаляя наблюдение, рефлексию и план по очереди. Каждая абляция снижает правдоподобие:

- Без **наблюдения** агент теряет контекст и действует на основе устаревших убеждений.
- Без **рефлексии** агент не может формировать убеждения более высокого порядка; взаимодействия остаются поверхностными.
- Без **плана** поведение становится реактивным шумом; цели рассеиваются.

Оценки правдоподобия от людей-оценщиков при наличии всех трёх; удаление любой одной даёт измеримую регрессию.

### Эмерджент Дня святого Валентина

Один агент, Isabella Rodriguez, получает цель «хочет устроить вечеринку в День святого Валентина в Hobbs Cafe 14 февраля в 17:00». 24 других агента не получают такой установки. В ходе симулируемых дней:

1. План Isabella включает приглашение людей.
2. Каждое приглашение становится наблюдением в потоке памяти соседа.
3. Рефлексия соседа генерирует убеждения: «Isabella устраивает вечеринку».
4. План соседа включает «посетить вечеринку 14 февраля».
5. Соседи рассказывают другим соседям. Приглашение распространяется без центральной координации.
6. 14 февраля в 17:00 несколько агентов сходятся в Hobbs Cafe.

Это эмерджент в техническом смысле: поведение системы (вечеринка) возникло из локальных взаимодействий (двусторонние приглашения + индивидуальное планирование) без центрального оркестратора.

### Задокументированные режимы отказа

Park et al. явно документируют:

- **Ошибки пространственных норм.** Агенты заходят в закрытые магазины. Агенты пытаются использовать один и тот же одиночный туалет. Агенты едят в комнатах, не предназначенных для еды. Модель не выводит социо-физические нормы только из окружения.
- **Переполнение памяти.** Глубокие симуляции приводят к росту стоимости извлечения памяти. Практическое средство: периодическая компактизация памяти (резюмирование + обрезка) и затухание для записей с низкой важностью.
- **Галлюцинация рефлексий.** Рефлексии могут изобретать отношения, которых нет в потоке памяти. Смягчение: включайте ID исходных воспоминаний в промпты рефлексий и проверяйте при извлечении.

Это продакшен-актуальные режимы отказа: любая агентная симуляция 2026 года наследует их.

### Правила реализации трёхкомпонентной архитектуры

1. **Память — только добавление.** Никогда не мутируйте запись памяти. Коррекции — новые записи.
2. **Оценки важности дешёвые.** Вызовите LLM для оценки важности 1-10 при записи. Кэшируйте оценку.
3. **Извлечение ранжировано, а не фильтруется.** Top-k по комбинированной оценке; не используйте жёсткие фильтры (теряют контекст).
4. **Рефлексия запускается периодически.** Триггер, когда сумма важности необработанных воспоминаний превышает порог (например, 150).
5. **Планы пересматриваются.** Когда новое наблюдение противоречит плану, перегенерируйте только затронутый сегмент, а не весь план.

### Генеративные агенты за пределами Smallville

Литература 2024-2026 годов расширяет архитектуру:

- **Мультиагентная социальная симуляция для политики/рыночных исследований.** Популяции в стиле Smallville моделируют поведение пользователей в ответ на функции. Быстрее A/B тестов; точность обсуждается.
- **NPC-ИИ для игр.** RPG с агентами Smallville генерируют эмерджентные сюжетные линии вместо сценарных квестов.
- **Бенчмарки оценки генеративных агентов.** Вместо точности задачи метрикой становится правдоподобие + согласованность поведения при долгих запусках.

Архитектура — референс. Расширения заменяют компоненты (векторное хранилище для памяти, retrieval-augmented рефлексия, нейросимволический план), но сохраняют трёхчастную структуру.

### Почему это важно для мультиагентной инженерии

Smallville — доказательство концепции, что мультиагентный эмерджент дешёв, когда компоненты правильные. Архитектура теперь воспроизведена на open-source моделях (меньшие LLM теряют правдоподобие плавно, а не резко). Любая продакшен-система, которой нужно **эмерджентное социальное поведение**, использует эту форму. Любая система, которой нужное **строгое выполнение задач**, использует паттерны супервайзер/роли/примитивы из начала этой фазы.

## Реализуем

`code/main.py` реализует три компонента на stdlib Python со сценарными политиками агентов (без реального LLM). Демо воспроизводит эмерджент вечеринки в миниатюре:

- `MemoryStream` — лог только добавления с извлечением по актуальности/важности/релевантности.
- `reflect(stream)` — сценарная рефлексия по недавним важным воспоминаниям.
- `plan(agent_state)` — планы на день и час на основе текущих убеждений.
- Сценарий: 5 агентов. Агент 1 начинает с «устроить вечеринку в 17:00». В ходе симулируемых тиков приглашение распространяется и агенты сходятся.

Запуск:

```
python3 code/main.py
```

Ожидаемый вывод: пошаговая трассировка. К последнему тику минимум 3 из 5 агентов показывают вечеринку в своем плане и сходятся на место вечеринки. Одна установка породила согласованное прибытие без какого-либо оркестратора.

## Результат

Правила для продакшен-симуляций:

- **Память — это база данных.** Выберите реальное хранилище (векторная БД, Postgres) для масштаба. In-memory stdlib — для прототипов.
- **Логируйте трассировку извлечения.** Для каждого действия логируйте top-k воспоминаний, которые его определили. Это ваша способность отладки.
- **Бюджет токенов на агента.** Каждый агент: retrieve + reflect + plan на тик = O(k) вызовов LLM. N агентов × T тиков × вызовов-на-тик может превысить ваш бюджет.
- **Компактизируйте память периодически.** Резюмируйте и обрезайте записи с низкой важностью. Политика удержания — решение проектирования, а не деталь.
- **Обнаруживайте нарушения пространственных/социальных норм** явно. Архитектура не обучает их.
