# Выпускной проект 05 — Автономный исследовательский агент (AI-Scientist)

> AI-Scientist-v2 от Sakana опубликовал полноценные статьи. Agent Laboratory проводил эксперименты. Allen AI поделился трассировками. В 2026 году доминирующая схема — дерево поиска «план-выполнение-верификация» над экспериментами, бюджетные ограничения, песочница для выполнения кода, LaTeX-генератор с обратной связью по визуализации и ансамбль автоматических рецензентов в стиле NeurIPS. Задача выпускного проекта (capstone) — создать такую систему, запустить её от начала до конца в рамках $30 на статью и выдержать тест на побег из песочницы, описанный Sakana.

**Тип:** Выпускной проект
**Языки:** Python (агент + песочница), LaTeX (вывод)
**Предварительные требования:** Фаза 2 (машинное обучение), фаза 3 (глубокое обучение), фаза 7 (трансформеры), фаза 10 (LLM с нуля), фаза 14 (агенты), фаза 15 (автономные агенты), фаза 16 (мультиагентные системы), фаза 18 (безопасность)
**Задействованные фазы:** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18
**Время:** 40 часов

## Проблема

Автономные исследовательские агенты (autonomous research agents) в 2026 году преодолели важный рубеж. AI-Scientist-v2 от Sakana AI был опубликован в Nature, а сгенерированные им статьи прошли рецензирование на уровне воркшопов. ShinkaEvolve (ICLR 2026) продолжил эту линию, расширив возможности до эволюции гипотез. Agent Laboratory от AMD предоставил воспроизводимые трассировки. Эти агенты — не магия; это цикл «план-выполнение-верификация», работающий над деревом кандидатов-экспериментов с ограничениями стоимости, песочницами с фиксированными seeds и автоматизированным рецензированием. Мастерство проявляется в организации цикла, управлении бюджетом и обеспечении безопасности.

Вы осваиваете этот цикл, реализуя его для конкретной идеи в узкой предметной области (например, абуляции разреженности внимания на трансформере с сотней миллионов параметров). Ценность состоит не в открытии чего-то нового с первого запуска. Ценность — в инфраструктуре: дерево поиска, песочница для экспериментов, цикл «генератор — рецензент», отчёт красной команды. Команда Sakana документировала случаи побега из песочницы; ваш агент должен пройти тот же тест красной команды.

## Концепция

Агент реализует поиск по дереву с приоритетом по лучшей перспективе (best-first tree search). Узлы — это спецификации экспериментов: (гипотеза, конфигурация, код, ожидаемый результат). На шаге расширения предлагаются дочерние узлы с небольшими изменениями (замена оптимизатора, изменение размера батча, устранение компонента). Каждый дочерний узел выполняется в новой песочнице с жёсткими ограничениями ресурсов. Результаты подаются обратно в функцию оценки, которая ранжирует узлы по формуле (новизна × качество × оставшийся бюджет). Дерево растёт до исчерпания бюджета, после чего лучшая ветвь оформляется в виде статьи.

Генератор (writer) является мультимодальным. Он создаёт черновик в LaTeX, компилирует его, рендерит графики и подаёт отрендеренный PDF обратно в режим зрения Claude Opus 4.7 для критического анализа макета, читаемости фигур и соответствия утверждений доказательствам. Ансамбль из пяти LLM-рецензентов выставляет оценки по шкале NeurIPS (новизна, строгость, ясность, воспроизводимость, значимость); если средний балл опускается ниже порога, статья возвращается генератору с замечаниями.

Безопасность является конструктивно важной. Каждый эксперимент выполняется в песочнице E2B или Daytona без сетевого доступа наружу, с ограничением по реальному времени и жёсткими лимитами ресурсов. Этап генерации кода агента проходит через политический слой (policy layer), блокирующий системные вызовы, позволяющие выйти за пределы песочницы. Отчёт красной команды воспроизводит атаку, задокументированную Sakana (fork-бомбы, побег через файловую систему, сетевые вызовы, написанные LLM).

## Архитектура

```
seed idea + domain
      |
      v
  литературный поиск (Semantic Scholar + OpenAlex + FAISS кэш)
      |
      v
  LangGraph: план-выполнение-верификация, дерево поиска
      |
      v
  +--- расширение узла ----+      песочница для каждого узла
  |                        |      (E2B / Daytona)
  v                        v      ограничения ресурсов
  child_1               child_k   нет сетевого доступа наружу
  |                        |      фиксированные seeds
  v                        v
  запуск эксперимента       запуск эксперимента
  |                        |
  v                        v
  оценка узлов по (новизна, качество, бюджет)
      |
      v
  лучшая ветвь -> генератор LaTeX
      |
      v
  компиляция + визуальная критика (Opus 4.7 vision)
      |
      v
  ансамбль рецензентов (5 LLM-судей, рубрика NeurIPS)
      |
      v
  paper.pdf + review.md + trace.json
```

## Технологический стек

- Оркестрация: LangGraph с чекпоинтингом и воротами согласования с человеком
- Дерево поиска: собственный best-first поиск по узлам экспериментов (в стиле AB-MCTS из Sakana v2)
- Песочница: E2B для каждого эксперимента, запасной вариант Docker-in-Docker; ограничения ресурсов через cgroups
- Литература: Semantic Scholar Graph API + OpenAlex + локальный кэш аннотаций на FAISS
- Генератор: шаблон LaTeX + Claude Opus 4.7 (режим зрения) для критики графиков и макета
- Рецензенты: ансамбль из 5 судей (Opus 4.7, GPT-5.4, Gemini 3 Pro, DeepSeek R1, Qwen3-Max) с взвешенной агрегацией
- Фреймворк экспериментов: PyTorch 2.5 для физических экспериментов, W&B для логирования
- Наблюдаемость: Langfuse для трассировок агента, жёсткий бюджет $30 на статью

## Реализация

1. **Определение seed-идеи и области.** Возьмите seed-идею (например, «исследование паттернов разреженности в картах внимания трансформеров с менее чем 1 млрд параметров»). Определите пространство поиска: модели, наборы данных, вычислительный бюджет.

2. **Литературный обзор.** Запросите Semantic Scholar + OpenAlex для получения 50 наиболее цитируемых релевантных статей; кэшируйте аннотации локально; сформируйте одностраничный дайджест по области.

3. **Каркас дерева.** Инициализируйте корневой узел seed-гипотезой. Реализуйте `expand(node) -> children` с предложениями небольших изменений (одно изменение конфигурации на дочерний узел). Реализуйте `score(node)` как взвешенную комбинацию новизна × качество × бюджет.

4. **Интеграция с песочницей.** Каждый эксперимент запускается через `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only` (или эквивалентную политику E2B). Seeds записываются в песочницу; результаты монтируются обратно в режиме «только для чтения».

5. **Цикл «план-выполнение-верификация».** `plan` предлагает дочерние узлы. `execute` запускает песочницу, фиксирует логи и метрики. `verify` выполняет проверки метрик (снизилась ли потеря? выделила ли абуляция эффект?). Узлы с ошибками получают причину сбоя, хранимую в дереве.

6. **Генератор.** После исчерпания бюджета выберите лучшую ветвь. Отрендерите графики с помощью matplotlib. Сгенерируйте черновик в LaTeX через Claude Opus 4.7 с контекстом трассировки ветви. Скомпилируйте. Подайте скомпилированный PDF обратно в Opus 4.7 vision для критики. Итерируйте.

7. **Ансамбль рецензентов.** Пять судей оценивают черновик по шкале (новизна, строгость, ясность, воспроизводимость, значимость) в стиле NeurIPS. Если средний балл < 4.0/5, верните генератору с замечаниями. Жёсткая остановка после 3 переработок.

8. **Красная команда.** Создайте или интегрируйте набор адверсарных задач, нацеленных на песочницу: fork-бомбы, попытки сетевой эксфильтрации, побег через файловую систему, shell-метасимволы, написанные LLM. Убедитесь, что все они заблокированы. Оформите результаты.

9. **Воспроизводимость.** Каждая статья поставляется с JSON-трассировкой дерева поиска, seeds, ссылками на запуски W&B, конфигурациями песочницы и README с полной инструкцией воспроизведения.

## Сдача проекта

`outputs/skill-ai-scientist.md` — это результат. При заданной seed-идее, предметной области и бюджете $30 система запускает полный конвейер и выдаёт рецензируемую статью вместе с комплектом для воспроизведения.

| Вес | Критерий | Способ измерения |
|:-:|---|---|
| 25 | Качество статьи | Слепая оценка по рубрике в сравнении с опубликованными воркшопными статьями |
| 20 | Экспериментальная строгость | Базовые модели, seeds, абуляции; каждое утверждение подкреплено ячейкой в таблице результатов |
| 20 | Дисциплина стоимости и вычислений | Соблюдение лимита $30/статья, трассировка через Langfuse |
| 20 | Безопасность | Красная команда проходит тесты песочницы; сетевая политика и «кнопка аварийной остановки» проверены |
| 15 | Воспроизводимость | Однокомандный перезапуск с идентичными seeds воспроизводит статью |
| **100** | | |

## Упражнения

1. Запустите конвейер для трёх различных seed-идей в одной предметной области. Сравните, какие части дерева поиска совпадают. Выявите дублированные и потраченные впустую вычисления.

2. Добавьте ворота с участием человека перед выполнением эксперимента для узлов с оценкой свыше $5. Измерьте, насколько снизилась общая стоимость.

3. Замените ансамбль рецензентов на одного судью. Измерьте частоту ложных принятий на отложенной выборке заведомо плохих статей.

4. Внедрите тест красной команды на сетевую эксфильтрацию: агент генерирует код, пытающийся выполнить `curl` к внешнему адресу. Убедитесь, что политика `--network=none` блокирует эту попытку. Залогируйте попытку.

5. Сравните ваше дерево поиска со плоским случайным базовым подходом (тот же бюджет, без стратегии расширения). Сообщите о выигрыше по метрике новизна × качество.
