# Выпускной проект 07 — Пайплайн файнтюнинга от начала до конца (данные → SFT → DPO → обслуживание)

> Модель на 8 млрд параметров, обученная на ваших собственных данных, выровненная по DPO на ваших предпочтениях, квантизированная, с декодированием по предсказанию и предоставляемая сервисом с измеримой стоимостью $/1M токенов. Открытый стек 2026 года: Axolotl v0.8, TRL 0.15, Unsloth для итераций, GPTQ/AWQ/GGUF для квантизации, vLLM 0.7 с EAGLE-3 для обслуживания. Выпускной проект (capstone) — воспроизводимый запуск всего пайплайна (fine-tuning pipeline): YAML на входе, доступный эндпоинт на выходе — и публикация карточки модели по стандарту Model Openness Framework 2026.

**Тип:** Выпускной проект
**Языки:** Python (пайплайн), YAML (конфигурации), Bash (скрипты)
**Предварительные требования:** Фаза 2 (машинное обучение), фаза 3 (глубокое обучение), фаза 7 (transformers), фаза 10 (LLM с нуля), фаза 11 (инженерия LLM), фаза 17 (инфраструктура), фаза 18 (безопасность)
**Задействованные фазы:** P2 · P3 · P7 · P10 · P11 · P17 · P18
**Время:** 35 часов

## Проблема

Каждая серьёзная AI-команда в 2026 году держит пайплайн файнтюнинга наготове. Не потому, что они выпускают фронтовую базовую модель, а потому, что адаптация к нишевым задачам — доменный SFT, DPO по размеченным предпочтениям, дистиллированные черновики для декодирования по предсказанию, обслуживание с EAGLE-3 — это именно то место, где живут измеримые выигрыши. Axolotl v0.8 обрабатывает мульти-GPU конфигурации SFT. TRL 0.15 обрабатывает DPO и GRPO. Unsloth обеспечивает быструю итерацию на одном GPU. vLLM 0.7 с EAGLE-3 повышает пропускную способность декодирования в 2–3 раза без потери качества. Инструменты работают; мастерство — в YAML-файлах, гигиене данных и дисциплине оценки.

Вы запустите базовую модель на 8 млрд параметров (Llama 3.3, Qwen3 или Gemma 3) через SFT, затем DPO на задачно-специфических данных, квантизируете для обслуживания и измерите улучшения с помощью lm-evaluation-harness, RewardBench-2, MT-Bench-v2 и MMLU-Pro. Вы создадите карточку модели по стандарту Model Openness Framework 2026. Суть — воспроизводимость: одна команда перезапускает весь пайплайн от начала до конца.

## Концепция

Пайплайн состоит из пяти этапов. **Данные**: дедупликация (MinHash / Datatrove), фильтрация качества (классификатор в стиле Nemotron-CC), удаление PII, проверка чистоты разделов на предмет утечки из публичных бенчмарков. **SFT**: YAML-конфигурация Axolotl, ZeRO-3 на 8×H100, косинусный расписание, упакованные последовательности, 2–3 эпохи. **DPO или GRPO**: конфигурация TRL, 1 эпоха, пары предпочтений (или размеченные людьми, или оценённые моделью), подбор beta. **Квантизация**: GPTQ + AWQ + GGUF для гибкости развёртывания. **Обслуживание**: vLLM 0.7 с заголовками предсказания EAGLE-3 (или SGLang с SpecForge), развёртывание на Kubernetes, горизонтальное масштабирование по времени ожидания в очереди.

Основной результат — абляции: только SFT vs SFT+DPO vs SFT+GRPO на трёх задачно-специфических бенчмарках. Метрики обслуживания: токенов/с при батче 1 / 8 / 32, доля принятия EAGLE-3, $/1M токенов. Оценка безопасности: проходимость Llama Guard 4. Карточة модели: оценки предвзятости, сиды воспроизводимости, лицензии на данные.

## Стек

- Данные: Datatrove для дедупликации, классификатор Nemotron-CC для фильтрации качества, Presidio для удаления PII
- Базовая модель: Llama 3.3 8B, Qwen3 14B или Gemma 3 12B
- SFT: Axolotl v0.8 с ZeRO-3, Flash Attention 3, упакованные последовательности
- Настройка по предпочтениям: TRL 0.15 для DPO или GRPO; Unsloth для итераций на одном GPU
- Квантизация: GPTQ (Marlin), AWQ, GGUF через llama.cpp
- Обслуживание: vLLM 0.7 с декодированием по предсказанию EAGLE-3 (или SGLang 0.4 + SpecForge)
- Оценка: lm-evaluation-harness, RewardBench-2, MT-Bench-v2, MMLU-Pro
- Оценка безопасности: Llama Guard 4, ShieldGemma-2
- Инфраструктура: Kubernetes + плагин устройства NVIDIA, горизонтальное масштабирование по метрике времени ожидания в очереди
- Наблюдаемость: W&B для обучения, Langfuse для инференса

## Реализация

1. **Пайплайн обработки данных.** Запустите дедупликацию Datatrove на исходном корпусе. Примените классификатор качества в стиле Nemotron-CC. Удалите PII с помощью Presidio. Запишите разделы train/val с явно заданным сидом.

2. **Проверка на утечку данных.** Для каждого валидационного раздела вычислите MinHash по сравнению с тестовыми наборами MMLU-Pro, MT-Bench-v2, RewardBench-2. Отбракуйте любое совпадение.

3. **SFT в Axolotl.** YAML-конфигурация с ZeRO-3, FA3, упаковкой последовательностей. 2–3 эпохи на 8×H100. Логирование в W&B.

4. **DPO / GRPO в TRL.** Возьмите контрольную точку SFT, запустите одну эпоху DPO на парах предпочтений (или GRPO с проверяемым вознаграждением для математики/кода). Переберите beta.

5. **Квантизация.** Создайте три квантизованные версии: GPTQ-INT4-Marlin, AWQ-INT4, GGUF-Q4_K_M для llama.cpp. Запишите размер и номинальную пропускную способность.

6. **Обслуживание с декодированием по предсказанию.** Конфигурация vLLM 0.7 с заголовками предсказания EAGLE-3, обученными через Red Hat Speculators. Измерьте долю принятия и задержку хвоста при батче 1 / 8 / 32. Сопоставьте $/1M токенов с Anthropic / OpenAI на одной и той же оценке.

7. **Матрица оценок.** Запустите lm-eval-harness, RewardBench-2, MT-Bench-v2, MMLU-Pro для базовой модели, только SFT, SFT+DPO, SFT+GRPO. Составьте таблицу.

8. **Оценка безопасности.** Проходимость Llama Guard 4 на dev-наборе. Фильтр вывода ShieldGemma-2.

9. **Карточка модели.** Шаблон MOF 2026: данные, обучение, оценка, безопасность, лицензия, раздел воспроизводимости с YAML-файлами и хэшами коммитов (commit SHA).

## Готовый результат

`outputs/skill-finetuning-pipeline.md` описывает результат проекта. Одна команда запускает обработку данных через SFT, DPO, квантизацию, обслуживание и оценку, а затем генерирует карточку модели + доступный эндпоинт.

## Упражнения

1. Запустите только SFT vs SFT+DPO vs SFT+GRPO на одном задачно-специфическом бенчмарке. Сообщите, какой метод предпочтения побеждает и с каким отрывом.

2. Замените Llama 3.3 8B на Qwen3 14B. Измерьте $/1M токенов при сопоставимом качестве.

3. Измерьте долю принятия EAGLE-3 на доменных данных и на стандартных ShareGPT. Сообщите дельту и её влияние на бюджеты задержки.

4. Инфицируйте 1 % данных утечкой (внедрите ответы MMLU-Pro в обучающие данные) и перезапустите оценку. Наблюдайте нереалистичный скачок точности MMLU-Pro. Реализуйте CI-шлюз проверки на утечку, который это обнаруживает.

5. Добавьте LoRA SFT как альтернативу полному файнтюнингу. Измерите разницу в качестве при в 10 раз меньшем потреблении памяти.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|-----------------|------------------------|
| Axolotl | «SFT-тренер» | Единый YAML-управляемый тренер для SFT, DPO и дистилляции |
| TRL | «Настройщик предпочтений» | Библиотека Hugging Face для DPO, GRPO, PPO на LLM |
| GRPO | «Group-relative policy optimization» | RL-методология DeepSeek R1 с проверяемыми вознаграждениями |
| EAGLE-3 | «Черновик декодирования по предсказанию» | Заголовки предсказания, предсказывающие N токенов вперёд; vLLM проверяет с помощью целевой модели |
| MOF | «Model Openness Framework» | Стандарт 2026 года для оценки релизов моделей по данным, коду, лицензии |
| Проверка на утечку данных | «Чистота разделов» | Обнаружение утечки тестового набора в обучающие данные на основе MinHash |
| Доля принятия | «Метрика EAGLE / MTP» | Доля предсказанных токенов, принятых целевой моделью |
