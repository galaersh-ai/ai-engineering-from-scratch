# Библиотеки навыков и обучение на протяжении жизни (Voyager)

> Voyager (Wang et al., TMLR 2024) рассматривает исполняемый код как навык (skill). Навыки именованы, доступны для извлечения, композиции и уточняются с помощью обратной связи от среды. Это эталонная архитектура для навыков Claude Agent SDK, skillkit и шаблона библиотеки навыков 2026 года.

**Тип:** Разработка (Build)
**Языки:** Python (стандартная библиотека)
**Предварительные требования:** Фаза 14 · 07 (MemGPT), Фаза 14 · 08 (Letta Blocks)
**Время:** ~75 минут

## Цели обучения

- Назовите три компонента Voyager — автоматическая учебная программа (automatic curriculum), библиотека навыков (skill library), итеративный промптинг (iterative prompting) — и опишите роль каждого.
- Объясните, почему Voyager делает пространство действий кодом, а не примитивными командами.
- Реализуйте библиотеку навыков (skill library) средствами стандартной библиотеки (stdlib) с регистрацией, извлечением, композицией и уточнением на основе отказов (failure-driven refinement).
- Сопоставьте шаблон Voyager с навыками Claude Agent SDK 2026 года и экосистемой skillkit.

## Проблема

Агенты (agent), которые заново создают каждую возможность в каждой сессии, допускают три ошибки:

1. **Тратят токены (tokens).** Каждая задача заново вызывает одни и те же рассуждения.
2. **Теряют прогресс.** Исправление, усвоенное в сессии A, не переносится в сессию B.
3. **Не справляются с долгосрочной композицией (long-horizon composition).** Сложные задачи требуют иерархий возможностей; одношаговые промпты не могут их выразить.

Ответ Voyager: рассматривать каждую повторно используемую возможность как именованный фрагмент кода, хранящийся в библиотеке, доступный для извлечения по сходству, композиции с другими навыками и уточнения с помощью обратной связи от выполнения.

## Концепция

### Три компонента

Voyager (arXiv:2305.16291) структурирует агента вокруг:

1. **Автоматическая учебная программа (automatic curriculum).** Предлагатель, управляемый любопытством, выбирает следующую задачу на основе текущего набора навыков агента и состояния среды. Исследование идёт снизу вверх.
2. **Библиотека навыков (skill library).** Каждый навык — это исполняемый код. Новые навыки добавляются при успешном выполнении задачи. Навыки извлекаются по сходству запроса с описанием.
3. **Механизм итеративного промптинга (iterative prompting).** При отказе агент получает ошибки выполнения, обратную связь от среды и результаты самопроверки, затем уточняет навык.

Оценка в Minecraft (Wang et al., 2024): 3,3 раза больше уникальных предметов, 8,5 раза быстрее каменные инструменты, 6,4 раза быстрее железные инструменты, 2,3 раза большее перемещение по карте по сравнению с базовыми моделями. Цифры специфичны для Minecraft, но шаблон переносим.

### Пространство действий = код

Большинство агентов генерируют примитивные команды. Voyager генерирует JavaScript-функции. Навык — это:

```javascript
async function craftIronPickaxe(bot) {
  await mineIron(bot, 3);
  await mineStick(bot, 2);
  await placeCraftingTable(bot);
  await craft(bot, 'iron_pickaxe');
}
```

Составлен из поднавыков. Хранится с ключами по описанию и эмбеддингу (embedding). Извлекается как программа, а не как промпт.

Это навык Claude Agent SDK 2026 года: именованный, доступный для извлечения фрагмент кода плюс инструкции, которые агент загружает по запросу.

### Извлечение навыков (skill retrieval)

Новая задача: «сделать алмазную кирку». Агент:

1. Создаёт эмбеддинг описания задачи.
2. Запрашивает библиотеку навыков для поиска k наиболее похожих навыков.
3. Извлекает `craftIronPickaxe`, `mineDiamond`, `placeCraftingTable` и т.д.
4. Составляет новый навык из извлечённых примитивов + новой логики.

Это шаблон, который реализуют ресурсы MCP (Фаза 13) и навыки Agent SDK: извлечение по поверхности знаний/кода, ограниченное текущей задачей.

### Итеративное уточнение (iterative refinement)

Петля обратной связи Voyager:

1. Агент создаёт навык.
2. Навык запускается в среде.
3. Возвращается один из трёх сигналов: `success` (успех), `error` (ошибка с трассировкой стека), `self-verification failure` (отказ самопроверки).
4. Агент переписывает навык, используя сигнал как контекст.
5. Цикл (loop) повторяется до достижения успеха или максимального числа раундов.

Это Self-Refine (Урок 05), применённый к генерации кода с верификацией, основанной на среде. CRITIC (Урок 05) — тот же шаблон с внешними инструментами в качестве верификатора.

### Учебная программа и исследование

Модуль учебной программы (curriculum) Voyager предлагает задачи вроде «построить убежище у озера» на основе того, что агент уже умеет и чего ещё не делал. Предлагатель использует состояние среды + инвентарь навыков для выбора задачи, находящейся чуть выше текущего уровня возможностей — оптимальная точка для исследования.

Для продакшн-агентов это трансформируется в оператор «чего не хватает»: учитывая текущую библиотеку навыков и домен, какие навыки мы ещё не покрываем? Команды обычно реализуют это вручную как обзор учебной программы.

### Где этот шаблон даёт сбой

- **Деградация библиотеки навыков (skill library rot).** Один и тот же навык добавлен 10 раз с незначительно различающимися описаниями. Добавьте дедупликацию при записи; извлечение вернёт только один экземпляр.
- **Дрейф (drift) составных навыков.** Родительский навык зависит от дочернего, который был уточнён. Версионируйте (version control) навыки; родитель, привязанный к v1, не должен автоматически подхватывать v3.
- **Качество извлечения.** Векторное извлечение по описаниям навыков деградирует при росте библиотеки за несколько сотен записей. Дополняйте фильтрами по тегам и жёсткими ограничениями («только навыки с `category=tooling`»).

## Реализация

`code/main.py` реализует библиотеку навыков средствами стандартной библиотеки:

- `Skill` — имя, описание, код (в виде строки), версия, теги, зависимости.
- `SkillLibrary` — регистрация, поиск (перекрытие токенов), композиция (топологическая сортировка зависимостей) и уточнение (увеличение версии при обновлении).
- Запрограммированный агент, который регистрирует три примитивных навыка, составляет четвёртый, сталкивается с отказом и уточняет его.

Запуск:

```
python3 code/main.py
```

Трассировка показывает записи в библиотеку, извлечение, композицию, неудачное выполнение и уточнение до v2 — полный цикл Voyager от начала до конца.

## Применение

- **Навыки Claude Agent SDK** (Anthropic) — эталон 2026 года: каждый навык имеет описание, код и инструкции; загружается по запросу во время сессии агента.
- **skillkit** (npm: skillkit) — кросс-агентное управление навыками для 32+ AI-агентов кодирования.
- **Пользовательские библиотеки навыков** — доменно-специфичные (SQL-навыки для агентов данных, Terraform-навыки для агентов инфраструктуры). Шаблон Voyager масштабируется вниз.
- **`tools` OpenAI Agents SDK** — на нижнем уровне; каждый инструмент является облегчённым навыком.

## Упражнения

1. Добавьте детектор циклических зависимостей в `compose()`. Что происходит, когда навык A зависит от B, который зависит от A? Ошибка или предупреждение?
2. Реализуйте привязку версий для каждого навыка. Когда родительский навык композирует дочерний `crafting@1`, уточнение до `crafting@2` не должно молча обновлять родительский навык.
3. Замените извлечение по перекрытию токенов на эмбеддинги (embedding) sentence-transformers (или реализацию BM25 из стандартной библиотеки). Измерьте retrieval@5 на тестовой библиотеке из 50 навыков.
4. Добавьте агента «учебная программа»: учитывая текущую библиотеку и описание домена, предложите 5 недостающих навыков. Запускайте его еженедельно.
5. Прочитайте документацию по навыкам Anthropic Claude Agent SDK. Перенесите тестовую библиотеку на схему навыков SDK. Что меняется в обнаруживаемости?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|--------|-------------|----------------------|
| Навык (skill) | «Повторно используемая возможность» | Именованный фрагмент кода + описание, доступный для извлечения по сходству |
| Библиотека навыков (skill library) | «Память агента о том, как делать» | Постоянное хранилище навыков, доступное для поиска и композиции |
| Учебная программа (curriculum) | «Предлагатель задач» | Генератор целей снизу вверх, управляемый текущим разрывом возможностей |
| Композиция (composition) | «Навыковый DAG» | Навыки, вызывающие другие навыки; топологически сортируемые при выполнении |
| Итеративное уточнение (iterative refinement) | «Петля самокоррекции» | Обратная связь от среды + ошибки + самопроверка возвращаются в следующую версию |
| Пространство действий как код (action-space-as-code) | «Программные действия» | Генерируются функции, а не примитивные команды, для поведения, простирающегося во времени |
| Дедупликация при записи (dedup on write) | «Коллапс навыков» | Почти дублирующие описания сводятся к одному каноническому навыку |
