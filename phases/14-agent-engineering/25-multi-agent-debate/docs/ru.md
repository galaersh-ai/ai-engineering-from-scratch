# Мультиагентный дебат (Multi-Agent Debate) и сотрудничество

> Du et al. (ICML 2024, «Society of Minds») запускают N экземпляров (instance) моделей, которые независимо предлагают ответы, а затем итеративно критикуют друг друга на протяжении R раундов, стремясь к сходимости (convergence). Повышает фактологическую точность, следование правилам и рассуждение. Разреженная топология (sparse topology) превосходит полную сетку (full mesh) по стоимости в токенах (token).

**Тип:** Изучение + Сборка
**Языки:** Python (стандартная библиотека)
**Предварительные требования:** Фаза 14 · 12 (Паттерны воркфлоу), Фаза 14 · 05 (Self-Refine и CRITIC)
**Время:** ~60 минут

## Цели обучения

- Описать протокол дебат (debate protocol): N предлагающих, R раундов, сходимость на общий ответ.
- Объяснить, почему дебат повышает фактологическую точность, следование правилам и рассуждение.
- Описать разреженную топологию: не каждому участнику дебат (debater) нужно видеть каждого другого.
- Реализовать дебат на стандартной библиотеке Python для scripted LLM с вариантами полной сетки и разреженной топологии; измерить стоимость в токенах и точность.

## Проблема

Self-Refine (Урок 05) — это одна модель, которая критикует сама себя — возникает риск группового мышления (groupthink). CRITIC (Урок 05) основывает критику на внешних инструментах — которые не всегда доступны. Дебат вводит третий подход: несколько экземпляров, взаимная критика, сходимость через разногласия.

## Концепция

### Общество разумов (Society of Minds) (Du et al., ICML 2024)

- N экземпляров модели (instance) независимо предлагают ответы на один и тот же вопрос.
- На протяжении R раундов каждый экземпляр модели читает предложения других и критикует их.
- Модели (model) обновляют свои ответы на основе критики.
- После R раундов возвращается общий сходящийся ответ.

В оригинальных экспериментах использовались N=3, R=2 из-за стоимости. Точность возрастает с увеличением числа агентов и раундов на сложных задачах (MMLU, GSM8K, проверка допустимости шахматных ходов, генерация биографий).

Кросс-модельные комбинации превосходят дебаты одной модели: ChatGPT + Bard вместе > каждый по отдельности.

### Разреженная топология (Sparse Topology)

«Improving Multi-Agent Debate with Sparse Communication Topology» (arXiv:2406.11776, 2024–2025) показал, что дебат с полной сеткой не всегда оптимален. Разреженные топологии (звёздная, кольцевая, hub-and-spoke) могут обеспечить сопоставимую точность при меньшей стоимости в токенах. Каждый участник дебат видит только подмножество коллег.

Последствия:

- Полная сетка N=5, R=3 = 5 × 3 = 15 предложений, каждое читает 4 коллег = 60 операций критики.
- Звёздная топология N=5, R=3 (один центральный узел + 4 лучевых) = 15 предложений, лучевые узлы читают только центр = 12 операций критики.

### Когда дебат помогает

- **Точность (фактологическая).** N независимых предложений, взаимная проверка снижает галлюцинации (hallucination).
- **Следование правилам.** Проверка допустимости шахматных ходов — одна модель упускает правило, другие его находят.
- **Рассуждение без ограничений.** Несколько фреймворков сужаются к правильному ответу.

### Когда дебат вредит

- **Чувствительность к задержкам (UX).** N × R последовательных раундов — это задержка, которую вы можете не себе позволить.
- **Чувствительность к стоимости.** N × R токенов на вопрос.
- **Простые фактологические запросы.** Одно обращение к базе знаний дешевле, чем пять дебатов.

### Практические инстанциации 2026 года

- **Оркестратор-рабочие агенты (orchestrator-workers) от Anthropic** (Урок 12) — один из вариантов дебат (debate) с шагом синтеза.
- **Супервайзер (supervisor) в LangGraph** (Урок 13) — центральный маршрутизатор + специализированные агенты могут реализовать дебат как узел (node).
- **OpenAI Agents SDK** (Урок 16) — агенты передают (handoff) управление друг другу для итеративной критики.
- **Мультиагентная валидация (multi-agent evals)** — комбинация дебат + evaluator-optimizer для сигнала оценки (evaluation).

### Где этот паттерн может не сработать

- **Схлопывание сходимости.** Все агенты сходятся к первому неверному ответу. Смягчается обязательными раундами разногласий.
- **Отказ центрального узла (hub failure).** В звёздной топологии (star topology) неисправный центральный узел искажает результаты всех. Ротация или использование нескольких центральных узлов.
- **Однородность промптов.** Все агенты используют один и тот же промпт и дают одинаковые ответы. Используйте разнообразные промпты и/или модели.

## Реализация

`code/main.py` реализует дебат на стандартной библиотеке Python:

- Класс `Debater` (scripted LLM с дрейфом (drift) мнений для каждого участника).
- Запускаторы `FullMeshDebate` и `SparseDebate`.
- Три вопроса: один фактологический, один основанный на правилах, один на рассуждение.
- Метрики: сходящийся ответ, количество раундов до сходимости, общее число операций критики.

Запуск:

```
python3 code/main.py
```

Вывод: точность и стоимость для каждого протокола; разреженная топология соответствует полной сетке на 2/3 вопросов при меньшей стоимости.

## Применение

- **Оркестратор-рабочие агенты от Anthropic** для простых дебат из 2–3 рабочих агентов.
- **LangGraph** для состоящего (stateful) много раундового дебат с чекпоинтингом (checkpoint).
- **Кастомная реализация** для исследований или специализированных гарантий корректности.

## Деплой

`outputs/skill-debate.md` предоставляет шаблон (scaffold) для мультиагентного дебат с настраиваемой топологией, параметрами N и R, а также правилом сходимости.

## Упражнения

1. Реализуйте правило «вынужденного разногласия»: в раунде 1 каждый участник дебат обязан предложить уникальное предложение. Измерьте влияние на скорость сходимости.
2. Добавьте агрегацию с весами по уверенности: участники возвращают (ответ, уверенность); агрегатор взвешивает по уверенности. Помогает ли это?
3. Замените одного «агента» на другой scripted LLM с другими мнениями. Улучшает ли гетерогенность точность?
4. Измерьте стоимость в токенах для полной сетки и разреженной топологии на ваших 3 вопросах. Постройте график стоимость vs точность.
5. Прочитайте статью «Society of Minds». Перенесите вашу игрушечную реализацию на N=5, R=3. Что ломается? Что улучшается?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|--------|------------|----------------------|
| Дебат (Debate) | «Мультиагентная критика» | N предлагающих, R раундов взаимной критики, сходимость |
| Полная сетка (Full mesh) | «Все читают всех» | Каждый участник дебат читает каждого коллегу каждый раунд |
| Разреженная топология (Sparse topology) | «Ограниченный обзор коллег» | Участники дебат читают только подмножество коллег |
| Hub-and-spoke | «Звёздная топология» | Один центральный участник дебат, N-1 лучевых читают только его |
| Сходимость (Convergence) | «Согласие» | Участники дебат сходятся к общему ответу |
| Общество разумов (Society of Minds) | «Статья Du et al. о дебатах» | Метод мультиагентного дебат, ICML 2024 |
