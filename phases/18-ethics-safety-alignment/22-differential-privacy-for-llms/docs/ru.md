# Дифференциальная приватность для LLM

> DP-SGD остаётся стандартом — обновления градиентов с добавлением шума обеспечивают формальные гарантии (epsilon, delta). Издержки по вычислениям, памяти и полезности существенны; параметрически эффективная DP-адаптация (LoRA + DP-SGD) является распространённой конфигурацией 2025 года (ACM 2025). Два противоречивых источника свидетельств: метод canary-атак для вывода о принадлежности (Duan et al., 2024) сообщает об ограниченном успехе против языковых моделей; извлечение данных обучения (Carlini et al., 2021; Nasr et al., 2025) выявляет значительное дословное запоминание. Разрешение (arXiv:2503.06808, март 2025): расхождение заключается в том, что измеряется — вставленные canary-объекты по сравнению с «наиболее извлекаемыми» данными. Новые конструкции canary позволяют проводить MIA-тестирование на основе функции потерь без теневых моделей и дают первую незначительную DP-аудиторскую проверку LLM, обученной на реальных данных с реалистичными DP-гарантиями. Альтернативы: PMixED (arXiv:2403.15638) — приватное предсказание во время инференса посредством смеси экспертов для распределений следующих токенов; генерация синтетических данных с DP (Google Research 2024). Новая атака: обратное вычисление дифференциальной приватности через обратную связь LLM — утечка оценок уверенности.

**Тип:** Практическая работа
**Языки:** Python (стандартная библиотека, демонстрация добавления шума в DP-SGD и счётчика ε-δ)
**Предварительные знания:** Фаза 01 · 09 (информационная теория), Фаза 10 · 01 (обучение крупных моделей)
**Время:** ~60 минут

## Цели обучения

- Определить (epsilon, delta)-дифференциальную приватность (differential privacy) и сформулировать алгоритм DP-SGD.
- Объяснить противоречие 2024–2025 годов: canary MIA и извлечение данных обучения дают различные картины.
- Описать PMixED и объяснить, почему приватное предсказание во время инференса является альтернативой DP-обучению.
- Описать атаку «обратное вычисление дифференциальной приватности через обратную связь LLM».

## Проблема

LLM запоминают. Carlini et al. 2021 показали, что языковые модели-production воспроизводят дословный текст из данных обучения по запросу. DP — формальная защита: обучение должно обеспечивать доказуемую нечувствительность выхода к любому отдельному обучающему примеру. Свидетельства 2024–2025 годов показывают, что DP-SGD необходим, но используемые на практике значения ε могут не соответствовать модели угроз.

## Концепция

### (ε, δ)-дифференциальная приватность

Случайный алгоритм M является (ε, δ)-DP, если для любых двух наборов данных, отличающихся на один пример, и любого события S:
P(M(D) ∈ S) ≤ e^ε · P(M(D') ∈ S) + δ.

Интерпретация: распределение выходов достаточно близко (параметризовано ε), чтобы вклад любого отдельного человека не мог быть достоверно вычислен, за исключением случая с вероятностью δ.

### DP-SGD

Abadi et al. 2016. Стандартный алгоритм:
1. Выборка мини-пакета.
2. Вычисление градиентов для каждого примера.
3. Обрезка каждого отдельного градиента до порога C.
4. Суммирование обрезанных градиентов и добавление гауссова шума со стандартным отклонением σ · C.
5. Использование зашумлённой суммы для обновления параметров.

Стоимость приватности отслеживается с помощью счётчика (Moments Accountant, Rényi DP accountant). Опубликованные значения ε в литературе по LLM сильно различаются в зависимости от модели угроз, чувствительности данных и целей по полезности; универсального «безопасного» значения ε по умолчанию не существует. Опубликованные примеры варьируются примерно от ε ≈ 1 до 10 в некоторых конфигурациях обучения LLM, однако это иллюстративные значения — не рекомендуемые по умолчанию. Более низкие значения ε, как правило, требуют большего шума и могут увеличить потерю полезности.

### LoRA + DP-SGD

Полная реализация DP-SGD для фронтальной модели является непомерно дорогостоящей. LoRA (Hu et al. 2022) ограничивает обновления градиентов небольшим адаптером, сокращая хранение градиентов для каждого примера. LoRA + DP-SGD является распространённой конфигурацией 2025 года. Гарантии DP распространяются на адаптер; базовая модель фиксируется.

### Противоречие 2024–2025 годов

Два направления свидетельств:

- **Canary MIA (Duan et al. 2024).** Вставка уникальных canary-объектов в данные обучения и измерение того, может ли атакующий, использующий вывод о принадлежности (membership inference), их идентифицировать. Сообщается об ограниченном успехе на языковых моделях. Это указывает на сложность MIA.
- **Извлечение данных обучения (Carlini 2021, Nasr et al. 2025).** Подача модели префикса и измерение того, воспроизводит ли она дословный текст из данных обучения. Сообщается о значительном запоминании. Это указывает на то, что MIA уязвим в соответствующем смысле.

Разрешение марта 2025 (arXiv:2503.06808): два метода измеряют различные вещи. MIA задаёт вопрос «находится ли пример e в D?» на вставленных canary-объектах. Извлечение задаёт вопрос «что я могу восстановить из D?». «Наиболее извлекаемый» пример имеет значение для приватности; canary-объекты занижают эту оценку, поскольку они не оптимизированы для извлекаемости.

Новые конструкции canary. MIA на основе функции потерь без теневых моделей. Первая незначительная DP-аудиторская проверка LLM на реальных данных с реалистичными DP-гарантиями.

### Альтернативы DP-обучению

- **PMixED (arXiv:2403.15638).** Приватное предсказание во время инференса. Смесь экспертов (mixture of experts) для распределений следующих токенов; каждый эксперт видит фрагмент данных обучения; агрегация добавляет шум для DP. Полностью исключает необходимость DP-обучения.
- **Генерация синтетических данных с DP (Google Research 2024).** LoRA-адаптация с DP-SGD, генерация синтетических данных, обучение классификатора на downstream-задаче на синтетических данных.

Оба подхода обходят стоимость полезности полного DP-обучения ценой другой модели угроз.

### Обратное вычисление дифференциальной приватности через обратную связь LLM

Новая атака 2025 года. Использование оценок уверенности модели, обученной с DP, в качестве оракула для повторной идентификации лиц. Даже если выходы не утечают, распределения уверенности могут.

Защита: не раскрывать оценки уверенности или обрезать/квантизировать их перед раскрытием. Это дополнительное требование помимо (ε, δ)-DP-обучения.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|--------|-------------|----------------------|
| DP | «(ε, δ)-дифференциальная приватность» | Формальная приватность: распределение выходов близко при изменении набора данных на соседний |
| DP-SGD | «SGD с добавлением шума» | Обрезка градиентов + добавление гауссова шума; стандартное DP-обучение |
| LoRA + DP-SGD | «эффективная приватная адаптация» | DP-SGD на адаптерах низкого ранга; стандартная конфигурация 2025 года |
| MIA | «вывод о принадлежности» | Атака, определяющая, находился ли пример в данных обучения |
| Canary | «встроенный водяной знак-пример» | Уникальный обучающий пример, используемый для измерения утечки DP |

| Обратное вычисление DP | «атака на утечку уверенности» | Атака, использующая оценки модели уверенности в качестве оракула для повторной идентификации |
