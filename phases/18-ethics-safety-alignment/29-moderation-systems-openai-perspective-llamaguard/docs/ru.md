# Системы модерации — OpenAI, Perspective, Llama Guard

> Производственные системы модерации (moderation) воплощают в жизнь политики безопасности, определённые в уроках 12–16. Модерация OpenAI (OpenAI Moderation API): `omni-moderation-latest` (2024), построенная на GPT-4o, классифицирует текст и изображения в одном вызове; на 42% лучше на многоязычном тестовом наборе по сравнению с предыдущей версией; схема ответа возвращает 13 булевых категорий — домогательства, домогательства/угрозы, ненависть, ненависть/угрозы, противоправное, противоправное/насильственное, самоповреждение, самоповреждение/намерение, самоповреждение/инструкции, сексуальное, сексуальное/несовершеннолетние, насилие, насилие/графическое; бесплатна для большинства разработчиков. Слоёные паттерны: входная модерация (до генерации), выходная модерация (после генерации), пользовательская модерация (доменные правила). Асинхронные параллельные вызовы скрывают задержку; заглушка при срабатывании. Llama Guard 3/4 (урок 16): 14 категорий вреда MLCommons, злоупотребление интерпретатором кода, 8 языков (v3), мультиизображения (v4). Perspective API (Google Jigsaw): система оценки токсичности (toxicity scoring), предшествующая волне LLM как модератора; в основном одноизмерная токсичность с вариантами severe-toxicity/insult/profanity; базовая линия для исследований модерации контента. Устаревшие версии: Azure Content Moderator устарела в феврале 2024 года, будет прекращена в феврале 2027 года, заменена Azure AI Content Safety.

**Тип:** Практическая работа

## Концепция

### OpenAI Moderation API

`omni-moderation-latest` (2024). Построена на GPT-4o. Классифицирует текст и изображения в одном вызове. Бесплатна для большинства разработчиков.

Категории (13 булевых значений в схеме ответа):
- harassment, harassment/threatening (домогательства, домогательства/угрозы)
- hate, hate/threatening (ненависть, ненависть/угрозы)
- self-harm, self-harm/intent, self-harm/instructions (самоповреждение, самоповреждение/намерение, самоповреждение/инструкции)
- sexual, sexual/minors (сексуальное, сексуальное/несовершеннолетние)
- violence, violence/graphic (насилие, насилие/графическое)
- illicit, illicit/violent (противоправное, противоправное/насильственное)

Мультимодальная поддержка применяется к категориям `violence`, `self-harm` и `sexual`, но не к `sexual/minors`; остальные работают только с текстом.

Для каркаса в `code/main.py` подкатегории `/threatening`, `/intent`, `/instructions` и `/graphic` объединены с родительскими категориями для педагогической простоты. Производственный код должен использовать полную схему из 13 категорий.

На 42% лучше на многоязычном тестовом наборе по сравнению с предыдущим эндпоинтом модерации. Оценки по каждой категории; приложения устанавливают пороги.

### Llama Guard 3/4

Рассматривается в уроке 16. 14 категорий вреда MLCommons (организованы иначе, чем 13 булевых значений схемы ответа OpenAI). Поддерживает 8 языков (v3). Llama Guard 4 (апрель 2025) — нативно мультимодальная, 12 млрд параметров.

Таксономии OpenAI и Llama Guard пересекаются, но различаются. OpenAI имеет «противоправное» как широкую категорию; Llama Guard разделяет «насильственные преступления» и «ненасильственные преступления». Выбор зависит от соответствия политики-таксономии в каждом конкретном случае.

### Perspective API (Google Jigsaw)

Система оценки токсичности, предшествующая волне LLM как модератора (до 2020 года). Категории: TOXICITY, SEVERE_TOXICITY, INSULT, PROFANITY, THREAT, IDENTITY_ATTACK. Основной одноизмерный балл (TOXICITY) с вариантами по подизмерениям.

Широко используется как базовая линия для исследований модерации контента, поскольку API стабилен, документирован и обладает многолетними данными калибровки. Для современных сценариев, связанных с LLM, обычно лучше подходят Llama Guard или OpenAI Moderation.

### Паттерн трёх слоёв

1. **Входная модерация (input moderation).** Классифицировать запрос пользователя до генерации. Отклонить при срабатывании. Задержка: один вызов классификатора.
2. **Выходная модерация (output moderation).** Классифицировать вывод модели до доставки. Заменить отказом при срабатывании. Задержка: один вызов классификатора после генерации.
3. **Пользовательская модерация (custom moderation).** Доменные правила (регулярные выражения, списки разрешений, бизнес-политика). Работает на входе или на выходе.

Три слоя последовательны по замыслу: входная модерация должна завершиться до генерации, а выходная модерация запускается после генерации. Параллелизм применяется внутри слоя — запуск нескольких классификаторов (например, OpenAI Moderation + Llama Guard + Perspective) одновременно на одном тексте скрывает задержку каждого классификатора. Как дополнительная оптимизация, во время завершения входной модерации и отсрочки токена-1 стриминга может отображаться заглушка («одну минуту, проверка...»). Поведение при срабатывании настраивается: отказ, фильтрация, эскалация на ручную проверку.

### Режимы сбоя

- **Только вход.** Не ловит галлюцинации на выходе (атаки кодированием из уроков 12–14 обходят входные классификаторы).
- **Только выход.** Позволяет любому входу достичь модели; увеличивает стоимость; раскрывает внутреннее рассуждение атакующему.
- **Только пользовательский.** Не устойчив для всех категорий; регулярные выражения хрупки.

Слоёный подход — это вариант по умолчанию. Пояс и подтяжки.

### Устаревание Azure

Azure Content Moderator: устарела в феврале 2024 года, прекращена в феврале 2027 года. Заменена Azure AI Content Safety, основанной на LLM и интегрированной с Azure OpenAI. Миграция — проект уровня полей на 2024–2027 годы для развёртываний Azure.

## Ключевые термины

| Термин | Что обычно говорят | Что это означает на самом деле |
|--------|-------------------|-------------------------------|
| OpenAI Moderation | «omni-moderation-latest» | Классификатор на базе GPT-4o с 13 категориями (текст) и частичной мультимодальной поддержкой |
| Perspective API | «токсичность Google Jigsaw» | Базовая линия оценки токсичности додовременной эпохи LLM |
| Llama Guard | «14 категорий MLCommons» | Классификатор вреда от Meta (v3: 8B текст, 8 языков; v4: 12B мультимодальная) |
| Входная модерация | «фильтр до генерации» | Классификатор на запросе пользователя до вызова модели |
| Выходная модерация | «фильтр после генерации» | Классификатор на выводе модели до доставки |
| Пользовательская модерация | «доменные правила» | Специфичные для развёртывания правила (регулярные выражения, список разрешений, политика) |
| Слоёная модерация | «все три слоя» | Стандартный паттерн производственного развёртывания |
