# Семейство оптимизации прямых предпочтений (Direct Preference Optimization)

> Рафайлов и др. (2023) показали, что оптимум RLHF имеет замкнутую форму в терминах данных о предпочтениях (preference), поэтому можно обойтись без явной модели вознаграждения (reward model) и оптимизировать политику напрямую. Это наблюдение породило целое семейство методов — IPO, KTO, SimPO, ORPO, BPO — каждый из которых устраняет определённый режим сбоя DPO. К 2026 году алгоритмы прямого выравнивания (direct alignment algorithms) используются в большинстве пограничных процессов постобучения вместо PPO. Однако кривая чрезмерной оптимизации из Урока 2 по-прежнему применима: алгоритмы прямого выравнивания не избегают эффекта Гудхарта (Goodhart's law), а лишь переносят точку его воздействия.

**Тип:** Теория
**Языки:** Python (стандартная библиотека, сравнительный инструмент для шести вариантов функций потерь предпочтений)
**Предварительные требования:** Фаза 18 · 01 (InstructGPT), Фаза 18 · 02 (Хакинг вознаграждений), Фаза 10 · 08 (Основы DPO)
**Время:** ~75 минут

## Цели обучения

- Вывести замкнутую форму DPO из оптимума RLHF с KL-регуляризацией.
- Сформулировать режим сбоя, который устраняет каждый из методов IPO, KTO, SimPO, ORPO, BPO в DPO.
- Различать «неявный разрыв вознаграждений» и «силу предпочтений» и объяснять, почему тождественное отображение в IPO имеет значение.
- Объяснить, почему Рафайлов и др. (NeurIPS 2024) доказывают, что алгоритмы прямого выравнивания чрезмерно оптимизируют, несмотря на отсутствие явной модели вознаграждения.

## Проблема

Целевая функция RLHF (Урок 1):

```
max_pi E_{x,y~pi} [ r(x, y) ] - beta * KL(pi || pi_ref)
```

имеет известный оптимум:

```
pi*(y|x) = (1/Z(x)) * pi_ref(y|x) * exp(r(x, y) / beta)
```

Таким образом, вознаграждение неявно определяется отношением оптимальной политики к справочной:

```
r(x, y) = beta * log(pi*(y|x) / pi_ref(y|x)) + beta * log Z(x)
```

Подставляя это выражение в правдоподобие предпочтений Брэдли-Терри (Bradley-Terry), получаем, что функция нормировки `Z(x)` сокращается, поскольку зависит только от `x`. В результате остаётся функция потерь, выраженная только в параметрах политики — модель вознаграждения не требуется. Это и есть DPO.

Нюанс: данный вывод предполагает, что оптимум достижим, данные о предпочтениях находятся в распределении, а справочная политика является истинным якорем моды. Ни одно из этих условий не выполняется точно. Каждый метод семейства устраняет нарушение различных допущений.

## Теория

### DPO (Rafailov et al., 2023)

```
L_DPO = -log sigmoid(
  beta * log(pi(y_w | x) / pi_ref(y_w | x))
  - beta * log(pi(y_l | x) / pi_ref(y_l | x))
)
```

Возможные ошибки:

- Неявный разрыв вознаграждений `beta * (log(pi/pi_ref)_w - log(pi/pi_ref)_l)` неограничен. Даже слабое предпочтение может привести к произвольно большому разрыву.
- Функция потерь толкает логарифмические вероятности выбранного и отвергнутого ответов в противоположных направлениях. Она может снижать абсолютную логарифмическую вероятность выбранного ответа при условии, что отвергнутый снижается быстрее. Это явление деградации выбранного ответа (Degraded Chosen Response).
- Предпочтения, выходящие за пределы распределения (редкая пара против редкой пары), порождают произвольные неявные вознаграждения.

### IPO (Azar et al., 2024)

Оптимизация тождественных предпочтений (Identity Preference Optimization) заменяет логарифм-сигмоиду тождественным отображением на вероятности предпочтения. Функция потерь становится квадратичной ошибкой по ограниченному целевому значению:

```
L_IPO = (log(pi(y_w | x) / pi_ref(y_w | x)) - log(pi(y_l | x) / pi_ref(y_l | x)) - 1/(2 beta))^2
```

Отступ ограничен значением `1/(2 beta)`. Сила предпочтений и разрыв неявного вознаграждения пропорциональны. Взрывного роста не происходит.

### KTO (Ethayarajh et al., 2024)

Оптимизация Канемана-Тверски (Kahneman-Tversky Optimization) полностью отказывается от попарной структуры. При наличии единичной размеченной выходной последовательности и бинарного сигнала «желательно» или «нежелательно» метод преобразует данные в полезность в рамках теории перспектив:

```
v(x, y) = sigma(beta * log(pi(y|x) / pi_ref(y|x)) - z_ref)
```

с разными весами для выигрышей и потерь (неприятие потерь). Преимущество: можно использовать непарные данные, которых значительно больше.

### SimPO (Meng et al., 2024)

Простая оптимизация предпочтений (Simple Preference Optimization) согласует обучающий сигнал с процессом генерации. Полностью удаляет справочную политику и нормирует логарифмическое правдоподобие по длине:

```
L_SimPO = -log sigmoid(
  (beta / |y_w|) * log pi(y_w | x)
  - (beta / |y_l|) * log pi(y_l | x)
  - gamma
)
```

с отступом `gamma` для стабилизации. Нормализация по длине устраняет стимул к эксплуатации ошибки DPO, связанной с предвзятостью по длине (более длинный `y_w` априори даёт больший разрыв логарифмических вероятностей).

### ORPO (Hong et al., 2024)

Оптимизация предпочтений по отношению шансов (Odds-Ratio Preference Optimization) добавляет член предпочтений к стандартной функции отрицательного логарифмического правдоподобия SFT:

```
L_ORPO = L_NLL(y_w) + lambda * L_OR
L_OR = -log sigmoid(log(odds(y_w) / odds(y_l)))
```

Справочная политика не требуется — член SFT выполняет роль регуляризатора. Обучение проводится на одном этапе от базовой модели до выровненной. Отдельный чекпоинт SFT не нужен.

### BPO (заявка на ICLR 2026, OpenReview id=b97EwMUWu7)

Метод выявляет проблему деградации выбранного ответа: DPO сохраняет ранжирование `y_w > y_l`, однако абсолютная логарифмическая вероятность `y_w` может снижаться. BPO добавляет однострочную коррекцию, штрафующую снижение абсолютной логарифмической вероятности выбранного ответа. По результатам, точность на Llama-3.1-8B-Instruct в задачах математического рассуждения повышается на +10,1% по сравнению с DPO.

### Универсальный результат: алгоритмы прямого выравнивания по-прежнему чрезмерно оптимизируют

Рафайлов и др. «Законы масштабирования для чрезмерной оптимизации модели вознаграждения в алгоритмах прямого выравнивания» (NeurIPS 2024) обучали политики с DPO, IPO, SLiC на нескольких наборах данных в рамках различных бюджетов KL. Кривые «золотое вознаграждение — KL» имеют ту же форму «пик и обрушение», что и в работе Гао и др. Неявное вознаграждение запрашивает выборки, выходящие за пределы распределения, во время обучения; KL-регуляризация не стабилизирует этот процесс.

Алгоритмы прямого выравнивания не избегают эффекта Гудхарта. Они лишь изменяют поверхность его воздействия с «чрезмерная оптимизация модели вознаграждения» на «чрезмерная оптимизация отношения к справочной политике». Универсальное решение — лучшие данные, ансамбли, ранняя остановка — применимо к обоим случаям.

### Выбор метода (2026)

- Если имеются большие парные данные о предпочтениях: DPO с консервативным beta; SimPO, если наблюдается предвзятость по длине.
- Если имеется непарная бинарная обратная связь: KTO.
- Если необходим одноэтапный конвейер от базовой модели: ORPO.
- Если в логах DPO наблюдается деградация логарифмических вероятностей выбранного ответа: BPO.
- Если сила предпочтений сильно варьируется и DPO насыщается: IPO.

Каждая лаборатория запускает все пять методов и выбирает лучший для каждой задачи. Нет оснований считать, что оптимум одинаков для математических рассуждений и обеспечения безопасности.

## Практическое применение

`code/main.py` сравнивает шесть функций потерь (DPO, IPO, KTO, SimPO, ORPO, BPO) на модельном наборе данных о предпочтениях, где истинная сила предпочтений варьируется в зависимости от пары. Каждая функция потерь оптимизируется на одном и том же образце из 500 пар с использованием небольшой политики softmax. Строит графики конечного процента побед, дрейфа логарифмических вероятностей выбранного ответа и разброса неявного вознаграждения для каждого метода.

## Результат

Данный урок формирует `outputs/skill-preference-loss-selector.md`. На основе статистики набора данных (парные или непарные, переменная или однородная сила предпочтений, распределение длины) и целевого конвейера (одноэтапный или SFT-затем-предпочтения) система рекомендует функцию потерь предпочтений и описывает режим сбоя, от которого она защищает.

## Упражнения

1. Запустите `code/main.py`. Зафиксируйте итоговое снижение логарифмической вероятности выбранного ответа для DPO и BPO. BPO должен сохранять более высокую абсолютную вероятность выбранного ответа — проверьте это.

2. Измените данные о предпочтениях так, чтобы все пары имели одинаковую силу. Какой из шёсти методов наиболее устойчив? Какой деградирует? Объясните преимущество IPO в данном случае.

3. Сделайте отвергнутые ответы в среднем в 2 раза длиннее выбранных. Не изменяя ничего другого, продемонстрируйте численно эксплуатацию длины в DPO и исправление в SimPO.

4. Рафайлов и др. (NeurIPS 2024) утверждают, что алгоритмы прямого выравнивания чрезмерно оптимизируют. Воспроизведите упрощённый вариант: постройте график KL-расхождения (выбранное минус отвергнутое) и наблюдайте чрезмерную оптимизацию в DPO при большом beta.

5. Прочитайте аннотацию статьи о BPO (OpenReview b97EwMUWu7). Запишите однострочную коррекцию, которую BPO добавляет к DPO. Сверьте с реализацией в `code/main.py`.

## Ключевые термины

| Термин | Что говорят | Что это означает на самом деле |
|--------|-------------|-------------------------------|
| DPO | «RLHF без модели вознаграждения» | Функция потерь, выведенная из замкнутой формы оптимума RLHF; только параметры политики |
| Неявное вознаграждение | «логарифмическое отношение» | `beta * log(pi(y|x) / pi_ref(y|x))` — вознаграждение, подразумеваемое DPO |
| IPO | «ограниченный DPO» | Заменяет логарифм-сигмоиду тождественным отображением; разрыв неявного вознаграждения ограничен `1/(2 beta)` |
| KTO | «непарный DPO» | Полезность в рамках теории перспектив по единичным меткам с неприятием потерь |
| SimPO | «DPO без справочной политики» | Нормированная по длине логарифмическая вероятность + отступ; без справочной политики |
| ORPO | «одноэтапный DPO» | Отрицательное логарифмическое правдоподобие + член предпочтений по отношению шансов; обучение от базовой модели за один проход |
| BPO | «DPO с сохранением выбранного» | DPO плюс штраф за снижение абсолютной логарифмической вероятности выбранного ответа |
| Деградация выбранного | «выбранное снижается» | DPO снижает логарифмическую вероятность выбранного ответа при условии, что отвергнутый снижается быстрее |
| DAA | «алгоритм прямого выравнивания» | Любой метод на основе функции потерь предпочтений, не использующий явную модель вознаграждения |
