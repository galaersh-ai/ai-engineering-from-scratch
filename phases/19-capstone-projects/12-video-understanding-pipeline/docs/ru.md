# Выпускной проект 12 — Пайплайн (pipeline) понимания видео (сцены, QA, поиск)

> Twelve Labs выпустила коммерческий продукт на базе Marengo + Pegasus. VideoDB представила API для CRUD-операций с видео. Molmo 2 от AI2 опубликовала открытые контрольные точки VLM. Gemini с длинным контекстом обрабатывает часы видео нативно. TimeLens-100K определила задачу темпоральной локализации (temporal grounding) в промышленном масштабе. Пайплайн (pipeline) 2026 года стабилизировался: сегментация сцен, подпись и эмбеддинг для каждой сцены, выравнивание транскрипта, мультивекторный (multimodal) индекс и запрос, отвечающий с таймкодами (start, end) и превью кадров. Выпускной проект (capstone) предполагает обработку 100 часов видео, проверку по публичным бенчмаркам и измерение количества галлюцинаций (hallucination) на вопросах подсчёта и действий.

**Тип:** Выпускной проект
**Языки:** Python (пайплайн), TypeScript (интерфейс)
**Предварительные требования:** Фаза 4 (компьютерное зрение), Фаза 6 (речь), Фаза 7 (трансформеры), Фаза 11 (инженерия LLM), Фаза 12 (мультимодальность), Фаза 17 (инфраструктура)
**Задействованные фазы:** P4 · P6 · P7 · P11 · P12 · P17
**Время:** 30 часов

## Постановка задачи

Долгосрочное видео-QA — самая ресурсоёмкая мультимодальная задача в масштабе 2026 года. Gemini 2.5 Pro способен нативно обработать двухчасовое видео, однако загрузка 100 часов видео в запросо-доступный корпус по-прежнему требует индекса на уровне сцен. Промышленная конфигурация сочетает сегментацию сцен (TransNetV2 или PySceneDetect), формирование подписей для каждой сцены с помощью VLM (Gemini 2.5, Qwen3-VL-Max или Molmo 2), выравнивание транскрипта (Whisper-v3-turbo с таймкодами на уровне слов) и мультивекторный индекс, хранящий подпись, эмбеддинг кадра и транскрипт параллельно. Запросо-пайплайн отвечает с таймкодами (start, end) и превью кадров.

Бенчмарки являются публичными (ActivityNet-QA, NeXT-GQA), помимо этого формируется собственный набор из 100 запросов. Галлюцинации на вопросах подсчёта и классификации действий — известная проблемная категория; выпускной проект измеряет её отдельно.

## Концепция

При загрузке параллельно работают три пайплайна. **Сегментация сцен** разбивает видео на сцены. **VLM-подписывание** генерирует подпись для каждой сцены и эмбеддинг кадра из ключевого кадра (keyframe). **ASR-выравнивание** создаёт таймкоды на уровне слов. Три потока объединяются по полям (scene_id, time range). Каждая сцена получает три типа векторов в мультивекторном индексе (Qdrant): эмбеддинг подписи, эмбеддинг ключевого кадра и эмбеддинг транскрипта.

В момент запроса естественноязыковой вопрос обрабатывается по всем трём векторам; результаты объединяются методом RRF (Reciprocal Rank Fusion); адаптер темпоральной локализации (в стиле TimeLens) уточняет окно (start, end) внутри наиболее релевантной сцены. VLM-синтезатор (Gemini 2.5 Pro или Qwen3-VL-Max) принимает запрос, наиболее релевантные сцены и обрезанные кадры и отвечает с указанием таймкодов и превью кадра.

Измерение галлюцинаций имеет принципиальное значение. Вопросы подсчёта («сколько человек заходит в комнату?») и классификации действий («повар ли сначала наливает, а потом помешивает?») отличаются ненадёжностью. Точность по таким вопросам следует сообщать отдельно от описательных вопросов.

## Архитектура

```
video file / URL
      |
      v
PySceneDetect / TransNetV2  (сегментация сцен)
      |
      +--- ключевой кадр сцены --- VLM-подпись + эмбеддинг кадра
      |                            (Gemini 2.5 Pro / Qwen3-VL-Max / Molmo 2)
      |
      +--- аудиодорожка --- ASR Whisper-v3-turbo + таймкоды слов
      |
      v
мультивекторный индекс Qdrant: {caption_emb, keyframe_emb, transcript_emb}
      |
запрос:
  плотные запросы по всем трём векторам -> объединение RRF -> top-k сцен
      |
      v
темпоральная локализация TimeLens / VideoITG (уточнение start/end внутри сцены)
      |
      v
VLM-синтез: запрос + наиболее релевантные сцены + превью кадров
      |
      v
ответ + таймкоды (start, end) + превью кадров + ссылки на источники
```

## Технологический стек

- Сегментация сцен: TransNetV2 (state-of-the-art 2024–26) или PySceneDetect
- ASR: Whisper-v3-turbo через faster-whisper с таймкодами на уровне слов
- VLM-подписывание и генерация ответов: Gemini 2.5 Pro, Qwen3-VL-Max или Molmo 2
- Темпоральная локализация: адаптер, обученный на TimeLens-100K, или VideoITG
- Индекс: Qdrant с поддержкой мультивекторного поиска (подпись / кадр / транскрипт)
- Интерфейс: Next.js 15 с HTML5-плеером и миниатюрами сцен
- Оценка: ActivityNet-QA, NeXT-GQA, собственный набор из 100 вопросов с ручной разметкой
- Бенчмарк галлюцинаций: подмножества подсчёта и классификации действий с ручной разметкой

## Сборка

1. **Ходок загрузки (Ingest walker).** Принимает URL YouTube или локальные MP4-файлы. При необходимости понижает разрешение до 720p. Сохраняет `{video_id, file_path}`.

2. **Сегментация сцен.** Запускает TransNetV2 или PySceneDetect для получения `[{scene_id, start_ms, end_ms, keyframe_path}]`. Целевой объём: 100 часов — примерно 6–8 тысяч сцен.

3. **Проход ASR.** Запускает Whisper-v3-turbo на аудиодорожке; экспортирует таймкоды на уровне слов; разбивает транскрипт на сегменты по сценам.

4. **VLM-подписывание.** Для каждой сцены вызывает Gemini 2.5 Pro (или Qwen3-VL-Max) с ключевым кадром и шаблоном подписи. Генерирует подпись и эмбеддинг кадра.

5. **Мультивекторный индекс.** Коллекция Qdrant с тремя именованными векторами. Полезная нагрузка (payload): `{video_id, scene_id, start_ms, end_ms, keyframe_url}`.

6. **Запрос.** Естественноязыковой вопрос инициирует три плотных поиска; результаты объединяются методом Reciprocal Rank Fusion; top-k = 5 сцен.

7. **Темпоральная локализация.** Запускает адаптер в стиле TimeLens для наиболее релевантной сцены с целью уточнения окна (start, end) внутри сцены.

8. **VLM-синтез.** Вызывает Gemini 2.5 Pro с запросом, фрагментами трёх наиболее релевантных сцен (в виде изображений или коротких клипов) и транскриптами. Требуются ссылки в формате `(video_id, start_ms, end_ms)`.

9. **Оценка.** Запускает ActivityNet-QA и NeXT-GQA. Формирует собственный набор из 100 запросов. Сообщает общую точность и разбивку по категориям (подсчёт, действия, описание).

## Сдача

`outputs/skill-video-qa.md` — готовый результат. При указании URL YouTube или загрузке видео пайплайн индексирует сцены и отвечает на вопросы с указанием таймкодов-источников.

| Вес | Критерий | Способ измерения |
|:-:|---|---|
| 25 | Темпоральная локализация (IoU) | Intersection-over-union на контрольном наборе для локализации |
| 20 | Точность QA | NeXT-GQA и собственный набор из 100 запросов |
| 20 | Пропускная способность загрузки | Часы видео на доллар затрат |
| 20 | Интерфейс и UX-цитирования | Ссылки на таймкоды, полоса миниатюров, переход к кадру |
| 15 | Уровень галлюцинаций | Точность по подсчёту и классификации действий отдельно |
| **100** | | |

## Упражнения

1. Замените Gemini 2.5 Pro на Qwen3-VL-Max на этапе подписывания. Сообщите разницу в качестве подписей по результатам оценки 50 сцен человеком.

2. Замените мультивекторное представление кадра для каждой сцены на один агрегированный вектор. Измерите снижение качества поиска.

3. Реализуйте режим «строгий подсчёт»: синтезатор извлекает каждый подсчитанный экземпляр с таймкодом, а пользователь проверяет его вручную. Измерьте, снижает ли ручная проверка количество галлюцинаций.

4. Оцените стоимость загрузки: часы видео на доллар для трёх вариантов VLM. Определите оптимальный баланс.

5. Добавьте транскрипт с разделением по говорящим (speaker diarization): выполните разделение речи с помощью pyannote и создайте эмбеддинги транскриптов для каждого говорящего. Продемонстрируйте запросы вида «что говорил Алисей о X?».

## Ключевые термины

| Термин | Как говорят | Что это значит на самом деле |
|------|-----------------|------------------------|
| Сегментация сцен | «Детектирование склейков» | Разбиение видео на сцены по границам монтажных переходов |
| Мультивекторный индекс | «Подпись + кадр + транскрипт» | Коллекция Qdrant с именованными векторами для каждого типа представления |
| Темпоральная локализация | «Когда именно это происходило» | Уточнение окна (start, end) для ответа на запрос |
| Эмбеддинг кадра | «Визуальное представление» | Векторное представление ключевого кадра; используется для визуального сходства сцен |
| RRF-слияние | «Reciprocal Rank Fusion» | Стратегия объединения нескольких ранжированных списков; классический приём гибридного поиска |
| Галлюцинация подсчёта | «Неверный подсчёт» | Известный тип ошибки VLM на вопросах «сколько X» |
| ActivityNet-QA | «Бенчмарк видео-QA» | Бенчмарк точности долгосрочного видео-QA |
