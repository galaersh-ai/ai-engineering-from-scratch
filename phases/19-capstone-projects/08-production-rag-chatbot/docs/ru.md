# Выпускной проект 08 — Продуктовый RAG-чатбот для регулируемой отрасли

> Harvey, Glean, Mendable и LlamaCloud используют одинаковую продуктовую архитектуру в 2026 году. Загрузка документов через docling или Unstructured и ColPali для визуального контента. Гибридный поиск. Реранжирование через bge-reranker-v2-gemma. Синтез через Claude Sonnet 4.7 с кешированием промптов при 60–80% попаданиях. Защита через Llama Guard 4 и NeMo Guardrails. Мониторинг через Langfuse и Phoenix. Оценка через RAGAS на золотом наборе из 200 вопросов. Соберите систему в регулируемой области (юриспруденция, клинические исследования, страхование), а выпускной проект (capstone) — это прохождение золотого набора, красной команды (red team) и мониторинга дрейфа.

**Тип:** Выпускной проект (capstone)
**Языки:** Python (пайплайн + API), TypeScript (чат-интерфейс)
**Предварительные требования:** Фаза 5 (NLP), Фаза 7 (трансформеры), Фаза 11 (инженерия LLM), Фаза 12 (мультимодальность), Фаза 17 (инфраструктура), Фаза 18 (безопасность)
**Задействованные фазы:** P5 · P7 · P11 · P12 · P17 · P18
**Время:** 30 часов

## Проблема

RAG (Retrieval-Augmented Generation) в регулируемых областях (юридические договоры, протоколы клинических исследований, страховые полисы) — это самая распространённая продуктовая архитектура 2026 года, потому что окупаемость очевидна, а ставки Concrete. Harvey (Allen & Overy) создал его для юридической сферы. Mendable выпускает версию для документации разработчиков. Glean обеспечивает корпоративный поиск. Шаблон таков: загрузка документов высокой точности, гибридная выборка с реранжированием, синтез с принудительным цитированием и кешированием промптов, защита многоуровневыми средствами безопасности и непрерывный мониторинг дрейфа.

Сложность заключается не в модели. Это jurisdictions-осведомлённое соответствие требованиям (HIPAA, GDPR, SOC2), проверяемость на уровне цитат, управление стоимостью (кеширование промптов даёт скидку 60–90% при высоком уровне попаданий), обнаружение галлюцинаций через RAGAS faithfulness и обнаружение дрейфа при обновлении исходных документов без обновления индекса. Этот выпускной проект требует реализовать всё это на золотом наборе из 200 вопросов с набором красной команды (red team).

## Концепция

Пайплайн состоит из двух частей. **Загрузка:** docling или Unstructured парсит структурированные документы; ColPali обрабатывает визуально насыщенные документы; чанки получают саммари, теги и метки доступа на основе ролей. Векторы хранятся в pgvector + pgvectorscale (до 50 млн векторов) или Qdrant Cloud; разреженный BM25 работает параллельно. **Диалог:** LangGraph управляет памятью и многотурностью; каждый запрос выполняет гибридную выборку, реранжирование через bge-reranker-v2-gemma-2b, синтез через Claude Sonnet 4.7 (с кешированием промптов), проверку вывода через Llama Guard 4 и NeMo Guardrails, а затем формирует ответ с цитатами.

Стек оценки имеет четыре уровня. **Золотой набор** (200 размеченных вопрос/ответ с цитатами) для корректности. **Красная команда (red team)** (обходы защиты, попытки извлечения PII, вопросы вне тематики) для безопасности. **RAGAS** для автоматической оценки по каждому обороту: faithfulness / answer relevance / context precision. **Панель мониторинга дрейфа** (Arize Phoenix) отслеживает качество выборки и показатель галлюцинаций еженедельно.

Кеширование промптов — это рычаг управления стоимостью. Claude 4.5+ и GPT-5+ поддерживают кеширование системных промптов и извлечённого контекста. При 60–80% попаданиях стоимость за запрос снижается в 3–5 раз. Пайплайн должен быть спроектирован для стабильных префиксов (системный промпт и реранжированный контекст идут первыми) для достижения высокого процента попаданий в кеш.

## Архитектура

```
документы (договоры, протоколы, полисы)
      |
      v
парсинг docling / Unstructured + ColPali для визуального контента
      |
      v
чанки + саммари + роли-метки + jurisdictions-теги
      |
      v
pgvector + pgvectorscale  +  BM25 (Tantivy)
      |
запрос + роль + jurisdiction
      |
      v
диалоговый агент на LangGraph
   +--- выборка (гибридная)
   +--- фильтрация по роли + jurisdiction
   +--- реранжирование (bge-reranker-v2-gemma-2b или Voyage rerank-2)
   +--- синтез (Claude Sonnet 4.7, кеширование промптов)
   +--- защита (Llama Guard 4 + NeMo Guardrails + Presidio удаление PII из вывода)
   +--- цитирование + возврат
      |
      v
оценка:
  RAGAS faithfulness / answer_relevance / context_precision (онлайн)
  очередь аннотаций Langfuse (выборочно)
  дрейф Arize Phoenix (еженедельно)
  набор красной команды (перед релизом)
```

## Технологический стек

- Загрузка: Unstructured.io или docling для структурированных документов; ColPali для визуально насыщенных PDF
- Векторная БД: pgvector + pgvectorscale до 50 млн векторов; иначе Qdrant Cloud
- Разреженный индекс: Tantivy BM25 с весами полей
- Оркестрация: LlamaIndex Workflows (загрузка) + LangGraph (диалог)
- Реранжирование: bge-reranker-v2-gemma-2b (самостоятельный хостинг) или Voyage rerank-2 (хостинг)
- LLM: Claude Sonnet 4.7 с кешированием промптов; запасной вариант Llama 3.3 70B (самостоятельный хостинг)
- Оценка: RAGAS 0.2 (онлайн), DeepEval для наборов по галлюцинациям и обходам защиты
- Наблюдаемость: Langfuse (самостоятельный хостинг) с очередью аннотаций; Arize Phoenix для дрейфа
- Защита: Llama Guard 4 (классификатор ввода/вывода), NeMo Guardrails v0.12 (политики), Presidio удаление PII
- Соответствие требованиям: метки доступа на основе ролей для чанков; jurisdictions-теги для GDPR/HIPAA

## Реализация

1. **Загрузка.** Парсинг корпуса (1000–10000 документов для серьёзной сборки) через Unstructured или docling. Для отсканированных / визуально насыщенных страниц — обработка через ColPali. Формирование чанков с саммари, метками ролей и jurisdictions-тегами.

2. **Индексация.** Плотные эмбеддинги (Voyage-3 или Nomic-embed-v2) в pgvector + pgvectorscale. Разреженный индекс BM25 через Tantivy. Фильтры ролей и jurisdictions в метаданных.

3. **Гибридная выборка.** Сначала фильтрация по роли и jurisdiction; затем параллельная плотная + BM25 выборка; объединение через reciprocal rank fusion; топ-20 на реранжирование; топ-5 на синтез.

4. **Синтез с кешированием промптов.** Системный промпт + статические политики в заголовке кеша; реранжированный контекст как расширение кеша; вопрос пользователя как некешируемый суффикс. Целевой показатель — 60–80% попаданий в кеш в стационарном режиме.

5. **Защита.** Llama Guard 4 на входе; NeMo Guardrails блокирует вопросы вне тематики или темы, запрещённые политикой; Presidio удаляет случайно попавшие PII из вывода; постфильтр для принудительного цитирования.

6. **Золотой набор.** 200 пар вопрос/ответ, размеченных экспертом в области, с (ответом, цитатами). Оценка агента по точности цитирования, корректности ответа, faithfulness (RAGAS).

7. **Красная команда (red team).** 50 враждебных промптов: обходы защиты (PAIR, TAP), попытки извлечения PII, вопросы вне тематики, межjurisdiction-утечки. Оценка по прохождению/непрохождению и тяжести.

8. **Панель мониторинга дрейфа.** Arize Phoenix отслеживает качество выборки (nDCG, faithfulness цитат) еженедельно. Оповещение при снижении на 5%.

9. **Отчёт по стоимости.** Langfuse: процент попаданий в кеш промптов, токены на запрос, разбивка стоимости/запрос по этапам.

## Упражнения

1. Создайте второй фрагмент корпуса в другой jurisdiction (например, HIPAA наряду с GDPR). Продемонстрируйте, как фильтрация по роли и jurisdiction предотвращает межjurisdiction-утечки на наборе из 20 вопросов.

2. Измерьте процент попаданий в кеш промптов за неделю реального трафика. Определите, какие запросы ломают кеш-префикс. Перестройте систему.

3. Добавьте многотурную память с буфером-саммари на 10 000 токенов. Измерьте, снижается ли faithfulness по мере роста диалога.

4. Замените Claude Sonnet 4.7 на Llama 3.3 70B (самостоятельный хостинг). Измерьте стоимость за запрос и разницу в faithfulness.

5. Добавьте режим «неуверенность»: если реранжированные оценки ниже порога, агент отвечает «У меня нет надёжных цитат» вместо формирования ответа. Измерите снижение ложной уверенности.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|-----------------|------------------------|
| Кеширование промптов (Prompt caching) | «Кешированный системный + контекстуальный промпт» | Функция Claude/OpenAI: кешируемые префиксные токены со скидкой 60–90% при попадании |
| RAGAS | «Оценщик RAG» | Автоматическая оценка faithfulness, answer relevance, context precision |
| Золотой набор (Golden set) | «Размеченный набор для оценки» | 200+ размеченных экспертом вопрос/ответ с цитатами; эталон |
| Jurisdiction-тег (Jurisdiction tag) | «Метка соответствия» | Область действия GDPR/HIPAA/SOC2, прикреплённая к чанкам; обеспечивается фильтром выборки |
| Faithfulness цитат (Citation faithfulness) | «Доля обоснованных ответов» | Доля утверждений, подтверждённых воспроизводимыми отрывками из источников |
| Дрейф (Drift) | «Снижение качества выборки» | Еженедельное изменение nDCG или оценки цитат; порог оповещения — 5% |
| Красная команда (Red team) | «Агрессивная оценка» | Тестирование перед релизом: обходы защиты, извлечение PII, вопросы вне тематики |
