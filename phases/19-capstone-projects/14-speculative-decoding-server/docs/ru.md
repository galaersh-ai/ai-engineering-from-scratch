# Выпускной проект 14 — Сервер инференса (inference) со спекулятивным декодингом (speculative decoding)

> EAGLE-3 в vLLM 0.7 обеспечивает прирост пропускной способности в 2,5–3 раза на реальном трафике. P-EAGLE (AWS 2026) продвинул параллельную спекуляцию ещё дальше. SGLang SpecForge обучает черновые головы в промышленном масштабе. Хаб Red Hat Speculators публикует выровненные черновики для распространённых открытых моделей. TensorRT-LLM сделал спекулятивный декодинг базовой функцией на NVIDIA. Технологический стек 2026 года для обслуживания моделей — это vLLM или SGLang с черновиками семейства EAGLE, квантизацией FP8 или INT4 и горизонтальным масштабированием по метрике времени ожидания очереди. Данный выпускной проект (capstone) посвящён обслуживанию двух открытых моделей с пропускной способностью 2,5× и выше от базового уровня при полном отчёте о латентности хвоста распределения.

**Тип:** Выпускной проект  
**Языки:** Python (обслуживание), C++ / CUDA (инспекция ядер), YAML (конфигурации)  
**Предварительные требования:** Фаза 3 (глубокое обучение), Фаза 7 (трансформеры), Фаза 10 (LLM с нуля), Фаза 17 (инфраструктура)  
**Задействованные фазы:** Ф3 · Ф7 · Ф10 · Ф17  
**Время:** 30 часов

## Проблема

Спекулятивный декодинг стал стандартным решением в 2026 году. Черновые головы EAGLE-3 обучаются на скрытых состояниях целевой модели и предсказывают N токенов вперёд; целевая модель проверяет их за один проход. Коэффициенты принятия (acceptance rate) от 60 до 80% обеспечивают конечную пропускную способность в 2–3 раза выше исходной. vLLM 0.7 интегрирует это нативно. SGLang + SpecForge предоставляет пайплайн обучения. Red Hat Speculators публикует выровненные черновики для Llama 3.3 70B, Qwen3-Coder-30B MoE, GPT-OSS-120B.

Искусство состоит в операциях обслуживания, а не в модели. Коэффициент принятия меняется в зависимости от распределения трафика (ShareGPT против кода против доменных данных). Латентность хвоста распределения при отклонении черновика хуже, чем без спекуляции — необходимо отчитываться о p99 при нескольких размерах батча, а не только о стационарной пропускной способности в токенах/сек. Стоимость за 1 млн токенов по сравнению с API Anthropic / OpenAI — это ключевой показатель достоверности.

## Концепция

Спекулятивный декодинг состоит из двух уровней. **Черновая** модель (голова EAGLE-3, ngram или меньшая модель, выровненная с целевой) предлагает k кандидатных токенов на каждом шаге. **Целевая** модель проверяет все k за один проход; любой принятый префикс заменяет жадный путь. Коэффициент принятия зависит от выравнивания черновика и целевой модели, а также от распределения входных данных.

EAGLE-3 превосходит ngram-черновики на большинстве типов трафика. P-EAGLE выполняет параллельную спекуляцию для более глубоких деревьев черновиков. Компромисс: p99 латентность при отклонении выше, поскольку проход проверки больше. Конфигурация обслуживания должна отчитываться о латентности по размерам батчей, чтобы это выявить.

Развёртывание осуществляется в Kubernetes. vLLM 0.7 запускает по одному реплике на GPU или тензорно-параллельный шард. HPA масштабирует автоматически по времени ожидания очереди, а не по загрузке CPU. Квантизация FP8 (Marlin) и INT4 (AWQ) удерживает потребление памяти GPU в пределах H100 / H200. Полный отчёт включает пропускную способность, коэффициент принятия, p50/p99 при батчах 1/8/32 и стоимость в $/1M токенов.

## Технологический стек

- Обслуживание: vLLM 0.7 или SGLang 0.4
- Спекулятивные методы: черновые головы EAGLE-3, параллельная спекуляция P-EAGLE, ngram-запасной вариант
- Обучение черновиков: SpecForge (SGLang) или Red Hat Speculators
- Целевые модели: Llama 3.3 70B, Qwen3-Coder-30B MoE, GPT-OSS-120B
- Квантизация: FP8 (Marlin), INT4 AWQ
- Развёртывание: Kubernetes + NVIDIA device plugin; HPA по метрике времени ожидания очереди
- Оценка: ShareGPT, MT-Bench-v2, GSM8K, HumanEval для измерения коэффициента принятия по доменам
- Эталон: TensorRT-LLM спекулятивный декодинг как базовый вариант от поставщика

## Сборка

1. **Подготовка целевой модели.** Выберите Llama 3.3 70B. Квантизируйте до FP8 через Marlin. Развёртывайте на vLLM 0.7 на 1×H100 (или 2× в тензорно-параллельном режиме).

2. **Источник черновика.** Загрузите выровненную черновую голову EAGLE-3 из Red Hat Speculators (или обучите через SpecForge). Загрузите её в конфигурацию спекулятивного декодинга vLLM.

3. **Базовые показатели.** До спекуляции: токенов/с при батчах 1/8/32, p50/p99 латентность, загрузка GPU. Опубликуйте результаты.

4. **Включение EAGLE-3.** Переключите конфигурацию; повторите тот же бенчмарк. Отчитайтесь о приросте скорости, коэффициенте принятия, разнице p99 хвостовой латентности.

5. **P-EAGLE.** Включите параллельную спекуляцию; измерьте более глубокое дерево черновиков по сравнению с последовательным EAGLE-3. Определите точку, где P-EAGLE помогает, а где вредит.

6. **Доменный трафик.** Пропустите через тот же сервер ShareGPT, HumanEval и доменный трафик. Измерьте коэффициент принятия для каждого распределения. Определите, когда черновики начинают отклоняться.

7. **Вторая целевая модель.** Запустите тот же пайплайн на Qwen3-Coder-30B MoE. Черновик сложнее (шум маршрутизации MoE). Отчитайтесь.

8. **K8s HPA.** Развёртывание в K8s с HPA, отслеживающим `queue_wait_ms`. Продемонстрируйте горизонтальное масштабирование при утроенной нагрузке.

9. **Сравнение стоимости.** Рассчитайте стоимость в $/1M токенов по сравнению с Anthropic Claude Sonnet 4.7 и OpenAI GPT-5.4 на том же наборе задач оценки. Опубликуйте результаты.

## Упражнения

1. Измерьте деградацию коэффициента принятия, когда черновая модель отстаёт от целевой на одну версию (например, сдвиг от Llama 3.3 до 3.4). Создайте мониторинговое оповещение.

2. Реализуйте ngram-запасной вариант: если коэффициент принятия EAGLE-3 падает ниже порога, переключайтесь на ngram-черновики. Отчитайтесь об улучшении надёжности.

3. Проведите контролируемый эксперимент с MoE: один и тот же Qwen3-Coder-30B с внесённым шумом маршрутизации и без него. Измерьте чувствительность принятия черновика.

4. Расширьте до H200 (141 ГБ). Отчитайтесь о получении запаса по размеру модели на реплику и о возможности обслуживания неквантизированного Llama 3.3 70B.

5. Проведите бенчмарк спекулятивного декодинга TensorRT-LLM на том же оборудовании H100. Отчитайтесь о случаях превосходства над vLLM.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|-----------------|------------------------|
| Черновая модель (Draft model) | «Спекулятор» | Малая модель, предлагающая N токенов для проверки целевой моделью |
| EAGLE-3 | «Черновая архитектура 2026 года» | Черновая голова, обученная на скрытых состояниях целевой модели; ~75% принятия |
| P-EAGLE | «Параллельная спекуляция» | Дерево черновых ветвей, проверяемое за один проход целевой модели |
| Коэффициент принятия (Acceptance rate) | «Частота попаданий» | Доля принятых черновых токенов без повторного сэмплирования |
| Квантизация (Quantization) | «FP8 / INT4» | Веса с пониженной точностью для размещения большей части модели в памяти GPU |
| Время ожидания очереди (Queue wait) | «Метрика HPA» | Время ожидания запроса в очереди до начала инференса |
| Хаб Speculators | «Выровненные черновики» | Хаб Red Hat Neural Magic с черновиками EAGLE для распространённых открытых моделей |
