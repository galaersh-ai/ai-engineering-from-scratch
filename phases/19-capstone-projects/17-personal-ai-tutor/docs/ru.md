# Выпускной проект 17 — Персональный ИИ-репетитор (Personal AI Tutor) (адаптивный, мультимодальный, с памятью)

> Khanmigo (Khan Academy), Duolingo Max, Google LearnLM / Gemini for Education, Quizlet Q-Chat и Synthesis Tutor выпустили масштабируемые адаптивные мультимодальные системы репетиторства в 2026 году. Общая архитектура включает сократическую политику (никогда не выдавать ответ напрямую), модель учащегося, обновляющуюся после каждого взаимодействия (в духе байесовского отслеживания знаний), ввод голосом + текстом + распознавание математических уравнений по фотографии, извлечение из графа учебной программы, планирование с интервальным повторением и строгие фильтры безопасности для возрасто-соответствующего контента. Выпускной проект (capstone) — создание предметного репетиторора (K-12 алгебра или вводный Python), проведение двухнедельного исследования эффективности с 10 учащимися и прохождение аудита контентной безопасности.

**Тип:** Выпускной проект
**Языки:** Python (бэкенд, модель учащегося), TypeScript (веб-приложение), SQL (граф учебной программы через Postgres + Neo4j)
**Предварительные требования:** Фаза 5 (NLP), Фаза 6 (речь), Фаза 11 (инженерия LLM), Фаза 12 (мультимодальность), Фаза 14 (агенты), Фаза 17 (инфраструктура), Фаза 18 (безопасность)
**Задействованные фазы:** P5 · P6 · P11 · P12 · P14 · P17 · P18
**Время:** 30 часов

## Проблема

Адаптивное репетиторство когда-то было нишевым направлением ed-tech-исследований. К 2026 году это массовый потребительский продукт. Khanmigo развёрнут в большинстве школьных округов США. Duolingo Max достиг десятков миллионов активных пользователей в месяц. LearnLM / Gemini for Education от Google обеспечивает репетиторство в Google Classroom. Quizlet Q-Chat работает рядом с карточками. Synthesis Tutor стал вирусным, предлагая репетитора для любознательных детей. Общие элементы: мультимодальный ввод (набор текста, голос, фотографии уравнений), сократическая педагогика (сначала вопросы, потом объяснения), модель учащегося, обновляющаяся после каждого взаимодействия, и строгая возрасто-соответствующая безопасность.

Вы создадите одну такую систему для конкретной группы учащихся. Мера качества — реальное исследование эффективности: результаты предварительного и итогового тестирования за две недели с 10 учащимися. Голосовой интерфейс должен быть естественным (стек из выпускного проекта 03). Память должна уважать конфиденциальность. Фильтр безопасности должен пройти red-team проверку с учётом COPPA для K-12.

## Концепция

Четыре компонента. **Политика репетитора (Tutor policy)** — это сократический цикл: когда учащийся просит ответ, политика задаёт наводящий вопрос; когда учащийся отвечает правильно, переход к следующему понятию; когда учащийся застрял, предлагается пошаговая подсказка. **Модель учащегося (Learner model)** — байесовское отслеживание знаний (или простой вариант), обновляющее вероятность освоения для каждого узла учебной программы после каждого взаимодействия. **Граф учебной программы (Curriculum graph)** — это граф в Neo4j с узлами понятий и рёбрами предварительных требований; политика перемещается по графу для выбора следующего понятия. **Память (Memory)** — эпизодическое + семантическое хранилище (в стиле agentmemory), содержащее прошлые взаимодействия, ошибки и предпочтения.

Интерфейс — мультимодальный. Текстовый ввод для набора ответов. Голосовой ввод через LiveKit + Whisper (используем стек из выпускного проекта 03). Фото-ввод для математических задач через dots.ocr или PaliGemma 2. Голосовой вывод через Cartesia Sonic-2. Безопасность обеспечивается Llama Guard 4 плюс фильтр возрасто-соответствующего контента (блокирует контент для взрослых, насилие, самоповреждение) и политика хранения данных с учётом COPPA.

Исследование эффективности — это конечный продукт. 10 учащихся, предварительный и итоговый тест, две недели. Отчёт о приросте обученности и доверительном интервале. Сравнение с неадаптивным базовым вариантом (тот же контент, подаваемый линейно без политики репетитора).

## Архитектура

```
устройство учащегося
  |
  +-- текст         -> веб-приложение
  +-- голос         -> LiveKit Agents (ASR + TTS)
  +-- фото математ. -> dots.ocr / PaliGemma 2
       |
       v
  политика репетитора (LangGraph)
       - сократическое решающее ядро
       - выбор следующего понятия (обход графа учебной программы)
       - пошаговые подсказки
       - обновление освоения
       |
       v
  модель учащегося (BKT / теория ответа на задание)
       - вероятность освоения по понятиям
       - планировщик интервального повторения (SM-2 или FSRS)
       |
       v
  память (в стиле agentmemory)
       - эпизодическая: каждое взаимодействие
       - семантическая: усвоенные ошибки, предпочтения
       - политика хранения: с учётом COPPA / GDPR
       |
       v
  граф учебной программы (Neo4j)
       - рёбра предварительных требований
       - OER-контент, привязанный к узлам
       |
       v
  безопасность:
    Llama Guard 4 + фильтр возрасто-соответствующего контента
    доступ к памяти ограничен идентификатором учащегося
```

## Технологический стек

- Предмет: K-12 алгебра или вводный Python (выберите один для глубины)
- Политика репетитора: LangGraph поверх Claude Sonnet 4.7 (с кэшированием промптов)
- Модель учащегося: байесовское отслеживание знаний (классическое) или FSRS для интервального повторения
- Граф учебной программы: Neo4j с понятиями + рёбрами предварительных требований + OER-контент
- Память: постоянное векторное + эпизодическое + семантическое хранилище в стиле agentmemory
- Голос: LiveKit Agents 1.0 + Cartesia Sonic-2 (используем стек из выпускного проекта 03)
- Фото математики: dots.ocr или PaliGemma 2 для распознавания уравнений
- Безопасность: Llama Guard 4 + пользовательский фильтр возрасто-соответствующего контента
- Оценка: генерация вопросов по уровням Блума, инструменты для предварительного/итогового тестирования и исследования эффективности

## Реализация

1. **Граф учебной программы.** Создайте граф в Neo4j из 50–150 узлов понятий (например, K-12 алгебра от «числовая ось» до «формулы корней квадратного уравнения») с рёбрами предварительных требований. Привяжите OER-контент к каждому узлу (Open Textbook, OpenStax).

2. **Модель учащегося.** Инициализируйте байесовское отслеживание знаний с априорными вероятностями: угадывание, пропуск, скорость обучения. Обновляйте освоение по понятиям после каждого взаимодействия. Сохраняйте данные для каждого учащегося.

3. **Политика репетитора.** LangGraph с узлами: `read_signal` (был ли ответ учащегося правильным / частичным / учащийся застрял?), `select_concept` (обход графа учебной программы для выбора наивысшеприоритетного понятия), `scaffold` (сократический промпт), `update_mastery`.

4. **Память.** Каждое взаимодействие записывается в эпизодическое хранилище. Ошибки и предпочтения пополняют семантическую память. Политика хранения с учётом COPPA: автоматическое удаление через 1 год, доступ для родителей.

5. **Голосовой путь.** Worker LiveKit Agents, подключённый к политике репетитора. ASR через Whisper-v3-turbo. TTS через Cartesia Sonic-2. Поддержка перебивания (barge-in) (используем механики из выпускного проекта 03).

6. **Путь фото-математики.** Загрузка или снятие изображения; запуск dots.ocr или PaliGemma 2 для распознавания уравнения; передача в репетитор в виде структурированного ввода.

7. **Безопасность.** Каждый вывод модели проходит через Llama Guard 4 + фильтр возрасто-соответствующего контента (блокирует самоповреждения, контент для взрослых, насилие). Доступ к памяти ограничен идентификатором учащегося; интерфейс родительского доступа для удаления данных.

8. **Исследование эффективности.** 10 учащихся, предварительный тест (стандартизированный базовый тест из 30 вопросов), две недели взаимодействий с репетитором (3 сессии в неделю), итоговый тест. Сравнение с неадаптивной контрольной группой из 10 учащихся на том же контенте.

9. **Еженедельные отчёты о прогрессе.** Автоматическая генерация PDF-сводки для каждого учащегося: пройденные темы, динамика освоения и рекомендации по дальнейшим шагам.

## Демонстрация

```
учащийся: «Я не понимаю, почему 3x + 6 = 12 означает x = 2»
[сигнал]   застрял
[понятие]  'выделение переменной' (предварительное требование: сложение-вычитание-равенство)
[подсказка] «какое число вы вычтете из обеих сторон, чтобы начать?»
учащийся: «6»
[сигнал]   правильный ответ
[освоение] сложение-вычитание-равенство: 0.62 -> 0.77
[понятие]  продолжить 'выделение переменной'
[подсказка] «отлично. чему равно 3x / 3?»
```

## Упражнения

1. Проведите исследование эффективности с адаптивной моделью учащегося и без неё (случайный порядок понятий). Сравните результаты. Ожидается, что адаптивный подход покажет лучшие результаты, но величина разницы — это и есть интересный показатель.

2. Добавьте мультимодальный тест: одно и то же понятие задаётся в виде текста, голоса и фотографии. Измерьте, быстрее ли учащиеся достигают консенсуса с использованием предпочтительного модуса.

3. Создайте панель родителя: пройденные темы, динамика освоения, предстоящие понятия, события безопасности (все срабатывания фильтров). Соответствие COPPA.

4. Добавьте режим переключения языка: репетитор принимает ввод на испанском и преподаёт на испанском. Измерьте покрытие X-Guard.

5. Нагрузите конфиденциальность памяти: убедитесь, что учащийся A не может увидеть данные учащегося B даже через атаку повторного ввода аудиофрагмента. Фиксируйте попытки доступа и отправляйте оповещение.

## Ключевые термины

| Термин | Как говорят | Что это означает |
|------|-----------------|------------------------|
| Сократическая политика | «Спрашивай, а не выдавай ответ» | Репетитор задаёт наводящий вопрос вместо того, чтобы дать ответ |
| Байесовское отслеживание знаний | «BKT» | Классические уравнения модели учащегося для вероятности освоения по понятиям |
| FSRS | «Free Spaced Repetition Scheduler» | Планировщик интервального повторения 2024 года, лучше SM-2 |
| Граф учебной программы | «DAG понятий» | Neo4j с узлами понятий и рёбрами предварительных требований |
| Эпизодическая память | «Журнал взаимодействий» | Каждое взаимодействие сохраняется для последующего извлечения |
| Семантическая память | «Хранилище усвоенных шаблонов» | Уплотнённые ошибки и предпочтения, продвинутые из эпизодической памяти |
| COPPA | «Закон о конфиденциальности детей» | Закон США, ограничивающий сбор данных о детях младше 13 лет |
