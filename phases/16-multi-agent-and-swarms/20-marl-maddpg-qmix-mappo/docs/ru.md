# MARL — MADDPG, QMIX, MAPPO

> Наследие обучения с подкреплением в мультиагентной координации, которое по-прежнему информирует LLM-агентные системы в 2026 году. **MADDPG** (Lowe et al., NeurIPS 2017, arXiv:1706.02275) ввёл централизованное обучение, децентрализованное выполнение (CTDE): каждый критик видит состояния и действия всех агентов во время обучения; во время тестирования работают только локальные акторы. Работает для кооперативных, конкурентных и смешанных условий. **QMIX** (Rashid et al., ICML 2018, arXiv:1803.11485) — декомпозиция стоимости с монотонной смешивающей сетью; Q-значения на агента комбинируются в совместное Q, так что `argmax` распределяется чисто — доминирует на StarCraft Multi-Agent Challenge (SMAC). **MAPPO** (Yu et al., NeurIPS 2022, arXiv:2103.01955) — PPO с централизованной функцией стоимости; «удивительно эффективен» на particle-world, SMAC, Google Research Football, Hanabi с минимальной настройкой. Эти алгоритмы лежат в основе обучения политик для команд агентов, которые должны действовать децентрализованно. MAPPO — **дефолтный бейзлайн кооперативного MARL в 2026 году**. Этот урок строит каждый из маленькой игрушечной сетки и закрепляет три идеи в мышечной памяти до обращения к обучению LLM-агентов.

**Тип:** Изучение
**Языки:** Python (stdlib, маленькие реализации без NumPy)
**Предварительные требования:** Фаза 09 (Обучение с подкреплением), Фаза 16 · 09 (Параллельные сети роёв)
**Время:** ~90 минут

## Проблема

LLM-агентные системы всё больше обучают политики для межагентной координации: когда уступать, когда действовать, кого из пиров вызывать. Литература, которая учит как обучать такие политики — это Multi-Agent Reinforcement Learning (MARL), который предшествует волне LLM и имеет небольшой набор доминирующих алгоритмов.

Чтение статей MARL без словаря паттернов болезненно. Централизованное обучение с децентрализованным выполнением (CTDE), декомпозиция стоимости и централизованные критики — это не клише, а конкретные ответы на конкретные задачи:

- Независимый RL (каждый агент обучается отдельно) нестационарен с точки зрения каждого агента. Плохо.
- Централизованный RL (один агент управляет всеми) не масштабируется и нарушает ограничения выполнения.
- CTDE получает лучшее от обоих: обучение с глобальной информацией, развёртывание с локальными политиками.

## Концепция

### Три среды, которые используют статьи

- **Particle World (multi-agent particle env).** Простая 2D-физика с кооперативными/конкурентными задачами. Оригинальный стенд MADDPG.
- **StarCraft Multi-Agent Challenge (SMAC).** Кооперативный микроменеджмент, частичное наблюдение. Стенд QMIX. Дискретные действия, непрерывные состояния.
- **Google Research Football, Hanabi, MPE.** Бейзлайны MAPPO.

Разные среды имеют разные типы действий/наблюдений. Алгоритмы выбираются соответственно.

### MADDPG (2017) — паттерн CTDE

Каждый агент `i` имеет актор `mu_i(o_i)`, который отображает собственное наблюдение в действие. Каждый агент также имеет критик `Q_i(x, a_1, ..., a_n)`, который видит все наблюдения и все действия во время обучения. Актор обновляется через градиент политики относительно оценки критика.

```
обновление актора:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) при a_i=mu_i(o_i)]
обновление критика:   TD для Q_i(x, a_1..n) с совместной оценкой следующего состояния
```

Почему CTDE: во время обучения мы знаем действия всех; используем это для снижения дисперсии каждого критика. Во время развёртывания каждый агент видит только `o_i` и вызывает `mu_i(o_i)`.

Режим отказа: критики растут с числом N агентов (вход включает все действия). Не масштабируется за ~10 агентов без приближений.

### QMIX (2018) — декомпозиция стоимости

Только кооперативные. Глобальное вознаграждение — сумма монотонной функции Q-значений на агента:

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

Монотонность гарантирует, что `argmax_a Q_tot` может быть вычислен каждым агентом, выбирающим `argmax_{a_i} Q_i` независимо. Это **именно свойство децентрализованного выполнения**, которое вам нужно. Во время обучения смешивающая сеть производит `Q_tot` из Q-значений на агента.

Почему QMIX побеждает на SMAC: кооперативный микроменеджмент StarCraft имеет гомогенных агентов, локальные наблюдения, глобальное вознаграждение — идеально для декомпозиции стоимости.

Режим отказа: ограничение монотонности ограничительно; некоторые задачи имеют структуры вознаграждения, которые не являются монотонно декомпозируемыми (один агент жертвует за команду). Расширения (QTRAN, QPLEX) ослабляют это.

### MAPPO (2022) — незамеченный дефолт

Multi-Agent PPO: PPO с централизованной функцией стоимости. Каждый агент имеет собственную политику; все агенты разделяют (или имеют на агента) функции стоимости, которые видят полное состояние. Yu et al. 2022 сравнил MAPPO с MADDPG, QMIX и их расширениями на пяти бенчмарках и обнаружил:

- MAPPO соответствует или превосходит off-policy методы MARL на particle-world, SMAC, Google Research Football, Hanabi, MPE.
- Минимальная настройка гиперпараметров.
- Стабильное обучение; воспроизводимо между сидами.

Сообщество недооценивало on-policy MARL до этой статьи. В 2026 году MAPPO — дефолтный бейзлайн для кооперативного MARL; любой новый метод должен его обыграть.

### Почему инженерам LLM-агентов это важно

Три прямых применения:

1. **Обучение маршрутизатора.** Мета-агент выбирает, какой субагент обрабатывает задачу. Это задача MARL с N децентрализованными субагентами и одним централизованным маршрутизатором. MAPPO подходит.
2. **Ролевой эмерджент.** В симуляциях генеративных агентов обучение агентов принимать дополнительные роли со временем — это замаскированная задача MARL. Декомпозиция стоимости в стиле QMIX обеспечивает дополнительность по конструкции.
3. **Мультиагентное использование инструментов.** Когда агенты делят инструменты и конкурируют за бюджет, обучение через CTDE даёт локальные политики, соблюдающие ограничения ресурсов.

Практическая оговорка: в 2026 году большинство продакшен LLM-агентных систем промптят свои политики, а не обучают их. MARL подключается когда у вас (а) много данных взаимодействия, (б) чёткий сигнал вознаграждения и (в) готовность инвестировать в инфраструктуру обучения.

### CTDE как паттерн проектирования за пределами RL

Даже без обучения CTDE — полезный архитектурный паттерн:

- Во время *проектирования* предполагайте полную видимость команды.
- В *рантайме* обеспечивайте децентрализованное выполнение: каждый агент видит только `o_i`.

Паттерн заставляет вас держать состояние на агента явным и думать о частичной наблюдаемости заранее. Многие продакшен мультиагентные системы молча предполагают разделяемое состояние везде; дисциплина CTDE это предотвращает.

### Проблема нестационарности

Когда несколько агентов обучаются одновременно, среда каждого агента (включающая политики других) нестационарна. Классические доказательства одиночного RL ломаются. Алгоритмы MARL в этом уроке все решают это:

- MADDPG: глобальный критик видит все действия, поэтому его оценка стоимости стационарна.
- QMIX: декомпозиция стоимости переносит обучение в пространство совместного Q, где оптимальность определена чётко.
- MAPPO: централизованная функция стоимости подавляет дисперсию от изменений политик других.

В LLM-агентных системах нестационарность проявляется как «мой агент работал в прошлом месяце, теперь другой агент выше изменился, и мой ведёт себя неправильно». Обучение MARL с CTDE — принципиальное исправление; исправления на уровне промпта быстрее, но менее долговечны.

### Чем этот урок НЕ покрывает

Обучение реальных сетей — тема Фазы 09. Этот урок строит сценарные версии политик, демонстрирующие паттерны CTDE, декомпозиции стоимости и централизованной стоимости без обновлений градиентов. Цель — усвоить паттерны до того, как вы возьмёте полную библиотеку MARL (PyMARL, MARLlib, RLlib multi-agent).

## Реализуем

`code/main.py` реализует три демонстрации паттернов, все на крошечной кооперативной сетке 2×2 агента:

- Среда: 2 агента на сетке 4×4, одна награда-пелетка. Награда = 1, если любой агент достигает пелетки; задача завершается.
- `IndependentAgents` — каждый агент рассматривает других как среду. Бейзлайн.
- `MADDPGStyle` — централизованный критик вычисляет совместную стоимость; политики акторов обновляются от неё. Сценарное улучшение политики.
- `QMIXStyle` — декомпозиция стоимости с монотонным смешивателем.
- `MAPPOStyle` — централизованная функция стоимости; политики обновляются относительно разделяемого бейзлайна.

Все четыре запускают одни и те же эпизоды и сообщают среднее количество шагов до цели. Варианты CTDE сходятся к более коротким путям, чем независимый бейзлайн.

Запуск:

```
python3 code/main.py
```

Ожидаемый вывод: независимые агенты тратят в среднем ~6 шагов; варианты CTDE сходятся к ~3.5 шагам (оптимум для сетки 4×4 — 3). Разница в паттернах проявляется даже при сценарных политиках.

## Результат

MARL в продакшене редок. Когда вы всё же используете его:

- **Начните с MAPPO.** Статья 2022 года установила это как бейзлайн; воспроизведите его сначала, чтобы сэкономить недели на поиске более хитрых методов.
- **Логируйте поток наблюдений и действий каждого агента.** Отладка MARL без трассировок на агента безнадёжна.
- **Отделяйте код обучения от кода выполнения.** CTDE — дисциплина; позвольте пути выполнения видеть только `o_i`.
- **Предупреждение о shaped reward.** MARL предельно чувствителен к дизайну вознаграждения. Один баг координации в shaped reward, и агенты учатся его эксплуатировать. Запускайте враждебные тесты.
- **Для LLM-агентов**考虑айте политики на уровне промпта в первую очередь. Инвестируйте в обучение MARL только когда есть данные взаимодействия + сигнал вознаграждения + инфраструктура.

3. Прочитайте MADDPG (arXiv:1706.02275), раздел 3. Реализуйте точное правило обновления критика в псевдокоде своими словами.
4. Прочитайте MAPPO (arXiv:2103.01955). Почему авторы утверждают, что централизованная стоимость + PPO обыгрывает off-policy MARL на их бенчмарках? Перечислите три самых сильных утверждения.
5. Примените CTDE как паттерн проектирования к гипотетической LLM-агентной системе (например, исследовательский агент + резюматор + кодер). Какая совместная информация доступна во время проектирования, но не доступна в рантайме?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|----------------|------------------------|
| MARL | «Multi-Agent RL» | Обучение с подкреплением для мультиагентных систем. |
| CTDE | «Централизованное обучение, децентрализованное выполнение» | Обучение с глобальной информацией; развёртывание с локальными политиками. |
| MADDPG | «Multi-Agent DDPG» | CTDE с критиком на агента, видящим все наблюдения + действия. |
| QMIX | «Декомпозиция стоимости» | Монотонное смешивание Q-значений на агента. Кооперативный. |
| MAPPO | «Multi-Agent PPO» | PPO с централизованной функцией стоимости. Дефолтный бейзлайн 2026 года. |
| Декомпозиция стоимости | «Сумма индивидуальных Q» | Совместное Q представлено как монотонная функция Q-значений на агента. |
| Нестационарность | «Движущиеся цели» | Среда каждого агента меняется по мере обучения других. Центральная проблема MARL. |
| On-policy / off-policy | «Учиться от текущего / из воспроизведения» | PPO — on-policy (MAPPO); DDPG и Q-learning — off-policy. |
