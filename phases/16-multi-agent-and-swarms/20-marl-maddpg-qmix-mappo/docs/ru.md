# MARL — MADDPG, QMIX, MAPPO

Объяснение:
MARL (Multi-Agent Reinforcement Learning), MADDPG (Multi-Agent Deep Deterministic Policy Gradient), QMIX (Quantile Mixing) и MAPPO (Multi-Agent Proximal Policy Optimization

> Наследие обучения усилением в координации многоагентных систем, которое все еще влияет на системы агентов на основе LLM в 2026 году. **MADDPG** (Lowe et al., NeurIPS 2017, arXiv:1706.02275) представил Централизованное обучение, Декentralизированное выполнение (CTDE): во время обучения каждый критик видит состояния и действия всех агентов; в режиме тестирования только локальные акторы работают. Работает для кооперативных, конкурентных и смешанных сценариев. **QMIX** (Rashid et al., ICML 2018, arXiv:1803.11485) использует декомпозицию значений с монотонной сетью смешивания; персональные Q-оценки объединяются в общую Q-оценку так, что `argmax` распределяется легко — преуспевает на StarCraft Multi-Agent Challenge (SMAC). **MAPPO** (Yu et al., NeurIPS 2022, arXiv:2103.01955) это PPO с централизованной функцией ценности; "неожиданно эффективен" на мирах частиц, SMAC, футбольных играх Google Research и Hanabi при минимальном настройке. Эти методы служат основой для тренировочных политик агентских команд, которые должны действовать декентрализованно. MAPPO является **стандартной базовой линией кооперативного MARL в 2026 году**. Этот урок строит каждый из них на основе маленькой сетевой игры-игрушки и закладывает эти три идеи в мышечную память до того, как касаться тренировки агентов на основе LLM.

## Проблема

Системы агентов на основе больших языковых моделей все более активно тренируют политики для координации между агентами: когда следует передать задачу, когда действовать самому, к какому коллеге обращаться. Литература, которая объясняет, как обучать такие политики, называется Многоагентное усилительное обучение (MARL), что предшествует волны LLM и имеет небольшое количество доминирующих алгоритмов.

Чтение статей о MARL без терминологии неудобно. Централизованное обучение с децентрализованным выполнением (CTDE), декомпозиция значений и централизованные критики — это не buzz-слова, а конкретные решения для конкретных проблем:

- Индиpendентное обучение с помощью методов принятия решений (RL) (каждый агент учится самостоятельно) является нестационарным с точки зрения каждого агента. Это плохо.
- Централизованное обучение с помощью методов принятия решений (один агент управляет всеми) не масштабируется и нарушает ограничения выполнения.
- CTDE берет лучшее от обоих подходов: тренировка с глобальной информацией, развертывание с локальными политиками.

## Концепция

Три среды, которые используются в статьях:

- **Мир пылинок (многоагентная среда с частицами).** Простая 2D физика с кооперативными/конкурентными задачами. Оригинальная тестовая площадка MADDPG.
- **Командный вызов StarCraft Multi-Agent Challenge (SMAC).** Кооперативное макро-управление, частичное наблюдение. Тестовая площадка для QMIX. Дискретные действия, непрерывные состояния.
- **Футбол Google Research, Hanabi, MPE.** Базовые линии MAPPO.

Разные окружения имеют разные типы действий/наблюдений. Алгоритмы выбирают в соответствии с этим.

MADDPG (2017) — это шаблон МНКДЭ (многоагентное дифференциальное программирование агентов).

Каждый агент `i` имеет актора `mu_i(o_i)`, который отображает его собственное наблюдение в действие. Каждый агент также имеет критика `Q_i(x, a_1, ..., a_n)`, который видит все наблюдения и все действия во время обучения. Аktor обновляется с помощью градиента политики против оценки критика.

Каждый агент `i` имеет актора `mu_i(o_i)`, который отображает его собственное наблюдение в действие. Каждый агент также имеет критика `Q_i(x, a_1, ..., a_n)`, который видит все наблюдения и все действия во время обучения. Аktor обновляется с помощью градиента политики против оценки критика.

```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

CTDE за что: во время обучения мы знаем действия всех агентов; поэтому используем это для уменьшения дисперсии в каждом критике. Время развертывания: каждый агент видит только `o_i` и вызывает `mu_i(o_i)`.

Режим сбоя: критики растут с N агентами (вход включает все действия). Без приближений не масштабируется свыше ~10 агентов.

QMIX (2018) — декомпозиция значений

Кооперативная игра только. Общий вознаграждение является суммой монотонной функции значений Q для каждого агента:

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

Монотонность гарантирует, что `argmax_a Q_tot` можно вычислить путем независимого выбора каждого агента `argmax_{a_i} Q_i`. Это именно **свойство распределенного выполнения**, которое вам нужно. Время обучения сетка смешивания производит `Q_tot` из индивидуальных `Q_i`.

Почему QMIX выигрывает в SMAC: кооперативное управление макросоставляющими в StarCraft имеет однородных агентов, локальное наблюдение и глобальную награду — идеальная среда для декомпозиции значений.

Режим сбоев: ограничение монотонности слишком строго; некоторые задачи имеют структуры вознаграждений, которые не могут быть разложены по монотонным компонентам (один агент sacрифицирует себя для команды). Расширения (QTRAN, QPLEX) смягчают это ограничение.

MAPPO (2022) — незамеченное по умолчанию значение

Мультиагент PPO: PPO с централизованной функцией ценности. Каждый агент имеет свой собственный политический план; все агенты делят (или имеют индивидуальные) функции ценности, которые видят полное состояние. Ю и соавторы 2022 года провели сравнение MAPPO с MADDPG, QMIX и их расширениями на пяти бенчмарках и обнаружили:

- MAPPO соответствует или превосходит методы несвязанного MARL вParticle-world, SMAC, футболе Google Research, Hanabi и MPE.
- Минимальная настройка гиперпараметров.
- Устойчивая тренировка; воспроизводима при различных начальных условиях.

Комьюнити недооценивало наполитичность MARL до этой статьи. В 2026 году MAPPO станет стандартной базовой линейкой для кооперативного MARL; любая новая методика должна превзойти ее.

Эта фраза может быть использована в начале раздела или абзаца, где речь идет о важности определенных аспектов для инженеров, работающих с агентами на основе больших языковых моделей (LLM-agent engineers). Если вы хотите перевести весь этот заголовок без дополнений, это будет так.

Три прямых использования:

1. **Обучение роутера.** Мета-агент выбирает, какой под-агент будет обрабатывать задание. Это проблема несколько агентов с разнородными действиями (MARL), в которой N децентрализованных под-агентов контролируются одним центральным роутером. MAPPO подходит для этой задачи.
2. **Выделение ролей.** В симуляциях генеративных агентов обучение агентам принимать взаимозависимые роли со временем является, по сути, MARL-проблемой. Разложение ценности в стиле QMIX заставляет агентов работать вместе путем их конструктивного дизайна.
3. **Многокорпусное использование инструментов.** Когда агенты делятся инструментами и конкурируют за бюджет, обучение их через CTDE производит локальные политики, которые уважают ограничения ресурсов и могут быть внедрены в реальную среду.

Практическое замечание: в 2026 году большинство производственных систем агентов на основе LLM будут подсказывать свои политики вместо обучения им. MARL приходит, когда у вас есть (а) много данных взаимодействия, (б) ясный сигнал вознаграждения, и (в) готовность инвестировать в инфраструктуру для обучения.

CTDE как шаблон проектирования за пределами RL

Несмотря на то что у него нет специальной подготовки, CTDE является полезным архитектурным шаблоном:

- В процессе *проектирования* предполагается полная видимость всей команды.
- В процессе *выполнения*, обеспечьте децентрализованное выполнение: каждый агент видит только `o_i`.

Паттерн заставляет вас явно указывать состояние для каждого агента и думать о частичной наблюдаемости с самого начала. Многие производственные многокорпусные системы тихо предполагают общее состояние повсюду — дисциплина CTDE предотвращает это.

Проблема нестационарности

Когда несколько агентов учатся одновременно, окружение каждого агента (включая политики других агентов) нестационарно. Классические доказательства в области однородного обучения агента пошагово перестают работать. Все алгоритмы MARL в этом уроке направлены на решение этой проблемы:

- MADDPG: глобальный критик видит все действия, поэтому его оценка значения является стационарной.
- QMIX: декомпозиция значений переносит обучение в пространство совместного Q-значения, где оптимальность хорошо определена.
- MAPPO: централизованная функция ценности смягчает разброс, вызванный изменениями других политик.

В системах с LLM-агентами нестационарность проявляется как "мои агенты работали в прошлом месяце, теперь, когда вышеуровневый агент изменился, мои агенты начинают неадекватно себя вести." Принципиальным решением является обучение MARL с использованием CTDE; исправления на уровне команд работают быстрее, но менее надежно.

## Собери его

`code/main.py` реализует три демонстрации шаблонов, все они базируются на маленьком сетевом мире с кооперацией двух агентов:

- Окружение: 2 агента на сетке размером 4x4, одна награда в виде куска пищи. Награда равна 1, если любой из агентов достигает куска пищи; задача завершается.
- `IndependentAgents` — каждый агент воспринимает других как часть окружения. Базовая версия.
- `MADDPGStyle` — центральный критик вычисляет совместное значение; политики актёров обновляются на основе этого значения. Улучшение политик с помощью заранее заданных стратегий.
- `QMIXStyle` — декомпозиция значений с использованием монотонного смесителя.
- `MAPPOStyle` — центральная функция ценности; политики обновляются по отношению к общему базовому значению.

Все четыре версии используют одни и те же эпизоды и отчитываются о среднем количестве шагов до цели. Версии с вариациями CTDE сходятся к корочешим путям, чем базовая независимая версия.

## Отправляем!

MARL в производстве редко используется. Когда же его все-таки используют:

- **Начните с MAPPO.** В статье 2022 года это было установлено как базовый метод; воспроизведение его первым экономит несколько недель на тестировании более сложных подходов.
- **Записывайте каждое наблюдение и поток действий для каждого агента.** Отладка MARL без отслеживания по агентам практически невозможна.
- **Разделяйте код обучения и код выполнения.** CTDE — это дисциплина; пусть путь выполнения видит только `o_i`.
- **Предупреждение о формировании вознаграждений.** MARL чрезвычайно чувствителен к дизайну вознаграждений. Одна координационная ошибка в формировании и агенты научатся использовать её для своего выгодного положения. Выполняйте противоположные тесты.
- **Для агентов на основе LLM**, сначала рассмотрите политики на уровне запроса. Инвестируйте только в обучение MARL, когда есть взаимодействие данных + сигнал вознаграждения + инфраструктура.

## Ключевые термины

| Термин | Что говорят люди | На самом деле означает |
|--------|------------------|----------------------|
| MARL | "Многоагентное обучение с подкреплением" | Обучение с подкреплением для многоагентных систем. |
| CTDE | "Централизованное обучение, децентрализованное выполнение" | Обучение с глобальной информацией; развертывание с локальными политиками. |
| MADDPG | "Многоагентное DDPG" | CTDE с критерием для каждого агента, который видит все наблюдения и действия. |
| QMIX | "Декомпозиция значений" | Монотонная смешивание индивидуальных Q-значений. Коллаборативное. |
| MAPPO | "Многоагентное PPO" | PPO с централизованной функцией ценности. 2026 годовой стандартный базовый вариант. |
| Декомпозиция значений | "Сумма индивидуальных Q-значений" | Объединенное Q представлено как монотонная функция от индивидуальных Q-значений каждого агента. |
| Нестационарность | "Движущиеся цели" | Окружение каждого агента меняется с течением времени из-за обучения других агентов. Центральная проблема MARL. |
| Политика на пути / политика за пределами пути | "Учиться от текущего / воспроизведения" | PPO — это политика на пути (MAPPO); DDPG и Q-обучение — это политика за пределами пути. |
| SMAC | "Командный вызов многоагентного StarCraft" | Бенчмарк кооперативной макрорегулирования; домашняя почва для QMIX. |
