# Экономика агентов, токен-стимулы, репутация

> Долгосрочные автономные агенты (рабочая кривая METR от 1 до 8 часов) нуждаются в экономической дееспособности. Формирующийся **5-уровневый стек**: **DePIN** (физические вычисления) → **Идентификация** (W3C DID + капитал репутации) → **Познание** (RAG + MCP) → **Расчёты** (абстракция аккаунтов) → **Управление** (Agentic DAO). Продакшен-сети агентных стимулов включают **Bittensor** (подсети TAO вознаграждают модели для конкретных задач), **Fetch.ai / ASI Alliance** (LLM ASI-1 Mini + токен FET) и **Gonka** (proof-of-work на основе трансформеров, перераспределяющий вычисления на продуктивные ИИ-задачи). Академическая работа: децентрализованная LaMAS от AAMAS 2025 использует **атрибуцию заслуг по Шепли** для справедливого вознаграждения вносящих вклад агентов; «Mechanism design for large language models» от Google Research предлагает **токен-аукционы** со вторым ценовым платежом при монотонной агрегации. Этот урок строит минимальный рынок агентов, применяет атрибуцию заслуг по Шепли к мультиагентному пайплайну и запускает вторую ценовую токен-аукцион, чтобы теоретико-игровой механизм укрепился конкретно.

**Тип:** Изучение
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 16 · 16 (Переговоры и торг), Фаза 16 · 09 (Параллельные сети роёв)
**Время:** ~75 минут

## Проблема

Мультиагентные системы усложняются, когда агенты создают стоимость совместно, но должны вознаграждаться индивидуально. Классические механизмы — равный разделение, «последний забирает всё» — несправедливы или ломаемы. Коалиционное вознаграждение через значения Шепли справедливо по конструкции, но дорого в вычислении. Литература 2025-2026 годов толкает полезные приближения: сэмплирование Шепли, аукционы с монотонной агрегацией и ончейн-репутация, накапливаемая из подтверждённых вкладов.

Помимо атрибуции заслуг, область обратилась к реальным экономическим агентам: Bittensor TAO вознаграждает майнинг вычислений для файнтюнинга моделей в подсетях, Fetch.ai/ASI вознаграждает использование LLM ASI-1 Mini токенами FET, Gonka перераспределяет proof-of-work трансформеров на продуктивные ИИ-задачи. Агенты, ведущие автономные транзакции, существуют сегодня; вопрос — как выровнять стимулы.

Этот урок рассматривает экономику агентов как конкретное семейство задач — атрибуция заслуг, проектирование механизмов и репутация — и строит каждое с минимальной математикой, чтобы идеи закрепились.

## Концепция

### 5-уровневый стек экономики агентов

1. **DePIN (физические вычисления).** Децентрализованная инфраструктура, сдающая в аренду GPU, хранилище, пропускную способность. Подсети Bittensor, Render Network, Akash. Не агентно-специфичная; агенты её используют.
2. **Идентификация.** W3C Decentralized Identifiers (DID) дают каждому агенту долговечный ID, независимый от любой платформы. Репутация накапливается к DID. Agent Network Protocol (ANP) использует DID как слой обнаружения.
3. **Познание.** Цикл рассуждений агента: LLM + RAG + MCP. Это то, что строят другие фазы.
4. **Расчёты.** Абстракция аккаунтов (ERC-4337) позволяет агентам оплачивать газ из собственных балансов, не владея ETH. Агенты могут платить за сервисы, друг другу или за вычисления.
5. **Управление.** Agentic DAO: структуры управления, где люди *и* агенты голосуют за изменения протокола, с силой голоса, привязанной к репутации.

Не каждая продакшен-система использует все пять. Bittensor использует 1, 2, частично 3, частично 4, ни одного из 5. Агенты OpenAI используют только 3. Стек — карта-референс, а не требование.

### Bittensor, Fetch.ai, Gonka — что работает

**Bittensor (TAO).** Подсети — специализированные задачи (моделирование языка, генерация изображений, прогнозирование). Майнеры предоставляют выходы моделей. Валидаторы ранжируют их; скоринг с учётом стейка распределяет награды TAO. Каждая подсеть имеет собственную оценку. Экономический урок: платите за качество выхода конкретной задачи, а не за использованные вычисления.

**Fetch.ai / ASI Alliance.** LLM ASI-1 Mini работает в сети Fetch.ai; пользователи платят токенами FET за инференс. Нарратив «агенты как пиры» здесь сильнее: агент на Fetch может вызвать другого для задачи и заплатить в FET.

**Gonka.** Proof-of-work трансформеров: «работа» — это прямые проходы трансформера. Майнеры зарабатывают на запуске задач инференса, которые имеют известные правильные выходы (из данных обучения). Ресурсопродуктивный PoW вместо хэш- based PoW.

Все три являются продакшен-уровня на апрель 2026. Распределение выплат различается. Bittensor вознаграждает качество относительно валидаторов подсети; Fetch вознаграждает полезность, измеряемую платящими пользователями; Gonka вознаграждает верифицируемую работу инференса.

### Атрибуция заслуг по Шепли

Три агента сотрудничают над задачей. Выход набирает 0.8. Кто внёс что?

Значение Шепли: уникальное распределение заслуг, удовлетворяющее четырём аксиомам (эффективность, симметрия, линейность, нулевой вклад). Для агента `i`:

```
shapley(i) = (1/N!) * сумма по всем упорядочениям O (v(S_i_O ∪ {i}) - v(S_i_O))
```

где `S_i_O` — множество агентов до `i` в упорядочении `O`. На практике: перечислите все перестановки, запишите маргинальный вклад каждого агента в каждой перестановке, усредните.

Для N=3 агентов — 6 перестановок. Для N=10 — 3.6M, поэтому на практике сэмплируют упорядочения вместо перечисления.

### Вторая цена аукциона для агрегации

Google Research («Mechanism design for large language models») предлагает токен-аукционы со второй ценой для агрегации выходов LLM. Настройка: N агентов предлагают завершение; каждый имеет приватную стоимость за то, чтобы быть выбранным. Аукционист выбирает предложение с наибольшей стоимостью и платит *вторую по величине* стоимость. При монотонной агрегации (стоимость зависит от того, какое предложение выбрано, а не от того, сколько было предложено) это правдиво — агенты предлагают свою реальную стоимость.

Почему это важно для LLM-систем: можно передавать задачи завершения нескольким агентам с разными ценами; аукцион выбирает лучшего + платит справедливо, и агенты не имеют стимула искажать报价.

### Капитал репутации

Оценка репутации, привязанная к DID, накапливается из подтверждённых вкладов. Простое правило обновления:

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

С коэффициентом затухания `alpha` близким к 1. Репутация:

- Дешёва в чтении для решений маршрутизации («отправляйте сложные задачи агентам с высокой репутацией»).
- Дорога в подделке (накапливается со временем, привязана к DID).
- Может быть обрезана: вклады, не прошедшие верификацию, вычитаются.

### AAMAS 2025 децентрализованная LaMAS

Предложение LaMAS (AAMAS 2025) комбинирует: идентификацию DID, атрибуцию заслуг по Шепли и простой аукционный механизм. Ключевое утверждение: децентрализация шага атрибуции заслуг делает систему аудитируемой и устойчивой к точечной манипуляции.

### Где экономика ломается

- **Манипуляция ценовым оракулом.** Если функцию заслуг можно сломать, агенты её сломают. Каждый механизм нуждается во враждебном тесте.
- **Sybil-атаки.** Один оператор поднимает N фейковых агентов для раздувания собственного вклада. DID замедляют, но не останавливают это; стоимость подделки репутации — смягчение.
- **Стоимость верификации.** Атрибуция заслуг справедлива ровно настолько, насколько справедлив верификатор. Если верификация дешёвая (маленький LLM), её можно сломать; если дорогая (панель людей), система не масштабируется.
- **Регуляторный висящий.** Экономика агентов пересекается с финансовой регуляцией. Bittensor, Fetch и Gonka все работают в правовых серых зонах в некоторых юрисдикциях на 2026 год.

### Когда экономика агентов имеет смысл

- **Открытые сети с гетерогенными операторами.** Ни одна команда не контролирует всех агентов.
- **Верифицируемые выходы.** Без верификации атрибуция заслуг — догадка.
- **Долгосрочные рабочие процессы.** Задачи one-shot не выигрывают от накопления репутации.
- **Токенизированные платежи юридически жизнеспособны** в вашей юрисдикции.

В закрытых корпоративных системах экономика уступает более простому распределению (менеджеры назначают работу, метрики внутренние). Экономическая литература применима в основном к открытым сетям.

## Реализуем

`code/main.py` реализует:

- `shapley(value_fn, agents)` — точное вычисление Шепли перечислением для малых N.
- `second_price_auction(bids)` — правдивый механизм; победитель платит вторую цену.
- `Reputation` — привязанная к DID репутация с экспоненциальным затуханием и обрезкой.
- Демо 1: три агента сотрудничают, точный Шепли распределяет заслуги.
- Демо 2: пять агентов торгуют за слот задачи; аукцион со второй ценой выбирает победителя + платёж.
- Демо 3: 100 раундов назначения задач агентам с гетерогенной репутацией; маршрутизация с весом репутации обыгрывает случайную.

Запуск:

```
python3 code/main.py
```

Ожидаемый вывод: значения Шепли для каждого агента; результат аукциона, показывающий равновесие правдивой ставки; маршрутизация с весом репутации, показывающая 10-20% прирост качества над случайной после прогрева.

## Результат

Запуск экономики агентов в 2026:

- **Начните с репутации, а не с токенов.** Репутация дешёва в реализации и ценна сама по себе; токены добавляют юридическую и экономическую сложность.
- **Верифицируйте до вознаграждения.** Никогда не распределяйте заслуги без независимого шага верификации. Самооценка качества порождает sybil-игры.
- **Сэмплирование Шепли, а не точное Шепли.** Сэмплируйте 100-1000 упорядочений; точное перечисление не масштабируется.
- **Ограничьте коэффициент затухания и установите пол репутации.** Неограниченное затухание стирает легитимных вкладчиков; слишком медленное затухание вознаграждает устаревших агентов с высокой репутацией.
- **Проверяйте механизмы враждебно.** Запускайте краснокомандные сценарии до открытия сети. У каждого механизма есть теория игр; вы хотите найти дыры, а не атакующих.

## Упражнения

1. Запустите `code/main.py`. Подтвердите, что значения Шепли суммируются до общей стоимости (аксиома эффективности). Измените функцию стоимости; меняются ли распределения Шепли в ожидаемом направлении?
2. Реализуйте *сэмплирование* Шепли (Монте-Карло по K упорядочениям). Как K влияет на точность приближения? Сравните с точным для N=4.
3. Реализуйте шаг формирования коалиции перед аукционом: агенты могут объединяться в команды и торговать как единица. Какие коалиции формируются? Является ли результат Парето-лучшим по сравнению с индивидуальными ставками?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|----------------|------------------------|
| DePIN | «Децентрализованная физическая инфраструктура» | Токенизированные вычисления/хранилище/пропускная способность. Bittensor, Akash, Render. |
| DID | «Децентрализованный идентификатор» | Спецификация W3C для портируемых ID. Репутация агента привязана к DID, а не к платформе. |
| ERC-4337 | «Абстракция аккаунтов» | Контрактные аккаунты, которые могут спонсировать газ, обеспечивая платежи агентов. |
| Значение Шепли | «Справедливая атрибуция заслуг» | Уникальное распределение, удовлетворяющее эффективности, симметрии, линейности, нулевому вкладу. |
| Аукцион со второй ценой | «Аукцион Викри» | Правдивый механизм: победитель платит вторую ставку. Совместим с монотонной агрегацией. |
| Капитал репутации | «Накопленная оценка качества» | Оценка, привязанная к DID, из подтверждённых вкладов; затухает со временем. |
| Agentic DAO | «Агенты + люди управляют» | DAO с агентами-голосующими как первоклассными, сила голоса привязана к репутации. |
| TAO / FET / GPU credits | «Токенные номиналы» | Bittensor TAO, Fetch.ai FET, различные токены DePIN. |
