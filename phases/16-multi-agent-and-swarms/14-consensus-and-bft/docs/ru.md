# Консенсус и византийская отказоустойчивость для агентов

> Классическая BFT распределённых систем встречается со стохастическими LLM. В 2025-2026 возникли три направления исследований: **CP-WBFT** (arXiv:2511.10400) взвешивает каждый голос по зонду уверенности; **DecentLLMs** (arXiv:2507.14928) работает без лидера с параллельными предложениями работников и агрегацией по геометрической медиане; **WBFT** (arXiv:2505.05103) комбинирует взвешенное голосование с иерархической кластеризацией структуры для разделения узлов Core и Edge. Честный эмпирический результат из «Can AI Agents Agree?» (arXiv:2603.01213) в том, что даже скалярное согласие хрупко сегодня — один обманывающий агент может скомпрометировать Mixture-of-Agents. BFT необходима, но недостаточна. Этот урок строит минимальный BFT-протокол, инжектирует три агентно-специфических атаки (византийская ложь, льстивая конформность, коррелированная ошибка монокультуры) и измеряет, как справляется каждый вариант консенсуса.

**Тип:** Изучение + Практическая работа
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 16 · 07 (Общество разума и дебаты), Фаза 16 · 13 (Общая память)
**Время:** ~75 минут

## Проблема

У вас N LLM-агентов, каждый из которых производит ответ. Они не согласны. Голосование большинства выбирает неправильный, потому что два агента коррелированы (одна базовая модель, одни данные обучения, одинаковые режимы отказа). Третий агент оказывается неправ новым способом — поэтому большинство ложное.

Теперь добавьте обманывающего агента: он лжёт нарочно. Или льстивого агента: он соглашается с последним выступившим. В классической BFT предполагается, что византийские узлы — доля `f < n/3` и ведут себя произвольно. Реальность 2026 года в том, что LLM-узлы стохастичны даже при честности, коррелированы между моделями и влияют друг на друга. Их нельзя рассматривать как независимых Бернулли-голосующих.

Классическая BFT (PBFT, 1999) не неправильна — она неполна. Она обрабатывает произвольное переворачивание битов. Она не обрабатывает «три честных агента разделяют галлюцинацию, потому что разделяют данные обучения». Этот урок строится на фундаменте PBFT и наслаивает три адаптации 2025-2026.

## Концепция

### Что даёт классическая BFT

Практическая византийская отказоустойчивость (Castro & Liskov, OSDI 1999) допускает `f < n/3` византийских узлов. Протокол имеет три фазы (pre-prepare, prepare, commit) и два примитива (подписанные сообщения, сертификаты кворума). Согласие по одному значению среди `n >= 3f + 1` честных-или-вредоносных узлов.

Гарантии сильные, но предполагают:

1. **Независимые сбои.** Византийские не координируются.
2. **Честные узлы действительно честны.** Корректность честных выходов — не вопрос; протокол только выравнивает разногласия.
3. **Вопрос имеет фактический ответ.** Консенсус по неправильному факту всё ещё консенсус.

LLM-агенты нарушают все три. Два агента с одной базовой моделью разделяют сбои. «Честный» LLM всё ещё галлюцинирует. И на неоднозначных вопросах «истина» — это то, что решают агенты; внешнего оракула нет.

### Три атаки, специфичных для LLM

**Византийская ложь.** Один агент выводит намеренно неправильный ответ. Классическая BFT обрабатывает это если `f < n/3`.

**Льстивая конформность.** Один агент читает ответы других перед голосованием и выравнивается с последним выступившим. Не злонамеренно, но коррелирует с самым громким. Классическая BFT не предотвращает это, потому что агент проходит каждую проверку подписи.

**Коррелированная ошибка монокультуры.** Три агента разделяют одну базовую модель. Они галлюцинируют один неправильный ответ. Большинство неправильно. Классическая BFT не помогает, потому что все три «честно» согласны.

### Ответы 2025-2026

**CP-WBFT** (arXiv:2511.10400) — Confidence-Probed Weighted BFT. Каждый голосующий прикрепляет зонд уверенности к своему ответу (самооценочная вероятность или предсказание отдельной калибровочной модели). Веса голосов масштабируются с уверенностью. Сообщается +85.71% улучшение BFT на полных графах. Смягчение для: льстивой конформности (льстивые агенты tend иметь низкую уверенность на предложенной позиции).

**DecentLLMs** (arXiv:2507.14928) — Без лидера. Рабочие агенты предлагают параллельно, оценивающие агенты оценивают предложения, финальный ответ — геометрическая медиана оценённых позиций. Устойчиво при `f < n/2`. Смягчение для: византийской лжи и коррелированных ошибок (геометрическая медиана устойчива к выбросам и тянет к плотному кластеру, а не к среднему с систематическим сдвигом модели).

**WBFT** (arXiv:2505.05103) — Weighted BFT with Hierarchical Structure Clustering. Веса голосов назначаются по качеству ответа плюс оценка доверия, выученная из истории. Кластеризуйте агентов на Core и Edge; агенты Core должны достичь консенсуса первыми, агенты Edge следуют. Смягчение для: масштабируемости (консенсус Core маленький и быстрый) и частично для монокультуры (Core можно выбрать по разнообразию).

### Эмпирический результат: «Can AI Agents Agree?» (arXiv:2603.01213)

Статья измеряет скалярное согласие (LLM-агенты соглашаются об одном числовом значении) на нескольких передовых моделях. Находка тревожная:

- Даже без противников LLM-агенты не согласны по скалярным вопросам с частотой выше 30% на многих бенчмарках.
- Один агент, принимающий обманывающую личность, может сдвинуть консенсус Mixture-of-Agents на 40+ процентных пунктов от честного бейзлайна.
- Частота разногласий коррелирует с разнообразием моделей — гетерогенные ансамбли не согласны больше гомогенных (хорошо: некоррелированные ошибки), но также медленнее сходятся (плохо: дольше время до согласия).

Вывод: BFT даёт механизм для выравнивания выходов, но не говорит, правильный ли выровненный выход. Комбинируйте с верификацией (Фаза 16 · 08 ролевая специализация), разнообразием (Фаза 16 · 15 варианты дебатов) и оценивающими агентами (Фаза 16 · 24 бенчмарки).

### Ядро протокола, упрощённое

Минимальный раунд BFT для LLM-агентов:

```
1. задача поступает; каждый агент i производит ответ a_i
2. каждый агент прикрепляет зонд уверенности c_i в [0, 1]
3. агрегатор собирает (a_i, c_i) от всех n агентов
4. агрегатор группирует по семантическому кластеру (эквивалентные ответы)
5. агрегатор вычисляет вес для каждого кластера C:
     w(C) = sum_{i in C} c_i
6. победитель = кластер с макс. весом, если макс > порог * sum(c_i)
   иначе: повтор или эскалация
7. меньшинственные кластеры логируются с прослеживацией для постмортем-аудита
```

Шаг семантической кластеризации — LLM-специфический поворот. Два ответа «исследование сообщает 4.2%» и «улучшение на 4.2%» — один кластер. Наивная проверка строкового равенства это пропустит. В продакшене используйте дешёвую модель эмбеддингов или явную канонизацию.

### Настройка порога

Параметр `threshold` решает когда принимать, а когда повторять. Слишком низкий: принимаете слабые большинства. Слишком высокий: никогда ничего не принимаете. Эмпирический диапазон: 0.5-0.67 для `n=5-7` агентов, выше для меньших `n`. Ниже порога — эскалация человеку или другому ансамблю агентов.

### Где консенсус не помогает

- **Неоднозначные вопросы.** Если у вопроса нет фактического ответа, консенсус — мнение. Назовите это так.
- **Составные вопросы.** «Напишите код и объясните его» — два ответа. Голосуйте за каждый независимо.
- **Враждебный много раундов.** Если агенты наблюдают предыдущие раунды и имитируют (дебаты Du 2023), они начинают соглашаться независимо от истины. Ограничьте раунды (обычно 2-3).

## Реализуем

`code/main.py` реализует:

- `AgentVoter` — сценарная политика с (ответ, уверенность).
- `MajorityVote` — классическое простое большинство.
- `CPWBFT` — голосование с весом уверенности и семантической кластеризацией.
- `DecentLLMs` — агрегация по геометрической медиане оценённых предложений.
- `Scenario` — запускает каждый агрегатор под тремя паттернами атак.

Реализованные паттерны атак:

1. `byzantine`: один агент лжёт с высокой уверенностью.
2. `sycophancy`: один агент копирует первый увиденный ответ с совпадающей уверенностью.
3. `monoculture`: три агента разделяют неправильный ответ (коррелированная ошибка) с умеренной уверенностью.

Запуск:

```
python3 code/main.py
```

Ожидаемый вывод: таблица (атака, агрегатор) → финальный ответ, с выделенным правильным ответом. Простое большинство проваливает случай монокультуры. Взвешивание по уверенности в CPWBFT смягчает лесть. Геометрическая медиана DecentLLMs тянет к честному кластеру, когда монокультура меньше половины популяции.

## Результат

До выпуска любого механизма консенсуса:

- **Враждебно тестируйте как минимум тремя паттернами выше.** Ваш протокол должен предсказуемо падать, а не молча.
- **Логируйте каждый меньшинственный кластер** с прослеживанией. Меньшинственные кластеры — ваша система раннего предупреждения для коррелированных ошибок.
- **Обеспечьте ограниченные раунды.** Без «продолжайте дебаты до согласия» — это награждает лесть.
- **Разделяйте согласие и корректность.** Выход консенсуса идёт верификатору; верификатор независим от ансамбля.
- **Отслеживайте частоту согласия.** Резкий рост означает искажение конформности; резкое падение — дрейф модели.

## Упражнения

1. Запустите `code/main.py`. Подтвердите, что простое большинство проваливает атаку монокультуры, но CPWBFT частично смягчает её когда уверенность монокультуры ниже 0.7.
2. Добавьте четвёртый паттерн атаки: **молчаливый отказ** — один агент отказывается отвечать («Я не знаю»). Как каждый агрегатор должен относиться к отказам? Реализуйте свой выбор.
3. Замените семантическую кластеризацию со строковой канонизации на сходство эмбеддингов (используйте любую open-source модель эмбеддингов). Что происходит с атакой лести?
4. Прочитайте CP-WBFT (arXiv:2511.10400). Реализуйте шаг калибровки зонда уверенности (отдельная калибровочная модель проверяет самооценочную уверенность каждого агента). Измерьте прирост точности на сценарии монокультуры.
5. Прочитайте «Can AI Agents Agree?» (arXiv:2603.01213). Воспроизведите упрощённый эксперимент по скалярному согласию: три агента, один скалярный вопрос, промпт обманывающей личности. Ловит ли это CPWBFT или DecentLLMs?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|----------------|------------------------|
| BFT | «Византийская отказоустойчивость» | Протокол Castro-Liskov 1999 для консенсуса с `f < n/3` произвольными сбоями. |
| Византийский | «Любое плохое поведение» | Узел, который может лгать, терять сообщения, молча падать — всё кроме безопасного краха. |
| Зонд уверенности | «Насколько вы уверены?» | Самооценочная или калиброванная вероятность, прикреплённая к голосу. |
| Семантическая кластеризация | «Один ответ, другие слова» | Группировка эквивалентных ответов перед подсчётом голосов. |
| Геометрическая медиана | «Устойчивый центр» | Точка, минимизирующая сумму расстояний до точек выборки. Устойчива к выбросам, в отличие от среднего. |
| Монокультура | «Одна модель — одни сбои» | Коррелированные ошибки когда агенты разделяют данные обучения или базовую модель. |
| Льстивая конформность | «Согласие с громким» | Голос агента смещается в сторону первого/самого громкого выступающего. |
| Core/Edge | «Иерархическая BFT» | Разделение WBFT: маленький консенсус Core первыми, узлы Edge следуют. Ограничивает латентность. |
