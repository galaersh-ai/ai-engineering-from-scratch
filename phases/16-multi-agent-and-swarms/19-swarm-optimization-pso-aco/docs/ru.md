# Оптимизация стаи для ЛЛМ (ПСО, АКо)

Биоинспирированное оптимизация возвращается к жизни с LLM. **LMPSO** (arXiv:2504.09247) использует PSO, где скорость каждой частицы является запросом, а LLM генерирует следующий кандидат; работает хорошо на структурированных последовательностях выходных данных (математические выражения, программы). **Model Swarms** (arXiv:2410.11163) рассматривает каждый LLM-эксперта как частицу PSO в пространстве моделей и отмечает **средний прирост 13.3%** над 12 базовыми линиями на 9 наборах данных с только 200 экземплярами. **SwarmPrompt** (ICAART 2025) гибридизирует PSO + Grey Wolf для оптимизации запроса. **AMRO-S** (arXiv:2603.12933) — это специалисты по феромонам, вдохновленные ACO, для многоагентного маршрутизации LLM — **4,7 раза ускорение**, интерпретируемое доказательство маршрутизации, качественно-ограниченное асинхронное обновление, которое разделяет инференцию от обучения. Этот урок реализует PSO в пространстве параметров запросов и ACO в маршрутизации агентов, измеряет, почему эти классические алгоритмы подходят для эры LLM, и когда они этого не делают.

**Тип:** Учиться + Строить
**Языки:** Python (stdlib)
**Предварительные знания:** Фаза 16 · 09 (Параллельные сети спутников), Фаза 16 · 14 (Консенсус и BFT)

## Проблема

У вас есть стимул, который получает оценку 62% на оценке задачи. Вы хотите улучшить его. Простой способ — ручное безградиентное微调，但这会 scalability 很差。强化学习需要奖励信号和足够的rollouts来训练。通过反向传播对提示进行优化实际上是不可能的——提示是一个离散字符串，而不是可微参数。
以下是更自然的俄文翻译：

У вас есть стимул, который получает оценку 62% на задаче оценки. Вы хотите улучшить его. Простой способ — ручное безградиентное тюнинга, но он плохо масштабируется. Использование методов обучения с подкреплением требует наличия сигналов награды и достаточного количества роллоутов для тренировки. Обратная распространение ошибок через стимулы невозможно — стимул представляет собой дискретную строку, а не дифференцируемый параметр.

Классическое био-Inspired оптимизация — PSO для непрерывных пространств поиска и ACO для выбора пути — было разработано именно для этого режима: без градиентов, на основе популяции, дешёвый на оценку. Сочетайте их с LLMs для шага поиска без градиентов, и вы получите удивительно практический оптимизатор.

Те же паттерны применимы к агенту *ротации* в многоагентных системах. Траектория феромона стиля ACO登记记录哪个代理在何种任务类型上表现最佳，让路由器利用这条轨迹，并通过衰减 feromone 使得路径可以重新发现。 

请注意，这里的“pheromone”一词在俄文中通常没有直接对应的术语，因此我使用了“феромон”来表示，并且根据上下文进行了适当的调整。同时，“routing”在这里翻译为“ротации”，但更准确的翻译可能是“ротации代理”，以更好地反映其功能和角色。

## Концепция

Обновление по PSO (Кеннеди и Эберхарт 1995)

Пarticle Swarm Optimization: популяция частиц в непрерывном пространстве поиска. Каждая частьца имеет позицию \( x_i \) и скорость \( v_i \). Каждый итерационный шаг:

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

Где `p_best` — это лучший результат частицы самой себя, `g_best` — это лучший результат всего стайла (сwarма), `w`, `c1`, `c2` — это коэффициенты инерции, когнитивного и социального весов соответственно, а `r1`, `r2` — случайные факторы.

### ПСО на основе выводов ЛЛМ — LMPSO

Где:
- PSO — частицовый метод оптимизации (Particle Swarm Optimization)
- LLM —大型语言模型 (обозначение для крупных языковых моделей, такое же, как LLM в английском)
- LMPSO — возможно, это сокращение

arXiv:2504.09247 адаптирует PSO для структурированных выходных данных, сгенерированных LLM (математические выражения, программы). Каждая частица является кандидатом на выход. Скорость представляет собой *пушку*, которая описывает, как изменить текущий выход в направлении личного/глобального лучшего результата. LLM генерирует новый выход на основе пушки-предложения. "Инертность" скорости представляет собой предложение типа "сделайте небольшие инкрементальные изменения."

Это работает хорошо в случаях:
- Когда выход структурированный (парсимый, оцениваемый).
- Когда фитнес автоматический (тестовые запуски, арифметическое оценка).
- Когда популяция мала (~10-30 частиц), так что общее количество вызовов LLM управляемо.

Недостаточно эффективно, когда тренировка требует человеческого контроля — стоимость на каждый итерационный шаг становится недопустимой.

### Модельные стаи

arXiv:2410.11163 удаляет PSO с выходного слоя и перемещает его в *модельный* слой. Каждая "частица" является экспертом LLM (параметры). Сwarm перемещает параметры к коллективному лучшему значению без использования градиентного обновления. Отчет: средний прирост 13,3% над 12 базовыми линиями на 9 наборах данных с только 200 экземплярами на итерацию.

Ключевая мысль заключается в том, что экспертизные модели LLM уже находятся рядом в общей многообразии параметров (адаптерные веса, LoRA дельты). Поиск оптимальных решений с помощью метода PSO на этом низкоразмерном подпространстве дешёв и эффективен.

Обновление по АКО (Дориго 1992)

Алгоритм оптимизации колонией муравьев: муравьи перемещаются по графу; каждый путь имеет след феромона. Вероятность движения муравья зависит от силы следа феромона. Муравьи, завершившие задачу, выделяют феромон пропорционально качеству решения. Феромон разрушается со временем.

AMRO-S — Агентскоеrouting с использованием кошелька-агентов (ACO)

arXiv:2603.12933 использует АКТС для многоагентного маршрутизирования. Каждый тип задачи является "назначением"; каждый агент — возможным маршрутом. Феромоны укрепляют маршруты, которые дают хорошие результаты. Основные вклады:

- **Интерпретируемое доказательство маршрутизации.** Сила феромонов является человеческим читаемым сигналом.
- **Качество-ограниченное асинхронное обновление.** Феромоны обновляются только после того, как пройдут проверки качества, что разделяет инференцию от обучения.
- **Ускорение в 4,7 раза** на бенчмарке многороботного маршрутизации.

Качественный контроль важен: без него быстрые, но неправильные агенты накапливают феромоны, и система застревает на плохих маршрутах.

**Не используйте ни то, ни другое в следующих случаях:**
- Фитнес требует человеческого контроля (слишком дорого по каждому циклу).
- Пространство поиска дискретно и комбинаторно таким образом, что PSO не покрывает его (в таком случае используйте генетические алгоритмы вместо этого).
- Временные решения требуют строгой задержки (PSO/ACO сходятся медленнее по сравнению с однопроходными онуристическими методами).

Почему биоинспирированные методы все еще побеждают:

Градиентные методы требуют дифференцируемые сигналы. Выходы LLM и решения маршрутизации не являются тривиально дифференцируемыми. Методы с псевдо-градиентами (обученные по принципу репримандов агенты, тюнеры запросов в стиле DPO) работают, но требуют дорогостоящего обучения.

PSO и ACO требуют только функцию *оценки*. Если вы можете оценить кандидатское решение или решение маршрута, вы можете оптимизировать пространство. Это делает порог применения значительно更低一些。

翻译为：

PSO 和 ACO 只需要一个 *评估* 函数。如果你能够对候选输出或路由决策进行评分，你就可以在该空间内进行优化。这使得适用性门槛大大降低。

Практические ограничения

- **Бюджет населения:** N частиц × T итераций × стоимость оценки. Для оценок LLM при ~$0,02 за вызов, работа 20-частичной PSO в течение 50 итераций обойдется примерно в $20. Планируйте соответственно.
- **Исследование vs эксплуатация:** Скорость уменьшения феромонов и инертность PSO взаимозаменяемы; слишком быстрое уменьшение → забывает решения; слишком медленное → застревает на ранних локальных оптимумах.
- **Катастрофическое отклонение:** Оба алгоритма могут сначала сойтись, а затем разойтись при изменении ландшафта функциональной зависимости (новая распределенная данные). Следите за стабильностью наилучшей функциональной зависимости.

## Собери его

`файл code/main.py реализует:`

- `LMPSO` — PSO над числовыми параметрами запроса (температура, веса top_k). Генерация "LLM" каждой частицы моделируется как сценарий функции фитнесса. Запускает алгоритм на 30 итераций и показывает сходимость g_best.
- `AMRO_S` — маршрутизация в стиле ACO. 3 агента, 4 типа задач, матрица феромонов, 100 прорouted задач. Выводит распределение (тип задачи → выборы агентов) со временем для демонстрации формирования следа.
- Сравнение: случайная маршрутизация vs маршрутизация ACO на один и тот же поток задач. Измеряет качество и задержку.

Запустить:

```
python3 code/main.py
```

Ожидаемый результат:
- LMPSO: наилучшая фитнесс-оценка g_best улучшается от случайного значения до приближенного оптимального за 30 итераций.
- AMRO-S: таблица феромонов стабилизируется на правильном агенте для каждого типа задачи; маршрутное планирование蚁群优化算法（ACO）的质量优于随机选择约30-40%，同时还能减少延迟（重试次数较少）。 

请注意，这里的“AMRO-S”和“LMPSO”是技术术语或特定算法的缩写，在翻译时保持了原样。如果有具体的含义或者标准翻译方式，请告知以便进一步调整。

## Отправьте его

- **Начните с малого.** 10-20 частиц, 20-50 итераций. Только при увеличении масштаба, если кривая сходимости показывает явный прогресс.
- **Записывайте феромоны или g_best на каждой итерации.** Отладка оптимизаторов стай без следа очень затруднительна.
- **Обновление порога качества.** Особенно для маршрутизации ACO: быстро и неправильно действующие агенты не должны накапливать феромоны.
- **Сброс уменьшения при смещении распределения.** Когда изменяется распределение оценки, старые феромоны устарели; временно сбросьте или удвойте скорость уменьшения.
- **Ограничьте стоимость на итерацию.** Выведите метрику стоимости на итерацию. PSO, который стоит $500 за итерацию и при этом дает прирост в 0,5%, не может быть выпущен.
