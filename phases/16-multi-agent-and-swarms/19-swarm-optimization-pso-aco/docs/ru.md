# Роевая оптимизация для LLM (PSO, ACO)

> Био-вдохновлённая оптимизация переживает возвращение к LLM. **LMPSO** (arXiv:2504.09247) использует PSO, где скорость каждой частицы — это промпт, а LLM генерирует следующего кандидата; хорошо работает для структурированных последовательных выходных данных (математические выражения, программы). **Model Swarms** (arXiv:2410.11163) рассматривает каждого LLM-эксперта как частицу PSO на многообразии весов модели и сообщает о **13.3% среднем приросте** над 12 бейзлайнами на 9 датасетах всего с 200 экземплярами. **SwarmPrompt** (ICAART 2025) гибридизирует PSO + Серого волка для оптимизации промптов. **AMRO-S** (arXiv:2603.12933) — ACO-вдохновлённые феромонные специалисты для мультиагентного маршрутизирования LLM — **4.7× ускорение**, интерпретируемые доказательства маршрутизации, обновление с качественным порогом, которое отделяет инференс от обучения. Этот урок реализует PSO в пространстве параметров промпта и ACO для маршрутизации агентов, измеряет, почему эти классические алгоритмы подходят для эпохи LLM, и когда не подходят.

**Тип:** Изучение + Практическая работа
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 16 · 09 (Параллельные сети роёв), Фаза 16 · 14 (Консенсус и BFT)
**Время:** ~75 минут

## Проблема

У вас есть промпт, набирающий 62% на вашей оценке задачи. Вы хотите его улучшить. Наивный ход — ручная настройка без градиентов, которая масштабируется плохо. Обучение с подкреплением требует сигналов вознаграждения и достаточного количества прогонов для обучения. Обратное распространение через промпты невозможно — промпт — это дискретная строка, а не дифференцируемый параметр.

Классическая био-вдохновлённая оптимизация — PSO для непрерывных пространств поиска, ACO для выбора путей — была разработана именно для этого режима: без градиентов, на основе популяции, дешёвая за оценку. Сочетайте их с LLM для шага поиска без градиентов, и получите удивительно практичный оптимизатор.

Те же паттерны применимы к маршрутизации агентов в мультиагентных системах. Феромонный след в стиле ACO записывает, какой агент работал лучше всего для какого типа задач, позволяет маршрутизатору использовать след и затухает, чтобы маршруты могли быть переоткрыты.

## Концепция

### Напоминание PSO (Kennedy & Eberhart 1995)

Оптимизация роем частиц: популяция частиц в непрерывном пространстве поиска. Каждая частица имеет позицию `x_i` и скорость `v_i`. Каждая итерация:

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
оценить приспособленность(x_i)
обновить p_best_i если улучшилось
обновить g_best если глобальный лучший
```

Где `p_best` — собственный лучший результат частицы, `g_best` — лучший результат роя, `w, c1, c2` — веса инерции + когнитивный + социальный, `r1, r2` — случайные факторы.

### PSO на выходах LLM — LMPSO

arXiv:2504.09247 адаптирует PSO для структурированных выходных данных, генерируемых LLM (математические выражения, программы). Каждая частица — кандидатный выход. Скорость — это *промпт*, описывающий, как изменить текущий выход в сторону личного/глобального лучшего. LLM генерирует новый выход из промпта скорости. «Инерция» скорости — это промпт вроде «делай небольшие постепенные изменения».

Это хорошо работает когда:
- Выход структурированный (парсируемый, оцениваемый).
- Приспособленность автоматическая (тесты, арифметическая оценка).
- Популяция маленькая (~10-30 частиц), чтобы общее число вызовов LLM оставалось управляемым.

Это плохо работает когда приспособленность требует контроля человеком — стоимость за итерацию становится запретительной.

### Model Swarms

arXiv:2410.11163 берёт PSO с выходного слоя и переносит на *модельный* слой. Каждая «частица» — экспертный LLM (параметры). Рой перемещает параметры к коллективному лучшему через обновление без градиентов. Сообщается: 13.3% средний прирост над 12 бейзлайнами на 9 датасетах, всего с 200 экземплярами за итерацию.

Ключевой инсайт: экспертные модели LLM уже находятся рядом на общем параметрическом многообразии (веса адаптеров, дельты LoRA). PSO на этом низкоразмерном подпространстве дешёв и эффективен.

### Напоминание ACO (Dorigo 1992)

Оптимизация муравьиной колонией: муравьи проходят граф; каждый путь имеет феромонный след. Вероятности перемещения муравьёв взвешены по силе феромона. Муравьи, завершившие задачу, deposit феромон пропорционально качеству решения. Феромон затухает со временем.

### AMRO-S — ACO для маршрутизации агентов

arXiv:2603.12933 использует ACO для мультиагентного маршрутизирования. Каждый тип задачи — «назначение»; каждый агент — возможный маршрут. Феромоны укрепляют маршруты, дающие хорошие результаты. Ключевые вклады:

- **Интерпретируемые доказательства маршрутизации.** Сила феромона — сигнал, читаемый человеком.
- **Обновление с качественным порогом.** Феромоны обновляются только после прохождения проверки качества, отделяя инференс от обучения.
- **4.7× ускорение** на бенчмарке мультиагентного маршрутизирования.

Качественный порог важен: без него быстрые, но неправильные агенты накапливают феромон, и система закрепляется на плохих маршрутах.

### Когда использовать PSO / ACO для LLM

**Используйте PSO когда:**
- Пространство поиска непрерывное или отображается на непрерывные параметры (эмбеддинги промптов, веса LoRA, числовые параметры генерации).
- Приспособленность дешёвая и автоматическая.
- Популяция может быть маленькой (10-30).

**Используйте ACO когда:**
- У вас есть задача маршрутизации или выбора пути.
- Решения закрепляются со временем (одни и те же типы задач возвращаются).
- Нужны интерпретируемые доказательства для решений маршрутизации.

**Не используйте ни одно когда:**
- Приспособленность требует контроля человеком (слишком дорого за итерацию).
- Пространство поиска дискретное и комбинаторное способом, который PSO не покрывает (используйте генетические алгоритмы).
- Решения в реальном времени требуют строгой латентности (PSO/ACO сходятся медленно по сравнению с однопроходными эвристиками).

### Почему био-вдохновлённое всё ещё выигрывает

Методы на основе градиентов требуют дифференцируемых сигналов. Выходы LLM и решения маршрутизации не являются тривиально дифференцируемыми. Псевдоградиентные методы (маршрутизаторы на обучении с подкреплением, тюнеры промптов в стиле DPO) работают, но требуют дорогого обучения.

PSO и ACO требуют только функцию *оценщика*. Если вы можете оценить кандидатный выход или решение маршрутизации, вы можете оптимизировать по пространству. Это делает порог применимости гораздо ниже.

### Практические ограничения

- **Бюджет популяции.** N частиц × T итераций × стоимость за оценку. Для оценок LLM по ~$0.02/вызов, PSO из 20 частиц на 50 итерациях стоит ~$20. Планируйте соответственно.
- **Исследование vs эксплуатация.** Скорость затухания феромона и инерция PSO компенсируют друг друга; слишком быстрое затухание → забытые решения; слишком медленное → застревание на ранних локальных оптимумах.
- **Катастрофический дрейф.** Оба алгоритма могут сходиться, а затем расходиться, если ландшафт приспособленности сдвигается (новое распределение данных). Отслеживайте стабильность лучшей приспособленности.

## Реализуем

`code/main.py` реализует:

- `LMPSO` — PSO по числовым параметрам промпта (температура, веса top_k). «Генерация LLM» каждой частицы имитируется как сценарная функция приспособленности. Запускает алгоритм на 30 итерациях и показывает сходимость g_best.
- `AMRO_S` — маршрутизация в стиле ACO. 3 агента, 4 типа задач, феромонная матрица, 100 маршрутизированных задач. Печатает распределение (тип задачи → выбор агентов) по времени для показа формирования следа.
- Сравнение: случайная маршрутизация vs ACO-маршрутизация на том же потоке задач. Измеряет качество и латентность.

Запуск:

```
python3 code/main.py
```

Ожидаемый вывод:
- LMPSO: приспособленность g_best улучшается от случайного до почти оптимального за 30 итераций.
- AMRO-S: феромонная таблица стабилизируется на правильном агенте для каждого типа задач; ACO-маршрутизация обыгрывает случайную на ~30-40% по качеству и снижает латентность (меньше повторов).

## Применение

`outputs/skill-swarm-optimizer.md` помогает выбрать между PSO, ACO, генетическими алгоритмами и градиентными оптимизаторами для задач оптимизации LLM/агентов.

## Результат

- **Начинайте маленьким.** 10-20 частиц, 20-50 итераций. Масштабируйте только если кривая сходимости показывает явный прирост.
- **Логируйте феромоны или g_best за итерацию.** Отладка роевых оптимизаторов без следа болезненна.
- **Обновления с качественным порогом.** Особенно для ACO-маршрутизации: быстрые, но неправильные агенты не должны накапливать феромон.
- **Сбрасывайте затухание при сдвиге распределения.** Когда распределение оценок меняется, старые феромоны устаревают; сбросьте или удвойте скорость затухания временно.
- **Ограничьте стоимость за итерацию.** Выводите метрику стоимости-за-итерацию. PSO за $500/итерацию с приростом 0.5% не выпускается.

## Упражнения

1. Запустите `code/main.py`. Наблюдайте сходимость LMPSO. Варьируйте размер популяции 5, 10, 20, 50. При каком размере время сходимости насыщается?
2. Реализуйте эксперимент «катастрофический дрейф»: после 30-й итерации измените функцию приспособленности. Как быстро PSO адаптируется? Помогает ли сброс `p_best`?
3. Добавьте качественный порог в AMRO-S: феромон deposit только для запусков с оценкой > 0.7. Как это меняет сходимость по сравнению с версией без порога?
4. Прочитайте LMPSO (arXiv:2504.09247). Отобразите «скорость как промпт» статьи обратно на вашу числовую скорость. Что теряется в симуляции и что сохраняется?
5. Прочитайте AMRO-S (arXiv:2603.12933). Реализуйте разделённый «быстрый путь инференса» с асинхронным обновлением феромонов. Как это меняет латентность системы при постоянной нагрузке?
