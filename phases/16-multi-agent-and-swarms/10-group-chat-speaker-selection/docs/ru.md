# Групповой чат и выбор выступающего

> AutoGen GroupChat и AG2 GroupChat разделяют один разговор между N агентами; функция выбора (LLM, round-robin или кастомная) выбирает, кто говорит следующим. Это архетип эмерджентного мультиагентного разговора — агенты не знают свою роль в статическом графе, они просто реагируют на разделяемый пул. Семантика GroupChat из AutoGen v0.2 была сохранена в форке AG2; AutoGen v0.4 переписал её как event-driven модель акторов. Microsoft перевела AutoGen в режим обслуживания в феврале 2026 и объединила с Semantic Kernel в Microsoft Agent Framework (RC февраль 2026). Примитив GroupChat выживает и в AG2, и в Microsoft Agent Framework — выучите один раз, используйте везде.

**Тип:** Изучение + Практическая работа
**Языки:** Python (stdlib)
**Предварительные требования:** Фаза 16 · 04 (Примитивная модель)
**Время:** ~60 минут

## Проблема

Статические графы (LangGraph) хороши, когда рабочий процесс известен. Реальные разговоры не статичны: иногда программист спрашивает ревьюера, иногда исследователя, иногда писателя. Хардкод каждого возможного handoff порождает экспоненциальный рост рёбер. Вы хотите *агентов, реагирующих на разделяемый пул*, с некоторой функцией, решающей, кто говорит следующим.

Это именно то, что делает AutoGen GroupChat.

## Концепция

### Форма

```
              ┌─── разделяемый пул ────┐
              │   m1  m2  m3  ...  │
              └─────────┬──────────┘
                        │ (все читают всё)
      ┌───────┬─────────┼─────────┬───────┐
      ▼       ▼         ▼         ▼       ▼
    Агент A  Агент B  Агент C  Агент D  Выборщик
                                           │
                                           ▼
                                  "следующий выступающий = C"
```

Каждый агент видит каждое сообщение. На каждом ходу вызывается функция выбора, чтобы выбрать, кто говорит следующим.

### Три варианта выборщика

**Round-robin.** Фиксированный цикл. Детерминированный. Масштабируется линейно по N, но игнорирует контекст — программист получает ход, даже если тема — юридическая проверка.

**Выбор LLM.** Вызов LLM, который читает недавний пул и возвращает лучшего следующего выступающего. Учитывает контекст, но медленно: каждый ход добавляет вызов LLM. Дефолт AutoGen.

**Кастомный.** Python-функция с любой логикой. Типично: выбор LLM с правилами fallback (например, «всегда давать верификатору ход после программиста»).

### API ConversableAgent

```
agent = ConversableAgent(
    name="coder",
    system_message="You write Python.",
    llm_config={...},
)
chat = GroupChat(agents=[coder, reviewer, tester], messages=[])
manager = GroupChatManager(groupchat=chat, llm_config={...})
```

`GroupChatManager` хранит выборщика. Когда агент завершает ход, менеджер вызывает выборщика, который возвращает следующего агента. Цикл продолжается до условия завершения.

### Завершение

Три типичных паттерна:

- **Максимум раундов.** Жёсткий лимит на общее количество ходов.
- **Токен «TERMINATE».** Агенты могут генерировать sentinel-сообщение; менеджер останавливается при его появлении.
- **Проверка достижения цели.** Лёгкий верификатор запускается каждый ход и останавливает чат при завершении.

### Раздел AutoGen → AG2 и объединение Microsoft Agent Framework

В начале 2025 года Microsoft начала крупную переписку AutoGen (v0.4) вокруг event-driven модели акторов. Сообщество форкнуло семантику GroupChat из AutoGen v0.2 как AG2, сохранив API, который интегрировали ранние пользователи.

В феврале 2026 Microsoft объявила, что AutoGen перейдёт в режим обслуживания, а event-driven модель акторов объединится с **Microsoft Agent Framework** (RC февраль 2026, теперь объединён с Semantic Kernel). Концепция GroupChat выживает в обоих потоках; детали реализации различаются. AG2 — предпочтённый апстрим для кода, совместимого с v0.2.

### Когда GroupChat подходит

- **Эмерджентные разговоры.** Вы не хотите предварительно прописывать каждого возможного следующего выступающего.
- **Задачи со смешением ролей.** Программист спрашивает исследователя, исследователь — архивариуса, архивариус — обратно программиста. Поток — не DAG.
- **Исследовательское решение задач.** Думайте «мозговой штурм», а не «конвейер».

### Когда не работает

- **Строгий детерминизм.** Выборщик LLM может быть неконсистентным. Один промпт, разные запуски, разные следующие выступающие.
- **Каскады лести.** Агенты уступают тому, кто говорил 가장 уверенно. Противодействуйте явным промптом.
- **Раздувание контекста.** Каждый агент читает каждое сообщение; после 10 ходов контекст огромный. Используйте проекции (Урок 15) для ограничения областей.
- **Горячие выступающие.** Один агент доминирует в разговоре, потому что выборщик предпочитает его специальности. Введите балансировку выступающих как функцию выборщика.

### Групповой чат vs супервайзер

Одни примитивы, другие дефолты:

- Супервайзер: один агент планирует, другие выполняют. Выборщик — «спросите планировщика, что делать».
- Групповой чат: все агенты — пиры; выборщик — функция над разделяемым пулом.

Оба используют четыре примитива из Урока 04. Групповой чат по умолчанию использует оркестрацию с выбором LLM и разделяемое состояние полного пула.

## Результат

Чеклист:

- **Лимит максимума раундов.** Всегда. 10-20 для типичных задач.
- **Метрика баланса выступающих.** Отслеживайте ходы на агента; оповещайте при превышении порога дисбаланса.
- **Токен завершения.** `TERMINATE` или специализированный агент-верификатор.
- **Проекция или ограниченная память.** После ~10 сообщений рассмотрите предоставление каждому агенту только ограниченного обзора для предотвращения раздувания контекста.
- **Логирование выборщика.** Для вариантов с выбором LLM логируйте и вход, и выбор выборщика. Иначе отладка невозможна.

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|------|----------------|------------------------|
| GroupChat | «Агенты в одном чат-руме» | Разделяемый пул сообщений + функция выбора. Примитив AutoGen / AG2. |
| Выбор выступающего | «Кто говорит следующим» | Функция, выбирающая следующего агента. Round-robin, выбор LLM или кастомная. |
| GroupChatManager | «Ведущий встречи» | Компонент AutoGen, владеющий выборщиком и циклом ходов. |
| ConversableAgent | «Базовый агент» | Базовый класс AutoGen; агент, который может отправлять и получать сообщения. |
| Токен завершения | «Слово 'стоп'» | Sentinel-строка (обычно `TERMINATE`), завершающая чат. |
| Горячий выступающий | «Один агент доминирует» | Режим отказа, когда выборщик постоянно выбирает одного агента. |
| Раздувание контекста | «Пул растёт бесконечно» | Каждый агент читает все предыдущие сообщения; контекст растёт с ходами. |
| Проекция | «Ограниченный обзор» | Ролевой обзор разделяемого пула для предотвращения раздувания контекста. |
