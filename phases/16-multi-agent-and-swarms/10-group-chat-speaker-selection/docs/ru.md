# Групповой чат и выбор докладчика

> AutoGen GroupChat и AG2 GroupChat делят одну беседу между N агентами; функция выбора (LLM, по кругу или пользовательская) определяет, кто будет говорить дальше. Это типичный пример эмерджентного многоагентного разговора — агенты не знают свою роль в статической графовой структуре, они просто реагируют на общую池鱼不能了解池外的天地，正如这些自动生成的群聊机制，它们在N个代理之间共享一个对话；选择者函数（基于LLM、轮询或自定义）决定下一个发言的人。这代表了涌现多代理对话的典型模式——代理不知道自己在静态图中的角色，他们只是对共享池作出反应。AutoGen v0.2的群聊语义被保留到了AG2分支中；而AutoGen v0.4将其重写为事件驱动的行为者模型。微软于2026年2月将AutoGen置于维护模式，并将其与Semantic Kernel合并进Microsoft Agent Framework（RC 2026年2月）。群聊原语在AG2和Microsoft Agent Framework中都得以保留——学会一次，随处可用。

## Проблема

Статические графы (LangGraph) великолепны при известном потоке работы. Реальные разговоры не статичны: иногда программист обращается к проверяющему, иногда исследователю, иногда автору. Жесткое закодирование каждого возможного переключения приводит к взрыву ребер. ## Концепция

Форма

```
              ┌─── shared pool ────┐
              │   m1  m2  m3  ...  │
              └─────────┬──────────┘
                        │ (everyone reads all)
      ┌───────┬─────────┼─────────┬───────┐
      ▼       ▼         ▼         ▼       ▼
    Agent A  Agent B  Agent C  Agent D  Selector
                                           │
                                           ▼
                                  "next speaker = C"
```

Каждый агент видит каждое сообщение. Функция селектора вызывается на каждом ходу, чтобы выбрать, кто будет говорить следующим.

Три варианта селекторов:

**Ротация.** Фиксированный цикл. Определённый. Линейно масштабируется по N, но игнорирует контекст — программист получает право голоса даже при обсуждении вопросов юридического характера.

**Выбрано LLM.** Вызов LLM, который читает свежую пулл и возвращает лучшего следующего говорящего. Контекстуально осведомленный, но медленный: каждый ход добавляет вызов LLM. По умолчанию AutoGen использует это.

**Пользовательский код.** Функция на языке Python с логикой, которая вам нужна. Обычно она выбирается генератором LLM с резервными правилами (например, "всегда передавать проверяющему ход после разработчика").

API КонversableAgent

```
agent = ConversableAgent(
    name="coder",
    system_message="You write Python.",
    llm_config={...},
)
chat = GroupChat(agents=[coder, reviewer, tester], messages=[])
manager = GroupChatManager(groupchat=chat, llm_config={...})
```

`GroupChatManager` хранит селектор. Когда агент завершает свой ход, менеджер вызывает селектор, который возвращает следующего агента. Цикл продолжается до тех пор, пока не будет достигнута условие завершения.

- **Максимальное количество раундов.** Жесткий лимит на общее число ходов.
- **Токен "TERMINATE".** Агенты могут выдавать сигнал остановки; менеджер прекращает работу при появлении этого сигнала.

Концепция GroupChat сохраняется на обоих треках; детали реализации различаются. AG2 является предпочтительной верхней строкой для кода совместимого с v0.2.

Когда подходит GroupChat:

- **Эмерджентные беседы.** Вы не хотите заранее спланировать каждого возможного следующего говорящего.
- **Задачи с переплетенными ролями.** Разработчик обращается к исследователю, исследователь обращается к архивариусу, архиварь обращается обратно к разработчику. Поток не является DAG.
- **Эксплораторное решение задач.** Думайте о "мозговой штурм会议上," а не об "асsembly линии."

## Отправляем!

### Список проверки:

- **Ограничение раундов.** Всегда. 10-20 для типовых задач.
- **Метрика баланса говорящего.** Следить за количеством ходов каждого агента; предупреждать, когда несоответствие превышает порог.
- **Токен завершения.** `TERMINATE` или специальный валидаторный агент.
- **Проекция или ограниченное хранилище памяти.** После ~10 сообщений рассмотреть возможность предоставления каждому агенту только ограниченного представления для предотвращения перенасыщения контекстом.
- **Логирование выборчного селектора.** Для вариантов, выбранных LLM, логировать как вход селектора, так и его выбор. В противном случае отладка невозможна.

## Ключевые термины

| Термин | Что говорят люди | На самом деле означает |
|--------|------------------|-----------------------|
| GroupChat | "Агенты в одной чат-рум" | Общий пул сообщений + функция выбора. Примитив AutoGen / AG2. |
| Выбор речевого агента | "Кто будет говорить следующий" | Функция, которая выбирает следующего агента. В порядке обмена, выбранный LLM или пользовательский вариант. |
| GroupChatManager | "Хост встречи" | Компонент AutoGen, который владеет функцией выбора и перебирает ходы. |
| ConversableAgent | "Базовый агент" | Базовый класс AutoGen; агент, способный отправлять и получать сообщения. |
| Токен завершения | "Слово 'стоп'" | Сentinel строка (обычно `TERMINATE`), которая заканчивает чат. |
| Горячий речевой агент | "Один агент доминирует" | Фаилу, когда функция выбора постоянно выбирает один и тот же агент. |
| Контекстный бloat | "Пул растет безгранично" | Каждый агент читает все предыдущие сообщения; контекст увеличивается с каждым ходом. |
| Проекция | "Ограниченное представление" | Роль-специфическое представление в общем пуле, чтобы избежать роста контекста. |
