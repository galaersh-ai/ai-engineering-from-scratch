# Передачи и Ритуалы — Безсостоятельное Оркестрирование

> Сwarm OpenAI (октябрь 2024) свели многомодульное управление до двух примитивов: **рoutines** (инструкции + инструменты как системный пуштун) и **handoffs** (инструмент, возвращающий другого Агента). Нет машин состояний, нетDSL с ветвлениями — LLM роутится путем вызова правильного инструмента handoff. SDK OpenAI Agents (март 2025) является производственной версией преемника. Сам Swarm остается самым чистым концептуальным отражением — его весь исходный код помещается в несколько сотен строк. Паттерн вирусоподобный из-за того, что поверхность API примерно равна "агент = пуштун + инструменты; handoff = функция, возвращающая агента." Ограничение: без состояния, поэтому память — проблема зовущего.

**Тип:** Учиться + Строить

**Предварительные знания:** Фаза 16 · 04 (Примитивная модель)

Сwarм движется в противоположном направлении: используйте способность модели вызывать инструменты, которая уже есть. Передачи становятся вызовами инструментов. Оркестратором выступает тот агент, который в данный момент ведет разговор. Машину состояний неявно содержащую в системных подсказках агентов.

## Концепция

### Две примитивные структуры данных

**Рутинная работа.** Системный пуш, определяющий роль агента и доступные инструменты. Представьте себе это как набор инструкций с ограниченной областью действия: "вы трайж-агент; если пользователь спрашивает о возвратах, передавайте запрос агенту по вопросам возвратов."

**Передача.** Инструмент, которым может воспользоваться агент, возвращая новый объект Агента. Время выполнения Swarm обнаруживает значение возвращаемого объекта Агента и переключает активного агента на следующий ход.

Это вся абстракция.

```
def transfer_to_refunds():
    return refund_agent  # Swarm sees Agent return → switch active agent

```python
triage_agent = Агент(
    имя="диагностика",
    инструкции="Распределите пользователя к правильному специалисту.",
    функции=[перенаправить_на_возвраты, перенаправить_на_продажи, перенаправить_на_поддержку],
)
```

Системный пуш агента триажа позволяет ему выбирать правильную передачу в зависимости от сообщения пользователя. Редирект выполняется с помощью вызова инструментов ЛЛМ (системы обработки естественного языка).

Почему это вирусное?

- **Маленький API.** Два концепта для изучения.
- **Использует то, что уже делает модель.** Инструменты вызова уже являются производственными по качеству у всех провайдеров.
- **Нет бремени машины состояний.** Вы не описываете граф; запросы агентов описывают, кому они передают управление.

Бесправная торговля

Сwarm явно не сохраняет состояние между запусками. Фреймворк поддерживает историю сообщений во время выполнения, но ничего не сохраняет на постоянной основе. Память, непрерывность, долгосрочные задачи — все это проблема вызывающего程序，请您用俄语翻译以下内容：

Swarm是明确的无状态，在运行之间不保存任何数据。框架在一次运行中会保留消息历史，但不会持久化任何东西。内存、连续性、长期任务——这些都是调用者的责任。

В производстве (SDK OpenAI Agents, март 2025 года) это было одной из ключевых изменений: SDK добавляет встроенное управление сессиями, ограждающие меры и отслеживание, сохраняя при этом примитив передачи.

Когда Swarm/переключения подходят:

- **Триaging паттерны.** Передний оператор направляет пользователя к специалисту.
- **Навыкозаходные передачи.** "Если задача требует кодирования, звоните программисту; если нужна исследовательская работа, звоните исследователю."
- **Короткие ограниченные разговоры.** Поддержка клиентов, вопросы и ответы — обращения в тикеты, простые рабочие процессы.

Когда Сwarm сражается, можно сказать:

### Когда Сwarm борется,

- **Долгие сессии с общей памятью.** Передача перезагружает состояние разговора в новый запрос агента вместе с историей. Без управляющего хранилищем звонящего нет постоянного состояния между агентами.
- **Параллельное выполнение.** Передача происходит по одному — активный агент меняется. Параллелизм требует координации со стороны звонящего нескольких запусков Swarm.
- **Аудит и воспроизведение.** Безостоятельные запуски трудно повторить точно; выбор передачи LLM не определен заранее.

Продукция преемник добавляет:

- **Состояние сессии.** Постоянная нить между запусками.
- **Барьеры.** Hooks для валидации входных и выходных данных.

- **Фильтры передачи контекста.** Управление тем, что передается при передаче контекста.

### Сwarm против GroupChat

Если вам нужна более подробная информация или контекст для этого сравнения, дайте мне знать!

Оба используют маршрутизацию на основе LLM, но они различаются в том, кто выбирает следующий шаг:

- Групповой чат: селектор (функция или ИИ-система) выбирает следующего говорящего извне.
- Сwarm: текущий агент выбирает своего преемника, вызывая инструмент передачи.

Колония — это когда "агент решает, что делать дальше"; Групповой чат — это когда "менеджер решает, что делать дальше." Решение в колонии хранится в активном вызове инструмента агента; в групповом чате оно хранится в `GroupChatManager`.

## Отправьте его

### Список проверки:

- **Логирование передачи:** при каждой передаче записывается событие трассировки с информацией о передающем агенте, получателе и снимке контекста.
- **Правила передачи контекста:** решите, что будет передаваться при передаче: полная история (дорогостоящая), последние N сообщений или краткий обзор.
- **Ограничение на передачу:** передача к специалисту с другими правами инструментов должна быть аутентифицирована — иначе можно заставить произойти нежелательную передачу при помощи введения команд.
- **Обнаружение циклов:** передача между двумя агентами взад-вперед является распространенной ошибкой; обнаруживается с помощью простого проверочного кольца последних K элементов.
- **Запасной агент:** если целевой агент для передачи не существует, перейти к безопасному по умолчанию агенту.

## Ключевые термины

| Термин | Что говорят люди | На самом деле означает |
|--------|------------------|----------------------|
| Рутинная операция | "Агентный запрос" | Системный запрос + список инструментов. Определяет роль и доступные передачи. |
| Передача | "Перевод на другого агента" | Инструмент активного агента, который вызывает нового Агента. Время выполнения меняет активного агента. |
| Без состояния | "Нет памяти между запусками" | Сwarm ничего не сохраняет; память отвечает за пользователя. |
| Активный агент | "Кто сейчас говорит" | Агент, который ведет текущую беседу. Передача меняет это. |
| Перенос контекста | "Что передается при передаче" | Политика того, что истории видит входящий агент: полная, последние N или сокращенная. |
| Цикл передачи | "Агенты переключаются" | Фаиловый режим, когда два агента постоянно передают друг другу. |
| SDK OpenAI Agents | "Производственная Swarm" | Предшественник марта 2025 года; добавляет сессии, ограждающие элементы и отслеживание на основе примитива передачи. |
| Фильтр передачи | "Ворота для передачи" | Функция SDK для проверки и изменения контекста при передаче. |
