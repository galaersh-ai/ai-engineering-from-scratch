# Edge-инференс — Apple Neural Engine, Qualcomm Hexagon, WebGPU/WebLLM, Jetson

> Ключевое ограничение edge-устройств — пропускная способность памяти, а не вычислительная мощность. Мобильная DRAM обеспечивает 50–90 ГБ/с; датацентр-ное HBM3 — 2–3 ТБ/с, разница составляет 30–50 раз. Декодирование привязано к пропускной способности памяти, поэтому этот разрыв является определяющим. В 2026 году ландшафт делится на четыре направления. Apple M4/A18 Neural Engine достигает 38 TOPS с объединённой памятью (без копирования между CPU и NPU). Qualcomm Snapdragon X Elite / 8 Gen 4 Hexagon — 45 TOPS. WebGPU + WebLLM запускает Llama 3.1 8B (Q4) со скоростью ~41 токен/с на M3 Max (примерно 70–80% от нативной производительности); 17,6 тыс. звёзд на GitHub, API совместимый с OpenAI, ~70–75% мобильного покрытия. NVIDIA Jetson Orin Nano Super (8 ГБ) вмещает Llama 3.2 3B / Phi-3; AGX Orin запускает gpt-oss-20b через vLLM со скоростью ~40 токен/с; Jetson T4000 (JetPack 7.1) в 2 раза быстрее AGX Orin. TensorRT Edge-LLM поддерживает EAGLE-3, NVFP4, чанковый префилл — продемонстрирован на CES 2026 компаниями Bosch, ThunderSoft, MediaTek.

**Тип:** Обучение
**Языки:** Python (стандартная библиотека, учебный симулятор декодирования с ограничением пропускной способности)

**Время:** ~60 минут

## Цели обучения

- Объяснить, почему мобильный инференс LLM ограничен пропускной способностью памяти, а вычислительная мощность имеет второстепенное значение.
- Перечислить четыре target-платформы (Apple ANE, Qualcomm Hexagon, WebGPU/WebLLM, NVIDIA Jetson) и сопоставить каждую с областью применения.
- Назвать проблему покрытия WebGPU в 2026 году (Firefox для Android приближается) и дату появления Safari iOS 26.
- Выбрать формат квантизации (quantization) для каждой платформы (Core ML INT4 + FP16 для ANE, QNN INT8/INT4 для Hexagon, WebGPU Q4 для браузера, NVFP4 для Jetson Thor).

## Проблема

Заказчик хочет автономный чат-бот: голосовой интерфейс, приватность по умолчанию, работа в офлайне. На MacBook Pro M3 Max Llama 3.1 8B Q4 работает со скоростью ~55 токен/с — это хорошо. На iPhone 16 Pro та же модель работает со скоростью 3 токен/с — это плохо. На средненобильном Android с Snapdragon 8 Gen 3 — 7 токен/с. В браузере через WebGPU на Chrome Android v121+ — 4–8 токен/с в зависимости от устройства.

Разброс пропускной способности — это не проблема портирования. Это разрыв пропускной способности памяти, умноженный на формат квантизации, умноженный на доступность NPU из пользовательского пространства. Edge-инференс в 2026 году — это четыре различных задачи с четырьмя различными решениями.

## Концепция

### Пропускная способность — настоящий потолок

Декодирование считывает полный набор весов для каждого токена. Одна 7B-модель в Q4 весит 3,5 ГБ. Чтение 3,5 ГБ со скоростью 50 ГБ/с занимает 70 мс — теоретический потолок ~14 токен/с. При 90 ГБ/с (мобильная DRAM верхнего сегмента) потолок сдвигается до ~25 токен/с. Никакое количество вычислений не поможет преодолеть эту границу.

Датацентрное HBM3 со скоростью 3 ТБ/с считывает те же 3,5 ГБ за 1,2 мс — потолок составляет 830 токен/с. Та же модель, те же веса. Другая подсистема памяти.

### Apple Neural Engine (M4 / A18)

- До 38 TOPS. Объединённая память (CPU и ANE используют один общий пул) — без накладных расходов на копирование.
- Доступ через Core ML + скомпилированные модели `.mlmodel` или через Metal Performance Shaders (MPS) в PyTorch.
- Llama.cpp Metal использует MPS, а не ANE напрямую; нативная работа с ANE требует конвертации через Core ML.
- Оптимальный путь для iOS-приложений в 2026 году: Core ML с весами INT4 и активациями FP16.

### Qualcomm Hexagon (Snapdragon X Elite / 8 Gen 4)

- До 45 TOPS. Интегрирован с CPU и GPU в SoC, но с отдельным доменом памяти.
- QNN (Qualcomm Neural Network) SDK и AI Hub обеспечивают конвертацию из PyTorch/ONNX.
- Шаблоны чатов, Llama 3.2, Phi-3 доступны как первоклассные артефакты на AI Hub.

### Intel / AMD NPU (Lunar Lake, Ryzen AI 300)

- 40–50 TOPS. Программное обеспечение отстаёт от Apple/Qualcomm; OpenVINO совершенствуется, но остаётся нишевым.
- Лучший вариант для Windows ARM copilot-приложений; нативная работа на десктопах AMD/Intel для локально-ориентированных сценариев.

### WebGPU + WebLLM

- Запуск моделей в браузере через compute-шейдеры WebGPU; без установки.
- Llama 3.1 8B Q4 со скоростью ~41 токен/с на M3 Max — примерно 70–80% от нативной производительности на том же бэкенде.
- 17,6 тыс. звёзд на GitHub для WebLLM; JS API совместимый с OpenAI; Apache 2.0.
- Покрытие в 2026 году: Chrome Android v121+, Safari iOS 26 GA, Firefox для Android всё ещё догоняет. Общее покрытие — ~70–75% мобильных устройств.

### Семейство NVIDIA Jetson

- Orin Nano Super (8 ГБ): вмещает Llama 3.2 3B, Phi-3 с хорошей скоростью.
- AGX Orin: запускает gpt-oss-20b через vLLM со скоростью ~40 токен/с.
- Thor / T4000 (JetPack 7.1): в 2 раза выше производительность, чем AGX Orin, поддержка EAGLE-3 и NVFP4.
- TensorRT Edge-LLM (2026) поддерживает спекулятивное декодирование EAGLE-3, веса NVFP4, чанковый префилл — оптимизации датацентров перенесены на edge-устройства.

### Выбор квантизации для каждой платформы

| Target | Формат | Примечания |
|--------|--------|------------|
| Apple ANE | Веса INT4 + активации FP16 | Путь конвертации через Core ML |
| Qualcomm Hexagon | QNN INT8 / INT4 | Конвертеры AI Hub |
| WebGPU / WebLLM | Q4 MLC (q4f16_1) | Используйте `mlc_llm convert_weight` + скомпилированный `.wasm`; GGUF не поддерживается |
| Jetson Orin Nano | Q4 GGUF или TRT-LLM INT4 | Ограничение пропускной способности памяти |
| Jetson AGX / Thor | NVFP4 + FP8 KV | Путь Edge-LLM |

### Ловушка длинного контекста на edge-устройствах

128K-контекст Llama 3.1 — это функция датацентра. На телефоне с 8 ГБ ОЗУ: 4 ГБ модель + 2 ГБ KV-кэш для 32K токенов + накладные расходы ОС = нехватка памяти. Edge-развёртывания ограничивают контекст диапазоном 4K–8K, если не применяется агрессивная квантизация KV (Q4 KV).

### Голос — ключевое приложение

Голосовые агенты чувствительны к латентности (первый токен — менее 500 мс). Локальный инференс полностью устраняет сетевую задержку. В сочетании с распознаванием речи (варианты Whisper Turbo работают на edge-устройствах) edge-инференс превращается в голосовой контур production-уровня.

### Числа, которые стоит запомнить

- Apple M4 / A18 ANE: 38 TOPS.
- Qualcomm Hexagon SD X Elite: 45 TOPS.
- WebLLM M3 Max: ~41 токен/с на Llama 3.1 8B Q4.
- AGX Orin: ~40 токен/с на gpt-oss-20b через vLLM.
- Разрыв пропускной способности между датацентром и edge: 30–50 раз.
- Покрытие WebGPU на мобильных устройствах: ~70–75% (Firefox для Android отстаёт).

1. Запустите `code/main.py`. Для 7B-модели в Q4 на Snapdragon 8 Gen 3 (~77 ГБ/с пропускная способность) вычислите потолок декодирования. Сравните с наблюдаемыми 6–8 токен/с — является ли рантайм эффективным?

## Ключевые термины

| Термин | Что говорят | Что это на самом деле |
|--------|-------------|----------------------|
| ANE | «Apple neural engine» | NPU в M-серии и A-серии; объединённая память |
| Hexagon | «Qualcomm NPU» | NPU Snapdragon; доступ через QNN SDK |
| WebGPU | «GPU в браузере» | Стандартизированный W3C API для GPU в браузере; Chrome/Safari 2026 |

| Jetson | «NVIDIA edge» | Семейство Orin Nano / AGX / Thor / T4000 |
| TRT Edge-LLM | «TensorRT для edge» | Edge-порт TensorRT-LLM 2026 года; EAGLE-3 + NVFP4 |
| Unified memory | «общий пул» | CPU и NPU видят одну ОЗУ; нет накладных расходов на копирование |
| Bandwidth-bound | «ограничение памятью» | Декодирование ограничено скоростью чтения весов в байтах/с |
