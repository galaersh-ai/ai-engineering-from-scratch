# Translation guide

## Цель

Сделать не дословный, а понятный русскоязычный перевод курса, сохраняя техническую точность и связь с оригиналом.

## Стиль

- Обращение к читателю: нейтральное, на «вы».
- Тон: ясный, практичный, без канцелярита.
- Код, имена файлов, команды и API не переводим.
- Названия библиотек, моделей, алгоритмов оставляем на английском, если русский вариант не является общепринятым.
- При первом упоминании можно давать пару: `обучение с подкреплением (reinforcement learning, RL)`.

## Термины

| English | Русский вариант | Примечание |
|---|---|---|
| AI engineering | AI-инжиниринг / инженерия ИИ | Выбрать по контексту |
| from scratch | с нуля | |
| inference | инференс | Не «вывод», если речь о ML serving |
| fine-tuning | дообучение / fine-tuning | Можно оставить англ. в скобках |
| embeddings | эмбеддинги | |
| agents | агенты | |
| tool calling | вызов инструментов | |
| RAG | RAG | При первом упоминании раскрыть |

## Процесс перевода

1. Открыть issue `upstream-sync`, если оно есть.
2. Посмотреть список изменённых файлов и commits.
3. Переводить только изменённые участки, не переписывая лишнее.
4. Проверить ссылки и Markdown.
5. Обновить `UPSTREAM.md` до нового commit.
6. Закрыть issue со ссылкой на commit/PR перевода.
